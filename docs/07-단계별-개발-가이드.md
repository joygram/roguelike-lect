# 단계별 개발

**README 전체 순서**: **10~15단계** — **9b**에서 [README **세 폴더 표**](../README.md#folder-layers)로 합의 후 **Unity 구현** 본선. (옛 표기 「7단계」= 문서 번호 07과 혼동 주의.)

**코드 복사 부록(선택)**: [26 C# 코드복사 부록](26-CSharp-코드복사-부록.md) — 긴 C#만 모음. **단계 목표·에셋은 본 문서 표 + 08·09·10…**.  
**준비 확인**: **본 문서 각 Phase**의 *시작 전 준비* / *이전 Phase 완료 후* 표 + 아래 **구현 착수 전 게이트**.  
**세 폴더**: [README `#folder-layers`](../README.md#folder-layers) · **체크·GameEvents·I1~I5**: [#folder-depth](../README.md#folder-depth). **씬·전환·Build·씬별 프리팹**: [#scene-flow](../README.md#scene-flow) · 아래 Phase 0·**Phase별 세 폴더 점검**과 짝. **네 축**: [#dev-flow-axes](../README.md#dev-flow-axes). 인쇄: 세 폴더 [31](31-프레임워크-컨텐츠-UI-구분-및-구현.md) · 씬 흐름 [README #scene-flow](../README.md#scene-flow).

프로그래밍·아트 담당자나 개인이 **직접** 보고, Phase별 할 일을 골라 단계를 따라 구현하는 실무용 문서입니다. 각 Phase의 작업 목록·완료 조건을 체크하면서 진행하고, 막히면 해당 Phase 참고 문서(08, 09, 10 등)를 열어 구체 코드·절차를 따라 하면 됩니다.

게임기획서 리뷰·요구사항·시스템 설계·역할 분담이 끝난 뒤,
**개발을 단계적으로 진행**하기 위한 목표·작업·산출물·완료 조건을 정리한 문서입니다.

**사용 지식·영역**: 각 Phase마다 **어떤 개발 영역·어떤 지식을 사용하는지**를 표기합니다. 세부 정의는 [01 문서 §1.1](01-요구분석.md#11-이-프로젝트에서-사용하는-개발-영역지식-공통-기준)을 참고하세요.

- **단계적 동작**: Phase 0을 끝내면 화면이 나오고, Phase 1을 끝내면 플레이어가 움직이고, … **한 단계마다 실행 가능한 상태**가 됩니다. 다음 Phase에서 **시스템이 확장**됩니다.
- **코드·구현**: 각 Phase에서 필요한 코드와 절차는 **이 문서와 해당 Phase 참고 문서(08, 09, 10 등) 안에만** 있습니다. 외부에서 코드를 찾아보지 않고, 문서에 나온 내용을 **추가하면서** 따라 하면 됩니다.
- **코드 구분**: 구현 문서에서는 **[Unity]** = 유니티 엔진 제공, **[우리 구현]** = 우리가 작성하는 스크립트·함수로 구분합니다. 자세한 내용은 16번 문서 참고.

## 진행 순서 개요

**기간 표기**: **평균 순개발 투입시간**(문서+Unity+C#+검증). [README 일정](../README.md#schedule-estimate) 전제와 동일.

| 단계 | 목표 | 순개발 투입 (평균) |
|------|------|---------------------|
| **Phase 0** | 환경 설정·2D 프로젝트 뼈대 | **4~8시간** |
| **Phase 1** | 플레이어 이동·점프·카메라 | **8~16시간** |
| **Phase 2** | 스테이지·플랫폼 맵 배치 | **14~28시간** (28 기획서 장애물·씬 많으면 상한↑) |
| **Phase 3** | 적·전투·HP·사망 | **14~26시간** |
| **Phase 4** | 게임 오버·UI·재시작 | **8~16시간** |
| **Phase 5** | 연출·사운드·밸런스·테스트 | **8~16시간** |
| **합계 (Phase 0~5)** | 한 판·빌드까지 **6단계** | **약 56~110시간** (위 행 합산과 동일) |

※ 팀·경험·에셋 준비도에 따라 크게 달라집니다.  
※ **사전(01~31)·통합(18~19)·보스(30)** 시간은 [README 일정](../README.md#schedule-estimate) 표 참고.

**“5단계만?”에 대해**: Unity 본선은 **Phase 0·1·2·3·4·5 → 총 6블록**입니다(0을 빼고 세면 5가 되어 헷갈리기 쉬움). **Phase 5** 안에 사운드·밸런스·테스트·빌드가 **한꺼번에** 들어 있어 **주 단위로 쪼개는 것을 권장**합니다. 더 넣을 게 많으면 아래 **확장·분할**을 씁니다.

💪 **독려**: Phase가 많아 보여도, **한 번에 하나씩**만 하면 됩니다. "오늘은 플레이어가 뛰어다니기만 하면 성공!"처럼 작게 생각하면 부담이 덜해요.
