# Part 1. 이펙트 구성 방법

## 1.1 이펙트를 나누는 기준

게임에서 쓰는 이펙트를 **상황·역할**별로 나누어 두면, 스킬·공격·피격과 연결하기 쉽습니다.

| 구분 | 용도 | 생성 위치 예시 | 비고 |
|------|------|----------------|------|
| **공격·타격** | 플레이어 공격이 적에게 닿을 때 | 적 위치 또는 공격 끝점 | 스킬마다 다른 이펙트 가능 |
| **피격** | 플레이어/몬스터가 데미지를 받을 때 | 피격당한 캐릭터 위치 | 공통 1종 또는 캐릭터별 |
| **스킬 발동** | 스킬 사용 시 (범위·강타 등) | 플레이어 위치 또는 지정 위치 | 스킬 데이터에서 프리팹 참조 |
| **몬스터 사망** | 몬스터가 죽을 때 | 몬스터 위치 | 공통 또는 몬스터별 |
| **아이템 획득** | 아이템을 줍을 때 | 아이템 위치 | (선택) |

→ **스킬**은 “공격·타격” 또는 “스킬 발동” 이펙트와 1:1로 연결하고, **SkillData**에 `effectPrefab`을 두면 스킬마다 다른 이펙트를 쓸 수 있습니다.

---

## 1.2 이펙트 프리팹 구조 (권장)

모든 이펙트가 **같은 구조**를 쓰면 관리와 연결이 쉽습니다.

```
[이펙트 루트] (빈 오브젝트)
├── SimpleEffectLife (lifeTime 설정)
└── [시각 요소]
    └── SpriteRenderer + Sprite (또는 Particle System)
```

- **루트**: 위치만 받아서 `Instantiate`할 대상.  
- **SimpleEffectLife**: 일정 시간 후 `Destroy(gameObject)`로 제거.  
- **시각 요소**: 색/스프라이트/파티클 등.  
- (선택) **사운드**는 이펙트가 아니라 **스킬/공격 쪽에서** `SkillData.soundClip`으로 재생하는 것을 권장합니다.

---

## 1.3 이펙트 종류별 정리 (스킬·상황과 매핑)

| 이펙트 이름(예시) | 용도 | 연결 대상 |
|------------------|------|-----------|
| Effect_Hit_Normal | 일반 공격 타격 | 기본 공격 SkillData |
| Effect_Hit_Heavy | 강타 스킬 타격 | 강타 스킬 SkillData |
| Effect_Skill_Area | 범위 스킬 발동 | 범위 스킬 SkillData |
| Effect_Hurt | 피격 시 | PlayerHealth, EnemyController 피격 시 |
| Effect_EnemyDeath | 몬스터 사망 | EnemyController 사망 시 |

- **구성 방법**: 위 표처럼 “이펙트 이름 ↔ 어떤 스킬/상황”을 한 번 정리해 두고,  
  **SkillData**에는 해당 스킬용 이펙트 프리팹만 할당합니다.

---

## 1.4 폴더·네이밍 (구성 일관성)

```
Assets/
├── Prefabs/
│   └── Effects/
│       ├── Effect_Hit_Normal.prefab
│       ├── Effect_Hit_Heavy.prefab
│       ├── Effect_Skill_Area.prefab
│       ├── Effect_Hurt.prefab
│       └── Effect_EnemyDeath.prefab
```

- **네이밍**: `Effect_[용도]_[세부]` 형태로 통일하면, 나중에 스킬 추가 시 “이 스킬에는 Effect_xxx를 쓰자”라고 정하기 쉽습니다.

# Part 2. 나열된 스킬에 맞는 설계·구현 아이디어

## 2.1 스킬 목록 (예시)

아래는 로그라이크에서 다룰 수 있는 **스킬 목록**입니다.  
각 스킬마다 **설계(수치·연출)** 와 **구현 포인트**를 정리했습니다.

| 번호 | 스킬 이름 | 키(예시) | 설계 요약 | 구현 포인트 |
|:----:|-----------|----------|-----------|--------------|
| 1 | **기본 공격** | X 또는 좌클릭 | 짧은 쿨, 낮은 데미지, 근접 | 단일 대상, 즉시 데미지, 타격 이펙트는 적 위치 |
| 2 | **강타** | Z | 긴 쿨, 높은 데미지, 근접 | 기본 공격과 동일 로직, 데미지·쿨·이펙트만 다름 |
| 3 | **범위 공격** | C | 중간 쿨, 범위 내 다수 데미지 | 반경(radius) 체크, 범위 이펙트는 플레이어 또는 지정 위치 |
| 4 | **돌진** | (선택) | 짧은 거리 이동 + 경로상 적 데미지 | 이동 + 충돌 검사, 발동 이펙트는 시작/끝 위치 |

→ **구현 아이디어**: 1·2는 **같은 “단일 타격” 로직**, 3은 **범위(OverlapSphere 등)** 추가, 4는 **이동 + 데미지** 추가.  
**SkillData**에 `range`, `radius`, `isArea` 등만 두면, `DoSkill(SkillData s)` 한 곳에서 분기로 처리 가능합니다.

## 2.2 스킬별 설계 상세

### 스킬 1: 기본 공격

| 항목 | 설계 |
|------|------|
| **역할** | 평타, 짧은 쿨로 연속 사용 |
| **데미지** | 10 전후 (밸런스 조정) |
| **쿨타임** | 0.3~0.5초 |
| **사거리** | 1~1.5 유닛 (근접) |
| **이펙트** | Effect_Hit_Normal, **적이 맞은 위치**에 생성 |
| **사운드** | 짧은 타격음 1종 |

**구현 아이디어**:  
- 레이캐스트 또는 충돌로 “앞쪽 적 1마리” 검출 → 그 위치에 데미지 + `Instantiate(effectPrefab, hitPoint)`.

---

### 스킬 2: 강타

| 항목 | 설계 |
|------|------|
| **역할** | 높은 데미지, 쿨 길어서 신중히 사용 |
| **데미지** | 25~35 (기본 공격의 2~3배) |
| **쿨타임** | 1.5~2.5초 |
| **사거리** | 기본 공격과 동일 (근접) |
| **이펙트** | Effect_Hit_Heavy, **적이 맞은 위치**에 생성 |
| **사운드** | 묵직한 타격음 1종 |

**구현 아이디어**:  
- 기본 공격과 **같은 “단일 타격” 로직** 재사용. **SkillData**만 다르게 (damage, cooldown, effectPrefab, soundClip).  
- `DoSkill(s)`에서 `s.damage`, `s.effectPrefab`만 쓰면 됨.

---

### 스킬 3: 범위 공격

| 항목 | 설계 |
|------|------|
| **역할** | 주변 다수 적에게 동시 데미지 |
| **데미지** | 15~20 (1인당) |
| **쿨타임** | 2~3초 |
| **범위** | 반경 2~3 유닛 (SkillData.radius) |
| **이펙트** | Effect_Skill_Area, **플레이어 위치(또는 발동 지점)** 에 1개 생성 |
| **사운드** | 폭발/충격음 1종 |

**구현 아이디어**:  
- `DoSkill(s)` 안에서 `Physics.OverlapSphere(transform.position, s.radius)` 등으로 범위 내 적 검출.  
- 각 적에게 `s.damage` 적용.  
- 이펙트는 **한 번만** 플레이어 위치에 `Instantiate(s.effectPrefab, transform.position)`.

---

### 스킬 4: 돌진 (선택)

| 항목 | 설계 |
|------|------|
| **역할** | 짧은 거리 이동 + 지나간 적에게 데미지 |
| **데미지** | 10~15 |
| **쿨타임** | 2~3초 |
| **이동 거리** | 3~4 유닛 |
| **이펙트** | 발동 시 플레이어 위치 또는 이동 궤적 |
| **사운드** | 돌진/질주음 1종 |

**구현 아이디어**:  
- 이동은 `Vector3.MoveTowards` 또는 `Rigidbody.MovePosition`으로 짧은 구간 이동.  
- 이동 전/후 또는 매 프레임 경로상에서 `OverlapSphere`로 적 검출 후 1회만 데미지.  
- 이펙트는 시작 위치 또는 이동 중 위치에 생성.

# Part 3. 구현 (동작 보장)

아래 순서대로 구현하면, **스킬 발동 → 데미지 → 이펙트·사운드**가 실제로 동작하도록 만들 수 있습니다.

---

## 3.1 Step 1: SkillData에 이펙트·사운드 필드 넣기

`SkillData`에 아래 필드가 있어야 스킬마다 다른 이펙트·사운드를 쓸 수 있습니다.

```csharp
[CreateAssetMenu(fileName = "Skill", menuName = "Roguelike/Skill Data")]
public class SkillData : ScriptableObject
{
    public string skillName = "기본 공격";
    public int damage = 10;
    public float cooldown = 0.5f;
    public float range = 1.5f;           // 단일 타격 사거리
    public float radius = 0f;             // 0이면 단일, >0이면 범위 스킬
    public GameObject effectPrefab;      // 이펙트 프리팹 (스킬별로 다르게 할당)
    public AudioClip soundClip;          // 발동 사운드
}
```

- **동작 보장**: Inspector에서 각 스킬 에셋에 **effectPrefab**, **soundClip**을 반드시 할당합니다.

---

## 3.2 Step 2: 이펙트 프리팹 준비 (구성 방법 적용)

1. **Effect_Hit_Normal** 등 프리팹 생성.  
2. 루트에 **SimpleEffectLife** (lifeTime 0.3~0.5초), 자식에 시각 요소(Sprite/파티클).  
3. `Prefabs/Effects/` 아래에 저장.  
4. **SkillData** 에셋에서 해당 스킬용 이펙트 프리팹을 참조하도록 연결.

---

## 3.3 Step 3: DoSkill에서 이펙트·사운드 재생 (동작 연결)

**DoSkill(SkillData s)** 안에서 다음을 수행하면, 나열된 스킬 모두에 대해 동작이 보장됩니다.

- **사운드**: 스킬 발동 시 1회 재생.  
- **이펙트**:  
  - **단일 타격**(range만 사용): 레이캐스트 등으로 맞은 위치 `hitPoint` 계산 → `Instantiate(s.effectPrefab, hitPoint)`.  
  - **범위 스킬**(radius 사용): 플레이어 위치에 1개 → `Instantiate(s.effectPrefab, transform.position)`.

```csharp
void DoSkill(SkillData s)
{
    // 1) 사운드 (항상)
    if (s.soundClip != null && SFXPlayer.Instance != null)
        SFXPlayer.Instance.Play(s.soundClip);

    // 2) 단일 타격 (range 사용, radius 없음)
    if (s.radius <= 0f)
    {
        Vector3 hitPoint = GetAttackHitPoint(s.range);  // 레이캐스트 등으로 적 위치
        ApplyDamageAtPoint(hitPoint, s.damage);
        if (s.effectPrefab != null && hitPoint != transform.position)
            Instantiate(s.effectPrefab, hitPoint, Quaternion.identity);
    }
    // 3) 범위 스킬 (radius 사용)
    else
    {
        ApplyDamageInRadius(transform.position, s.radius, s.damage);
        if (s.effectPrefab != null)
            Instantiate(s.effectPrefab, transform.position, Quaternion.identity);
    }
}
```

- `GetAttackHitPoint`: 앞쪽으로 레이캐스트해 적이 있으면 hit point, 없으면 플레이어 앞쪽 끝점 반환.  
- `ApplyDamageAtPoint`: 해당 위치에 있는 적 1마리에게 데미지.  
- `ApplyDamageInRadius`: OverlapSphere로 범위 내 적들에게 데미지.

→ 이렇게 하면 **나열된 스킬(기본 공격, 강타, 범위 공격)** 이 모두 **SkillData만 다르게** 같은 코드로 동작하고, **스킬별 이펙트·사운드**가 연결됩니다.

---

## 3.4 Step 4: 검증 (동작 보장 체크리스트)

| 확인 항목 | 방법 |
|-----------|------|
| 스킬 사용 시 사운드 재생 | 각 스킬 키 입력 → 해당 스킬 사운드 1회 재생 |
| 단일 타격 이펙트 위치 | 기본 공격/강타 사용 → **적 위치**에 이펙트 생성 |
| 범위 스킬 이펙트 위치 | 범위 공격 사용 → **플레이어 위치**에 범위 이펙트 1개 생성 |
| 스킬마다 다른 이펙트 | 기본 공격 = 작은 이펙트, 강타 = 큰 이펙트 등 시각적으로 구분 |
| 쿨타임 동작 | 연타 시 쿨 만료 후에만 다시 발동 |

위를 체크하면 **이펙트 구성**과 **스킬 설계·구현**이 문서대로 동작하는지 확인할 수 있습니다.

## 3.5 구현 순서 요약 (동작되도록 하는 순서)

| 순서 | 할 일 | 결과 |
|:----:|-------|------|
| 1 | SkillData에 effectPrefab, soundClip, range, radius 추가 | 스킬별로 이펙트·사운드·범위 지정 가능 |
| 2 | 이펙트 프리팹 N개 구성 (Normal, Heavy, Area 등), SimpleEffectLife 부착 | 스킬에 할당할 에셋 준비 |
| 3 | DoSkill에서 soundClip 재생 + effectPrefab을 적 위치/플레이어 위치에 Instantiate | 스킬 사용 시 사운드·이펙트 동작 |
| 4 | 단일 타격용 GetAttackHitPoint / ApplyDamageAtPoint 구현 | 기본 공격·강타 동작 |
| 5 | 범위용 ApplyDamageInRadius 구현 | 범위 공격 동작 |
| 6 | 검증 체크리스트로 확인 | 동작 보장 완료 |

# Part 4. 한눈에 보기

| 항목 | 내용 |
|------|------|
| **이펙트 구성** | 상황별(타격/피격/스킬 발동/사망) 분류, 프리팹 구조(루트+SimpleEffectLife+시각), 스킬↔이펙트 매핑표 |
| **스킬 설계** | 기본 공격·강타·범위 공격·(돌진) 목록, 스킬별 damage/cooldown/range/radius/effect/sound 설계 |

| **동작 보장** | SkillData 필드 → 이펙트 프리팹 연결 → DoSkill에서 재생/Instantiate → 검증 체크리스트로 확인 |

이 문서를 따라 구현하면 **이펙트 구성 방법**과 **나열된 스킬에 맞는 설계·구현**이 **실제로 동작하는 형태**로 완성됩니다.
