# 06. 사운드·빌드 — 에셋

> **시스템**: 오디오·빌드 | **에셋 담당**: D2 (BGM · SFX 파일 수집 및 임포트 설정)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

사운드는 게임에서 시각과 함께 플레이어의 행동에 즉각적인 피드백을 준다. 어떤 포맷을 쓰느냐, Unity에서 어떻게 로드하느냐에 따라 게임 성능과 재생 지연이 달라진다. 파일을 가져오기 전에 이 원리를 이해하면 설정이 자연스럽게 따라온다.

## BGM과 SFX의 역할 차이

BGM과 효과음은 역할이 다르고, 그 차이가 파일 포맷과 로드 방식 선택에 직결된다.

| 항목 | BGM | SFX |
|------|-----|-----|
| 역할 | 게임 분위기를 유지하는 배경 음악 | 행동에 즉각 반응하는 피드백 신호 |
| 길이 | 1~3분, 루프 | 0.1~2초 |
| 재생 방식 | 연속 루프 | 이벤트 발생 시 즉시 재생 |
| 여러 개 동시 재생 | 불필요 | 필요 (공격음이 겹쳐 재생될 수 있음) |

BGM이 너무 크면 SFX가 묻힌다. 반대로 SFX가 없으면 게임이 무감각해진다. 두 가지의 균형이 중요하다. 일반적으로 BGM 볼륨을 SFX의 절반 정도로 설정하는 것이 기준이 된다.

---

## 파일 포맷 선택 이유

### BGM — `.ogg` 권장

- `.ogg`는 손실 압축 포맷이다. 원본 대비 파일 크기가 작지만, 압축 과정에서 일부 주파수가 제거된다.
- BGM은 길이가 길어서 `.wav` (비압축) 로 저장하면 파일이 수십 MB가 되어 게임 용량이 급격히 커진다.
- 사람의 귀로는 `.ogg`와 `.wav`의 차이를 BGM에서 거의 구분하지 못한다.
- Unity에서 `.ogg` BGM은 `Streaming` 로드로 설정해 메모리에 전부 올리지 않고 재생하면서 파일에서 읽어온다.

### SFX — `.wav` 권장

- `.wav`는 비압축 포맷이다. 파일 크기가 크지만, 디코딩 과정이 없어 즉시 재생된다.
- 효과음은 짧아서 파일 크기가 커봤자 수백 KB 수준이다. 비압축이 용량에 큰 영향을 주지 않는다.
- Unity에서 `Decompress On Load`로 설정해 메모리에 풀어둔 채로 대기시키면 이벤트 발생 즉시 재생 지연 없이 나온다.
- `.ogg`로 SFX를 만들면 압축 해제 시간 때문에 미세한 지연이 생길 수 있다.

## 사운드 설계 원칙

어떤 행동에 어떤 소리를 붙일지는 단순히 "소리가 있으면 좋다"가 아니라, 플레이어에게 **무엇을 알려주는가**를 기준으로 설계한다.

| 행동 | 소리가 전달하는 정보 |
|------|---------------------|
| 공격 | "지금 공격이 발동됐다" |
| 타격 | "공격이 적에게 맞았다" |
| 피격 | "플레이어가 데미지를 받았다" |
| 점프 | "점프 중이다" |
| 적 사망 | "적이 제거됐다" |
| 게임 오버 | "게임이 끝났다" |

공격음과 타격음을 구분하는 이유: 공격음은 버튼을 눌렀을 때 나오고, 타격음은 실제로 적에게 맞았을 때 나온다. 둘이 같으면 공격이 빗나갔는지 맞았는지 소리로 구분할 수 없다.

## 파일 목록 및 저장 위치

```
Assets/Audio/
  BGM/
    bgm_game.ogg          ← 게임 플레이 BGM (필수)
    bgm_menu.ogg          ← 메인 메뉴 BGM (선택)
  SFX/
    sfx_attack.wav        ← 공격 발동 시
    sfx_hit.wav           ← 공격이 적에게 맞았을 때
    sfx_player_hurt.wav   ← 플레이어 피격 시
    sfx_game_over.wav     ← 게임 오버 시
    sfx_enemy_die.wav     ← 적 사망 시 (선택)
    sfx_jump.wav          ← 점프 시 (선택)
```

## 무료 오디오 리소스

| 사이트 | 주요 라이선스 | 특징 |
|--------|:-----------:|------|
| [freesound.org](https://freesound.org) | CC0 / CC-BY 혼재 | 효과음 중심, 파일별 확인 필요 |
| [opengameart.org](https://opengameart.org) | CC0 많음 | 게임 전용, 패키지 형태 |
| [itch.io 무료 에셋](https://itch.io/game-assets/free) | 각 에셋 개별 | BGM+SFX 세트 묶음 있음 |

**라이선스 확인이 중요한 이유**: 무료라도 저작권이 있는 파일을 출처 표기 없이 쓰면 법적 문제가 생길 수 있다. 특히 게임을 공개하거나 발표할 때는 반드시 라이선스를 확인한다.

- **CC0**: 저작권 없음, 자유롭게 사용
- **CC-BY**: 사용 가능, 출처 표기 필요
- **CC-BY-NC**: 비상업적 용도에만 사용 가능

## Unity 임포트 설정

### BGM

1. `bgm_game.ogg` 선택 → Inspector
2. **Load Type** → `Streaming`
3. **Compression Format** → `Vorbis`
4. **Load In Background** → 체크 (씬 로드 중 백그라운드에서 미리 읽기 시작)
5. **Apply**

### SFX

1. SFX `.wav` 파일 선택 → Inspector
2. **Load Type** → `Decompress On Load`
3. **Compression Format** → `PCM`
4. **Apply**
