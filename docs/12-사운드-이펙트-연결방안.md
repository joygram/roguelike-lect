# 사운드 및 이펙트 연결 방안

**사운드(BGM·효과음)** 와 **이펙트(시각 효과)** 를 게임 상황에 어떻게 연결할지 정리한 문서입니다.  
“어떤 순간에 무엇을 재생/표시할지”와 “코드에서 어떻게 연결할지”를 중심으로 설명합니다.

**다른 시스템과의 순서**: 사운드는 **그 이벤트가 나는 코드**(전투·트리거·UI 클릭)와 **같은 시점**에 붙이면 됩니다. 맵 트리거·`GameEvents`·Phase 배치는 루트 README **[인터랙션·이벤트](../README.md#interaction-events)**.

## Phase 5 — 사운드·연출 단계 목표·에셋

| | |
|--|--|
| **단계 목표** | 공격·피격·사망·BGM·UI 클릭 등 **이벤트 ↔ 사운드·이펙트** 한 줄 연결 |
| **필수 에셋** | **BGM 1** + **효과음 최소 3종**(공격·피격·게임오버/버튼 등). 이펙트는 (선) [09·15](09-이펙트-몬스터-스킬-레벨-가이드.md) 프리팹 재사용 |
| **진행 뼈대** | [07 Phase 5](07-단계별-개발-가이드.md) |

# Part 1. 연결 개요

## 1.1 기본 생각

- **한 게임 이벤트**가 일어날 때, **사운드**와 **이펙트**를 **같은 시점**에서 각각 재생·생성하면 됩니다.
- 예: “플레이어가 몬스터를 공격했다” → **공격 효과음** 재생 + **타격 이펙트** 생성.

---

## 1.2 이벤트별 연결표 (참고)

| 게임 이벤트 | 사운드 | 이펙트 | 비고 |
|-------------|--------|--------|------|
| **게임 시작** | BGM 재생 | (선택) 타이틀 연출 | BGM은 루프 |
| **플레이어 공격** | 공격 효과음 | 공격/타격 이펙트 (공격 위치 또는 적 위치) | 동시에 실행 |
| **플레이어 피격** | 피격 효과음 | 피격 이펙트 (플레이어 위치) | 동시에 실행 |
| **몬스터 사망** | 처치 효과음 (선택) | 사망 이펙트 (몬스터 위치) | 동시에 실행 |
| **아이템 획득** | 획득 효과음 | 획득 이펙트 (아이템 위치) | 동시에 실행 |
| **게임 오버** | 게임 오버 효과음, BGM 정지(선택) | (선택) 화면 연출 | 오버 시 1회 |
| **레벨업** | (선택) 레벨업 효과음 | (선택) 레벨업 이펙트 | 동시에 실행 |

→ 위 표를 기준으로 “이 이벤트일 때 이 사운드 + 이 이펙트”만 정해 두면, 코드에서 **한 군데에서 둘 다 호출**하면 됩니다.

---

# Part 2. 사운드 연결 방법

## 2.1 방안 비교

| 방안 | 난이도 | 장점 | 단점 | 추천 |
|------|--------|------|------|:----:|
| **A. 오브젝트별 AudioSource** | ★☆☆ | 구현 단순, 해당 오브젝트에서만 재생 | 소리 많을 때 관리 어려움 | **소수 효과음** |
| **B. 전역 AudioSource + 클립만 바꿔 재생** | ★★☆ | 씬에 오디오 1~2개로 정리 | 동시 재생 개수 제한 | **가장 추천** |
| **C. AudioManager(싱글톤)** | ★★★ | BGM·효과음 통합, 볼륨 등 제어 쉬움 | 초기 구성 필요 | 효과음 종류 많을 때 |

---

## 2.2 방안 B: 전역 AudioSource + 클립만 바꿔 재생 (가장 쉬움 ★)

### (1) 구성

- **BGM용 AudioSource** 1개: Loop 체크, Play On Awake(선택).
- **효과음용 AudioSource** 1개: `PlayOneShot(클립)` 으로 짧은 소리 여러 번 재생 가능.

### (2) 효과음 한 번 재생 (가장 쉬운 연결)

```csharp
// 씬에 빈 오브젝트 "SFXPlayer" 만들고 AudioSource 붙인 뒤
public class SFXPlayer : MonoBehaviour
{
    public static SFXPlayer Instance;  // (선택) 간단한 싱글톤
    public AudioSource source;

    void Awake() { Instance = this; }

    public void Play(AudioClip clip)
    {
        if (clip != null && source != null)
            source.PlayOneShot(clip);
    }
}
```

- **연결**: 공격/피격/아이템 획득 등 **이벤트가 일어나는 스크립트**에서 `SFXPlayer.Instance.Play(attackClip);` 처럼 호출.

### (3) BGM 재생

- BGM용 AudioSource에 클립 할당 후 `Play()` 한 번 호출 (게임 시작 또는 해당 씬 로드 시).  
- 게임 오버 시 `Stop()` 또는 볼륨 0으로 끄면 됨.

---

## 2.3 이벤트 위치에서 사운드만 연결하는 예

```csharp
// 플레이어 공격이 성공했을 때 (예: PlayerCombat.cs)
void OnAttackHit(Vector3 hitPosition)
{
    SFXPlayer.Instance?.Play(attackHitClip);  // 사운드 연결
}
```

```csharp
// 플레이어가 데미지를 받았을 때 (예: PlayerHealth.cs)
public void TakeDamage(int amount)
{
    currentHp -= amount;
    SFXPlayer.Instance?.Play(hurtClip);      // 사운드 연결
    if (currentHp <= 0) OnDeath();
}
```

- `attackHitClip`, `hurtClip` 등은 Inspector에서 AudioClip으로 할당해 두면 됩니다.

---

# Part 3. 이펙트 연결 방법

## 3.1 이벤트 위치에서 이펙트만 연결하는 예

- 이펙트는 **해당 이벤트가 일어난 위치**에 프리팹을 생성하면 됩니다. (기존 09 문서 방식과 동일.)

```csharp
// 플레이어 공격이 성공했을 때 (같은 함수에서 이펙트도 호출)
void OnAttackHit(Vector3 hitPosition)
{
    if (hitEffectPrefab != null)
        Instantiate(hitEffectPrefab, hitPosition, Quaternion.identity);  // 이펙트 연결
}
```

- **위치**: 공격이 닿은 지점(`hitPosition`) 또는 몬스터/플레이어 transform 위치.

---

# Part 4. 사운드 + 이펙트 동시 연결 (한 군데에서 호출)

## 4.1 같은 이벤트에서 둘 다 호출

아래처럼 **한 함수**에서 **사운드**와 **이펙트**를 같이 호출하면, 항상 동시에 재생·표시됩니다.

```csharp
// 예: 공격이 적에게 닿았을 때
void OnAttackHit(Vector3 hitPosition)
{
    // 사운드
    SFXPlayer.Instance?.Play(attackHitClip);
    // 이펙트
    if (hitEffectPrefab != null)
        Instantiate(hitEffectPrefab, hitPosition, Quaternion.identity);
}
```

```csharp
// 예: 플레이어가 데미지를 받았을 때
public void TakeDamage(int amount)
{
    currentHp -= amount;
    SFXPlayer.Instance?.Play(hurtClip);
    if (hurtEffectPrefab != null)
        Instantiate(hurtEffectPrefab, transform.position, Quaternion.identity);
    if (currentHp <= 0) OnDeath();
}
```

```csharp
// 예: 몬스터 사망 시
void OnEnemyDeath()
{
    SFXPlayer.Instance?.Play(deathClip);
    if (deathEffectPrefab != null)
        Instantiate(deathEffectPrefab, transform.position, Quaternion.identity);
    Destroy(gameObject);
}
```

---

## 4.2 연결 포인트 정리표

| 이벤트가 일어나는 스크립트 | 호출 시점 | 사운드 | 이펙트 |
|----------------------------|-----------|--------|--------|
| PlayerCombat | 공격이 적에게 닿음 | attackHitClip | hitEffectPrefab |
| PlayerHealth | TakeDamage() | hurtClip | hurtEffectPrefab |
| EnemyController | 사망 처리 직전 | deathClip | deathEffectPrefab |
| ItemPickup | 아이템 획득 시 | getItemClip | getItemEffectPrefab |
| GameManager | 게임 오버 처리 시 | gameOverClip | (선택) |

- **BGM**: GameManager 또는 별도 BGM 오브젝트에서 게임 시작/씬 로드 시 `AudioSource.Play()`, 게임 오버 시 `Stop()` 또는 볼륨 0.

---

# Part 5. (선택) 데이터로 연결하기 — ScriptableObject

효과음·이펙트를 **데이터**로 묶어 두면, 기획에서 “이 공격에는 이 소리 + 이 이펙트”만 바꿔 끼우기 편합니다.

```csharp
// AttackHitData.cs (ScriptableObject 예시)
[CreateAssetMenu(menuName = "Roguelike/Hit Data")]
public class AttackHitData : ScriptableObject
{
    public AudioClip hitSound;
    public GameObject hitEffectPrefab;
}
```

- **연결**: `OnAttackHit` 등에서 `AttackHitData`를 참조하고,  
  `SFXPlayer.Instance?.Play(data.hitSound);`  
  `if (data.hitEffectPrefab != null) Instantiate(data.hitEffectPrefab, hitPosition, Quaternion.identity);`  
  이렇게 하면 “어떤 소리 + 어떤 이펙트”를 에셋만 바꿔서 조합할 수 있습니다.
