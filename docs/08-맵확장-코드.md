## 구분하기: 내가 만드는 것 vs Unity(API)가 주는 것

| 구분 | 이번 단계 예시 | 설명 |
|------|----------------|------|
| **내가 만드는 것** | 스테이지별 플래그(hasConveyor 등), 출구 열림 조건, 경고 시간·복원 시간 | 맵 규칙·데이터 설계는 **우리 책임** |
| **Unity(API)가 주는 것** | `OnCollisionStay2D`, `OnCollisionEnter2D`, `OnTriggerEnter2D`, `ContactPoint2D.normal`, `WaitForSeconds` | 충돌·접촉면 판별·시간 대기는 **엔진**이 제공 |

## 구현 개요

**스크립트 역할 요약**

| 스크립트 | 부착 위치 | 역할 |
|----------|-----------|------|
| **LevelSettingsData** | 에셋(ScriptableObject) | 스테이지 ID·다음 씬·장애물 플래그 |
| **LevelBootstrap** | 씬 빈 오브젝트 | Start에서 SO 읽어 장애물 그룹 SetActive |
| **ConveyorBelt** | 컨베이어 오브젝트 | 위에 서 있으면 X축 속도 부여 |

| **StageExit** | 출구 트리거 | Player 진입 시 다음 씬 로드 (필요 시 StageCleared 후 열림) |
| **EnemyTracker** | (선택) 씬 오브젝트 | 적 전멸 시 GameEvents.RaiseStageCleared |

## LevelSettingsData — 스테이지 설정 에셋

### 개요

- **역할**: 스테이지마다 **다음 씬 이름**, **이 스테이지에 컨베이어/낙하 발판이 있는지** 플래그를 에셋으로 담는다. LevelBootstrap이 이 에셋을 읽어 해당 그룹만 켠다.
- **부착**: ScriptableObject 에셋. 씬에 오브젝트 없음.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **ScriptableObject** | 씬을 복사해 재사용할 때, 스테이지마다 "어떤 장애물을 켤지"만 에셋으로 바꾸면 된다. 코드 수정 없이 스테이지 구성 변경 가능. |

### SerializeField를 쓴 이유

모든 필드는 에셋 데이터이므로 public 또는 SerializeField로 Inspector 편집. SO는 에셋 파일이 데이터이므로 public이 일반적.

### 사용하는 Unity API (LevelSettingsData)

| API | 역할 |
|-----|------|
| `ScriptableObject` | 씬과 분리된 데이터 에셋 |
| `[CreateAssetMenu]` | Project 우클릭 메뉴에 생성 항목 추가 |

### 코드 (LevelSettingsData.cs)

**파일**: `Assets/Scripts/LevelSettingsData.cs`. 에셋은 Create → Roguelike → Level Settings.

```csharp
using UnityEngine;

/// <summary>
/// 스테이지마다 다른 설정(사용 장애물, 다음 씬)을 담는 데이터 에셋입니다.
/// LevelBootstrap이 이 에셋을 읽어 해당 스테이지에 맞는 장애물을 활성화합니다.
/// </summary>
[CreateAssetMenu(fileName = "LevelSettings", menuName = "Roguelike/Level Settings")]
public class LevelSettingsData : ScriptableObject
{
    public string stageId           = "Stage_1_1";
    public string stageName         = "공장 1층";
    public bool   hasConveyor       = false;
    public bool   hasFallingPlatform = false;
    public string nextSceneName     = "Stage_1_2";
}
```

## LevelBootstrap — 장애물 그룹 활성화

### 개요

- **역할**: 씬 시작 시 **LevelSettingsData**를 읽어, hasConveyor·hasFallingPlatform 플래그에 따라 해당 **장애물 그룹 GameObject**를 SetActive(true/false)한다. 같은 씬 레이아웃을 쓰면서 스테이지별로 다른 장애물만 켤 수 있다.
- **부착**: 스테이지 씬에 빈 오브젝트 하나. 씬마다 한 개.
- **흐름**: `Start()`에서 settings 확인 → conveyorGroup.SetActive(settings.hasConveyor), fallingPlatformGroup.SetActive(settings.hasFallingPlatform).

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **그룹 단위 SetActive** | 장애물을 개별로 켜고 끄는 대신, Hierarchy에서 "컨베이어 그룹", "낙하 발판 그룹"으로 묶어 한 번에 켜거나 끈다. 구성 변경이 쉬워진다. |
| **SO 플래그** | 스테이지마다 "이 스테이지에는 컨베이어 있다/없다"만 에셋에서 바꾸면 된다. 코드 수정 불필요. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `settings` | **씬에서 연결**. 이 스테이지용 LevelSettingsData 에셋을 Inspector에서 지정한다. |
| `conveyorGroup`, `fallingPlatformGroup` | **씬에서 연결**. Hierarchy의 장애물 그룹 오브젝트를 드래그로 연결한다. |

### 사용하는 Unity API (LevelBootstrap)

| API | 역할 |
|-----|------|
| `GameObject.SetActive(bool)` | 오브젝트 활성/비활성 전환 |
| `void Start()` | 씬 시작 시 1회 실행 |
| `Debug.LogWarning(msg)` | 경고 메시지 출력 (오류는 아니지만 주의가 필요할 때) |

```csharp
using UnityEngine;

/// <summary>
/// 씬 시작 시 LevelSettingsData를 읽어 장애물 그룹을 활성화/비활성화합니다.
/// 씬마다 하나씩 배치합니다.
/// </summary>
public class LevelBootstrap : MonoBehaviour
{
    [SerializeField] LevelSettingsData settings;

    [Header("장애물 그룹 오브젝트 (Hierarchy에서 연결)")]
    [SerializeField] GameObject conveyorGroup;
    [SerializeField] GameObject fallingPlatformGroup;

    void Start()
    {
        if (settings == null)
        {
            Debug.LogWarning("LevelBootstrap: LevelSettingsData가 연결되지 않았습니다.");
            return;
        }

        // SO의 플래그에 따라 해당 그룹 활성화
        if (conveyorGroup != null)
            conveyorGroup.SetActive(settings.hasConveyor);

        if (fallingPlatformGroup != null)
            fallingPlatformGroup.SetActive(settings.hasFallingPlatform);
    }
}
```

## ConveyorBelt — 컨베이어 벨트

### 개요

- **역할**: 플레이어가 **위에 올라서 있는 동안** X축 속도를 `beltSpeed`로 강제한다. `OnCollisionStay2D`에서 접촉 법선이 "위에서 밟음"(normal.y < -0.5f)일 때만 Rigidbody2D의 linearVelocity.x를 설정한다.
- **부착**: 컨베이어 벨트 오브젝트(콜라이더 있음).
- **흐름**: 매 프레임 접촉 유지 시 OnCollisionStay2D → contacts에서 normal 확인 → 위에서 밟았으면 linearVelocity.x = beltSpeed.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **OnCollisionStay2D** | "위에 서 있는 동안" 계속 속도를 줘야 하므로, 접촉이 유지되는 매 프레임 호출되는 Stay를 쓴다. Enter는 처음 닿을 때 한 번뿐이다. |
| **contact.normal.y < -0.5f** | 접촉 법선이 아래쪽을 가리키면(플레이어가 발로 밟고 있으면) normal.y가 음수이다. 옆면 충돌은 수평이라 이 조건에 안 걸린다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `beltSpeed` | **기획/밸런스 수치**. 양수=오른쪽, 음수=왼쪽. Inspector에서 방향·속도 조절. |

### 사용하는 Unity API (ConveyorBelt)

| API | 역할 |
|-----|------|
| `OnCollisionStay2D(Collision2D)` | 두 콜라이더가 계속 접촉 중인 동안 매 프레임 호출 |
| `Collision2D.contacts` | 접촉 지점 배열. 법선 방향으로 어느 면에서 닿았는지 판별 |
| `ContactPoint2D.normal` | 접촉 지점의 법선 벡터. 위에서 밟으면 y < 0 |
| `Rigidbody2D.linearVelocity` | 오브젝트의 현재 속도 벡터 (Unity 6 이상) |

### 코드 (ConveyorBelt.cs)

**파일**: `Assets/Scripts/ConveyorBelt.cs`. 컨베이어 오브젝트에 부착.

```csharp
using UnityEngine;

/// <summary>
/// 플레이어가 위에 올라서면 beltSpeed 방향으로 밀어냅니다.
/// 양수 = 오른쪽, 음수 = 왼쪽.
/// </summary>
public class ConveyorBelt : MonoBehaviour
{
    [Tooltip("이동 속도. 양수 = 오른쪽, 음수 = 왼쪽")]
    [SerializeField] float beltSpeed = 3f;

    void OnCollisionStay2D(Collision2D col)
    {
        foreach (ContactPoint2D contact in col.contacts)
        {
            // 법선 y가 -0.5 미만이면 이 오브젝트의 위쪽에서 닿은 것 (플레이어가 위에 있음)
            if (contact.normal.y < -0.5f)
            {
                var rb = col.rigidbody;
                if (rb != null)
                {
                    // X축만 강제 설정, Y축(중력·점프)은 유지
                    // Unity 6 미만: rb.velocity = new Vector2(beltSpeed, rb.velocity.y)
                    rb.linearVelocity = new Vector2(beltSpeed, rb.linearVelocity.y);
                }
                break;
            }
        }
    }
}
```

> Unity 2022 이하에서는 `rb.linearVelocity` 대신 `rb.velocity`를 사용한다.

## FallingPlatform — 밟으면 사라지는 발판

### 개요

- **역할**: 플레이어가 **위에서 밟으면** 경고 색상 → 일정 시간 후 콜라이더 비활성·투명 → 다시 일정 시간 후 복원. **코루틴**으로 "경고 대기 → 사라짐 대기 → 복원" 순서를 한 번에 표현한다.
- **부착**: 낙하 발판 오브젝트(SpriteRenderer, Collider2D 있음).
- **흐름**: OnCollisionEnter2D에서 위에서 밟음 확인 → `StartCoroutine(FallSequence)` → yield로 단계별 대기·색·enabled 변경.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **코루틴** | "경고 N초 → 사라짐 N초 → 복원"처럼 **시간 순서가 있는 단계**를 Update 타이머 여러 개로 관리하면 변수가 많아진다. 코루틴은 yield return WaitForSeconds로 흐름을 읽기 쉽게 쓸 수 있다. |
| **contact.normal.y < -0.5f** | 위에서 밟은 경우만 트리거. 옆면 접촉은 제외. |
| **_triggered 플래그** | 코루틴이 진행 중일 때 다시 OnCollisionEnter2D가 불려도 시퀀스를 중복 실행하지 않도록 막는다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `warningDelay` | **기획/튜닝**. 밟은 후 경고 색이 유지되는 시간(초). |
| `disappearTime` | **기획/튜닝**. 발판이 사라져 있는 시간(초). |
| `warningColor` | **기획/연출**. 경고 시 적용할 색. Inspector에서 조절. |

### 사용하는 Unity API (FallingPlatform)

| API | 역할 |
|-----|------|
| `OnCollisionEnter2D(Collision2D)` | 두 콜라이더가 **처음 충돌**했을 때 1회 호출 |
| `ContactPoint2D.normal` | 어느 면에서 충돌했는지 판별 |
| `StartCoroutine(IEnumerator)` | 코루틴 시작 |
| `IEnumerator` + `yield return new WaitForSeconds(t)` | t초 동안 실행을 일시정지 후 재개 |
| `Collider2D.enabled` | 콜라이더 활성/비활성 (false면 통과 가능) |
| `SpriteRenderer.color` | 스프라이트 색상 변경 |

### 코드 (FallingPlatform.cs)

**파일**: `Assets/Scripts/FallingPlatform.cs`. 발판 오브젝트에 부착.

```csharp
using System.Collections;
using UnityEngine;

/// <summary>
/// 플레이어가 밟으면 경고 후 사라지고, 일정 시간 후 복원됩니다.
/// </summary>
public class FallingPlatform : MonoBehaviour
{
    [SerializeField] float warningDelay  = 0.8f;    // 밟은 후 경고 지속 시간
    [SerializeField] float disappearTime = 2.5f;    // 사라져 있는 시간
    [SerializeField] Color warningColor  = new Color(1f, 0.3f, 0.3f);

    private SpriteRenderer _sr;
    private Collider2D     _col;
    private Color          _originalColor;
    private bool           _triggered;

    void Awake()
    {
        _sr  = GetComponentInChildren<SpriteRenderer>();
        _col = GetComponent<Collider2D>();
        if (_sr != null) _originalColor = _sr.color;
    }

    void OnCollisionEnter2D(Collision2D col)
    {
        if (_triggered) return;   // 이미 시퀀스 진행 중

        // 위에서 밟은 경우만 트리거
        foreach (ContactPoint2D contact in col.contacts)
        {
            if (contact.normal.y < -0.5f)
            {
                _triggered = true;
                StartCoroutine(FallSequence());
                break;
            }
        }
    }

    IEnumerator FallSequence()
    {
        // 1단계: 경고 색상으로 변경
        if (_sr != null) _sr.color = warningColor;
        yield return new WaitForSeconds(warningDelay);

        // 2단계: 투명 처리 + 콜라이더 비활성 (통과 가능)
        if (_sr != null)  _sr.color     = new Color(0, 0, 0, 0);
        if (_col != null) _col.enabled  = false;
        yield return new WaitForSeconds(disappearTime);

        // 3단계: 복원
        if (_sr != null)  _sr.color    = _originalColor;
        if (_col != null) _col.enabled = true;
        _triggered = false;
    }
}
```

## StageExit — 출구 트리거

### 개요

- **역할**: **Player**가 출구 **Trigger**에 들어오면 다음 씬으로 이동한다. `requireEnemiesClear`가 true이면 `GameEvents.StageCleared`를 구독해, 적 전멸 후에만 `_isOpen`이 true가 되어 출구가 열린다.
- **부착**: 출구 오브젝트(BoxCollider2D Is Trigger).
- **흐름**: Start에서 requireEnemiesClear면 StageCleared 구독, 아니면 처음부터 _isOpen=true → OnTriggerEnter2D에서 Player이고 _isOpen이면 GameManager.GoToScene(nextSceneName).

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **OnTriggerEnter2D** | 출구는 "닿으면" 씬 전환이므로 물리 충돌(튕김)이 필요 없다. Trigger로 겹침만 감지한다. |
| **StageCleared 구독** | "적 전멸 후 출구 열림"을 StageExit이 직접 적 수를 세지 않고, 다른 스크립트(EnemyTracker 등)가 RaiseStageCleared를 부르면 열리게 한다. 책임 분리. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `nextSceneName` | **씬/기획**. 다음 스테이지 씬 이름. 스테이지마다 다르므로 Inspector에서 지정. |
| `requireEnemiesClear` | **기획**. true면 적 전멸 후에만 출구 활성. 스테이지별로 다르게 둘 수 있다. |

### 사용하는 Unity API (StageExit)

| API | 역할 |
|-----|------|
| `OnTriggerEnter2D(Collider2D)` | Is Trigger 콜라이더에 다른 콜라이더가 들어왔을 때 호출 |
| `Collider2D.CompareTag(string)` | 오브젝트의 Tag 비교 |
| `GameManager.Instance.GoToScene` | 직접구현 — GameManager에 씬 이동 요청 |

### 코드 (StageExit.cs)

**파일**: `Assets/Scripts/StageExit.cs`. 출구 오브젝트에 부착.

```csharp
using UnityEngine;

/// <summary>
/// 플레이어가 출구에 들어오면 다음 씬으로 이동합니다.
/// requireEnemiesClear가 true이면 적 전멸 후에만 열립니다.
/// </summary>
public class StageExit : MonoBehaviour
{
    [SerializeField] string nextSceneName;
    [Tooltip("true이면 모든 적 처치 후에만 출구가 열립니다")]
    [SerializeField] bool requireEnemiesClear = false;

    private bool _isOpen;

    void Start()
    {
        // 적 전멸 조건이 없으면 처음부터 열려 있음
        _isOpen = !requireEnemiesClear;

        if (requireEnemiesClear)
            GameEvents.StageCleared += OnStageCleared;
    }

    void OnDestroy()
    {
        GameEvents.StageCleared -= OnStageCleared;
    }

    void OnStageCleared() => _isOpen = true;

    void OnTriggerEnter2D(Collider2D other)
    {
        if (!_isOpen) return;
        if (!other.CompareTag("Player")) return;

        if (string.IsNullOrEmpty(nextSceneName))
        {
            Debug.LogWarning("StageExit: nextSceneName이 비어 있습니다.");
            return;
        }
        GameManager.Instance?.GoToScene(nextSceneName);
    }
}
```

## EnemyTracker — 적 전멸 시 StageCleared (선택)

### 개요

- **역할**: 씬에 있는 **Enemy** Tag 오브젝트 수를 Start에서 세고, 적이 사망할 때마다 `NotifyEnemyDead()`가 호출되면 감소시킨다. 0이 되면 `GameEvents.RaiseStageCleared()`를 발행해 StageExit이 출구를 열 수 있게 한다.
- **부착**: 씬 빈 오브젝트(예: GameManager 또는 전용 오브젝트). EnemyController 사망 처리에서 `FindObjectOfType<EnemyTracker>()?.NotifyEnemyDead()` 호출 필요.

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `enemyTag` | **프로젝트 설정**. 기본 "Enemy". Tag 이름이 다르면 Inspector에서 변경. |

### 사용하는 Unity API (EnemyTracker)

| API | 역할 |
|-----|------|
| `GameObject.FindGameObjectsWithTag(tag)` | 씬에서 해당 Tag를 가진 모든 오브젝트 배열 반환 |

```csharp
using UnityEngine;

/// <summary>
/// 씬의 Enemy 태그 오브젝트 수를 추적해, 모두 사망하면 StageCleared를 발행합니다.
/// EnemyController의 사망 처리에서 NotifyEnemyDead()를 호출해 주세요.
/// </summary>
public class EnemyTracker : MonoBehaviour
{
    [SerializeField] string enemyTag = "Enemy";
    private int _remaining;

    void Start()
    {
        _remaining = GameObject.FindGameObjectsWithTag(enemyTag).Length;
        if (_remaining == 0) GameEvents.RaiseStageCleared();
    }

    public void NotifyEnemyDead()
    {
        _remaining = Mathf.Max(0, _remaining - 1);
        if (_remaining <= 0) GameEvents.RaiseStageCleared();
    }
}
```

**파일**: `Assets/Scripts/EnemyTracker.cs`. (선택) 씬에 부착. **EnemyController 사망 처리**에 `FindObjectOfType<EnemyTracker>()?.NotifyEnemyDead();` 한 줄 추가.
