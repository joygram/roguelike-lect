# 00. 프로젝트 준비 — Unity 환경 세팅

이 단계에서는 Unity 프로젝트를 처음 만들고, 팀 전체가 동일한 규칙으로 시작할 수 있는 기반을 잡는다.

환경 세팅을 따로 정리해두는 이유가 있다. 팀 프로젝트에서 가장 흔한 초기 문제는 "나는 됐는데 너는 왜 안 되냐"다. Unity 버전이 다르면 씬이 깨지고, 폴더 구조가 다르면 서로의 작업이 충돌한다. 이 단계에서 맞춰두면 나중에 merge 지옥을 피할 수 있다.

## Unity 버전 선택

Unity는 버전에 따라 API와 기능이 달라진다. 같은 팀 안에서 버전이 다르면 프리팹이나 씬을 열었을 때 경고가 뜨거나 레이아웃이 달라 보인다.

**권장: Unity 6.3 LTS**

LTS(Long Term Support)는 장기 지원 버전이다. 수업이나 팀 프로젝트에서는 LTS를 쓰는 것이 안전하다. 신기능이 없어도 버그 수정이 꾸준히 이루어지고, 온라인 자료가 많다.

Unity Hub에서 설치할 때 반드시 다음 모듈을 함께 선택한다:
- **Windows Build Support** (PC 빌드용)
- **Visual Studio** 또는 **Rider** (코드 편집기)

---

## 프로젝트 생성

1. Unity Hub 실행 → **New Project**
2. 템플릿: **2D (Built-in Render Pipeline)**
   - 3D 템플릿을 고르지 않는다. 2D 게임에서 3D 템플릿을 쓰면 카메라·씬 설정을 처음부터 다 바꿔야 한다.
3. 프로젝트 이름과 저장 위치 설정 후 **Create**

## 폴더 구조 만들기

Unity 프로젝트를 만들면 `Assets/` 폴더 하나만 있다. 폴더 없이 작업하면 파일이 수백 개가 되었을 때 어디에 무엇이 있는지 아무도 모르게 된다.

아래 구조를 `Assets/` 안에 만든다.

```
Assets/
  _Framework/     ← 씬 전환, 게임 상태, 이벤트 등 '게임 규칙' 코드
  _Content/       ← 적, 맵, 아이템 등 '게임 콘텐츠'
    Data/         ← ScriptableObject 에셋 파일
    Prefabs/      ← 프리팹
    Scenes/       ← 게임 씬
  _UI/            ← Canvas, HUD, 패널 등 UI 전용
  Scripts/        ← 분류하기 전 임시 스크립트 (초기에는 여기서 시작해도 됨)
  Sprites/        ← 이미지 파일
    Player/
    Enemy/
    Effects/      ← 타격·스킬 이펙트 스프라이트
    Map/
    UI/
  Audio/          ← 오디오 파일
    BGM/
    SFX/
  Fonts/          ← 폰트 파일 (선택)
```

폴더 이름 앞에 `_`를 붙이면 Project 창에서 알파벳 순으로 정렬될 때 맨 위로 올라온다. `_Framework`, `_Content`, `_UI`가 항상 상단에 보이도록 하기 위해서다.

### 폴더 분리 원칙

`_UI/`는 게임 로직을 **직접** 실행하지 않는다. HP를 깎는 것은 `_Framework/`의 역할이고, `_UI/`는 그 값을 화면에 **표시만** 한다. 처음부터 이 경계를 지키면 나중에 UI를 바꿔도 게임 로직이 망가지지 않는다.

## 씬 만들기

2D 게임에서 씬은 보통 3개로 시작한다.

| 씬 이름 | 역할 |
|---------|------|
| `MainMenu` | 게임 시작 화면. "시작" 버튼을 누르면 Game 씬으로 이동 |
| `Game` | 실제 게임 플레이가 일어나는 씬 |
| `Landing` | 로딩·스플래시 화면 (선택. 처음엔 없어도 됨) |

씬 파일을 만드는 방법:
1. `File > New Scene` → 2D 씬 선택
2. 저장: `Assets/_Content/Scenes/MainMenu.unity`
3. `Game.unity`도 같은 방법으로 생성

씬을 만들었으면 빌드에 등록해야 한다. `File > Build Settings`를 열고 두 씬을 드래그해서 추가한다. `MainMenu`가 위(index 0), `Game`이 아래(index 1)가 되어야 한다.

## 버전 관리 (Git)

팀으로 작업한다면 Git을 사용한다. Unity 프로젝트는 자동 생성 파일이 많아서 `.gitignore` 없이 커밋하면 불필요한 파일이 올라간다.

GitHub에서 Unity용 `.gitignore`를 검색하면 바로 나온다. 프로젝트 루트에 넣고 첫 커밋 전에 반드시 적용한다.

```
# Unity 프로젝트의 주요 무시 항목
Library/
Temp/
Obj/
Build/
Builds/
*.csproj
*.sln
```

`Library/` 폴더는 Unity가 자동으로 재생성하므로 커밋할 필요 없다. 이 폴더만 빠져도 저장소 크기가 크게 줄어든다.

---

## 여기까지 하면 되는 것

- Unity 프로젝트가 열리고 오류 없이 실행된다
- `Assets/` 안에 위 폴더 구조가 있다
- `MainMenu`와 `Game` 씬이 Build Settings에 등록되어 있다
- 팀원이 같은 Unity 버전으로 프로젝트를 열 수 있다
