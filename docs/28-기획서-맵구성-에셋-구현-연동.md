# Part 2. 공통 에셋 제작 (플랫폼·바닥)

## 2.1 아트(에셋) — 기획서 반영

| 오브젝트 | 시각 요소(기획서) | 제작 방법 |
|----------|-------------------|-----------|
| **일반 플랫폼** | 빨간 리본·초록 포장지 느낌 | 타일 또는 스프라이트 32×32~64×64 PNG, 투명 배경. 리본/포장 **패턴 2~3종** 타일셋. |
| **바닥 타일** | 테마별 색조만 다르게 | 1테마: 밝은 회색+포장. 2테마: 어두운 톤. 3테마: 녹·갈색. 4테마: 검정+적색 문양(기획서 4스테이지 참고). |

**Unity 임포트**: Texture Type **Sprite (2D and UI)**, PPU 팀 통일(예: 32). **Sprite Editor**에서 Multiple이면 슬라이스.

## 2.2 구현 — 타일맵 배치

1. **Grid** + **Tilemap** 생성. Sorting Layer `Ground` 권장.
2. **Tile Palette**에 타일 드래그 후 찍기.
3. **Tilemap Collider 2D** + **Composite Collider 2D**(Rigidbody2D Body Type Static)로 합쳐진 충돌.
4. 레이어 **Ground** 할당 → 플레이어 착지 마스크와 일치([27](27-데이터-코드-에셋-병렬개발-및-커밋.md)).

## 2.3 연동

- `LevelSettingsData`에 `stageId`, (선택) `backgroundMusic`, `enemyTheme`(1~4) 정수 저장.
- 씬 로드 후 **스폰 매니저**가 `stageId`로 적 프리팹 변형 선택(기획서 적 단계별 스킨).

## 3.1 기획 요약

회색 축, 검은 벨트, **왼쪽으로 이동**(플레이어를 밀어줌). **움직이는 애니메이션** 필요.

## 3.2 에셋 제작

| 산출물 | 설명 |
|--------|------|
| `spr_conveyor_belt_loop.png` | 벨트 무늬 가로 반복 타일(16×16 또는 32×32). |
| `spr_conveyor_roller.png` | (선택) 축 양끝 장식. |
| 애니 | 벨트 UV 오프셋 스크롤이면 **코드만**으로도 가능(스프라이트 1장). 또는 2~3프레임 스프라이트 시트. |

**프리팹 구조**

```
ConveyorPlatform (Layer: Ground 또는 전용 Conveyor)
├── SpriteRenderer (벨트)
├── BoxCollider2D (발판)
└── ConveyorBelt2D.cs
```

## 3.3 구현 (코드)

**원리**: 플레이어가 벨트 위에 있을 때 **추가 속도**를 X방향으로 더함. 또는 **Rigidbody2D**에 `velocity.x += beltSpeed * Time.deltaTime` 형태.

```csharp
using UnityEngine;

[RequireComponent(typeof(Collider2D))]
public class ConveyorBelt2D : MonoBehaviour
{
    [Tooltip("음수=왼쪽으로 밀기(기획서). 힘 크기는 Inspector에서 조정")]
    [SerializeField] private float beltForce = -40f;
    [SerializeField] private LayerMask playerLayer;

    private void OnCollisionStay2D(Collision2D collision)
    {
        if (((1 << collision.gameObject.layer) & playerLayer) == 0) return;
        var rb = collision.collider.GetComponent<Rigidbody2D>();
        if (rb == null) return;
        rb.AddForce(new Vector2(beltForce, 0f), ForceMode2D.Force);
    }
}
```

**물리**: `ForceMode2D.Force`는 질량을 반영한 가속. 미끄럽게 밀리게 하려면 플레이어 Rigidbody2D **마찰**을 낮추거나 `beltForce`를 키움.

## 3.4 연동

- 벨트 속도를 데이터로 빼려면 `LevelSettingsData` 또는 벨트 프리팹에 `SerializeField`만 두고 스테이지별 프리팹 복제.
- 애니: `Material`에 `_MainTex` 오프셋 스크롤하는 작은 스크립트 `[우리 구현] ConveyorScrollUV.cs`.

| 산출물 | 설명 |

## 4.3 구현 (코드)

```csharp
using UnityEngine;
using UnityEngine.Events;

public class HydraulicPress2D : MonoBehaviour
{
    [SerializeField] private Transform pressHead;
    [SerializeField] private float topY = 2f;
    [SerializeField] private float bottomY = -1f;
    [SerializeField] private float speed = 1f;
    [SerializeField] private int damage = 999; // 즉사면 큰 수
    [SerializeField] private LayerMask hurtLayers;
    [SerializeField] private UnityEvent onCrush; // 이펙트·사운드

    private float _dir = -1f;

    private void Update()
    {
        if (pressHead == null) return;
        Vector3 p = pressHead.localPosition;
        p.y += speed * _dir * Time.deltaTime;
        if (p.y <= bottomY) { p.y = bottomY; _dir = 1f; }
        else if (p.y >= topY) { p.y = topY; _dir = -1f; }
        pressHead.localPosition = p;
    }

    private void OnCollisionEnter2D(Collision2D c)
    {
        if (((1 << c.gameObject.layer) & hurtLayers) == 0) return;
        var hp = c.gameObject.GetComponent<PlayerHealth>();
        if (hp != null) hp.TakeDamage(damage);
        onCrush?.Invoke();
    }
}
```

**연동**: `PlayerHealth`가 프로젝트의 실제 클래스명과 맞추기. 데미지를 "큰 피해"만 주려면 `damage = 30` 등으로 조정.

# Part 5. 녹슨 발판 (3-1 ~ 4-3)

## 5.1 기획 요약

올라가면 **흔들림**, **3초 이상** 머무르면 **무너짐**. **타기 전에 나가면** 무너지지 않음.

## 5.2 에셋 제작

- 일반 플랫폼과 동일 크기 + **녹·갈색 텍스처**.
- 무너짐: 스프라이트 2~3장(균열 → 부서짐) 또는 파티클 `VFX_PlatformBreak`.

## 5.3 구현 (코드)

```csharp
using System.Collections;
using UnityEngine;

public class CrumblingPlatform2D : MonoBehaviour
{
    [SerializeField] private float shakeTimeBeforeBreak = 3f;
    [SerializeField] private float shakeStrength = 0.05f;
    [SerializeField] private LayerMask playerLayer;
    private int _playerCount;
    private float _timer;
    private Vector3 _origin;
    private bool _breaking;

    private void Start() { _origin = transform.position; }

    private void OnCollisionEnter2D(Collision2D c)
    {
        if (((1 << c.gameObject.layer) & playerLayer) != 0) _playerCount++;
    }
    private void OnCollisionExit2D(Collision2D c)
    {
        if (((1 << c.gameObject.layer) & playerLayer) != 0)
        {
            _playerCount--;
            if (_playerCount <= 0) { _timer = 0f; transform.position = _origin; }
        }
    }

    private void Update()
    {
        if (_breaking || _playerCount <= 0) return;
        _timer += Time.deltaTime;
        if (_timer >= shakeTimeBeforeBreak) StartCoroutine(BreakRoutine());
        else if (_timer > 0.1f)
            transform.position = _origin + (Vector3)Random.insideUnitCircle * shakeStrength;
    }

    private IEnumerator BreakRoutine()
    {
        _breaking = true;
        GetComponent<Collider2D>().enabled = false;
        // 이펙트 Instantiate
        yield return new WaitForSeconds(0.5f);
        gameObject.SetActive(false);
    }
}
```

## 5.4 연동

- **리셋**: 씬 재로드 시 발판 다시 활성화. 영구 파괴려면 `DontDestroyOnLoad` 대신 스폰 시스템 사용.
- 난이도: `shakeTimeBeforeBreak`를 `LevelSettingsData`에 넣어 테마별 조정.

# Part 6. 구덩이 (3-1 ~ 4-2)

## 6.1 기획 요약

떨어지면 **체력 풀피면 1 남기고**, **아니면 즉사**. 시각: **정사각 구멍 + 비스듬한 회색 빗금**.

## 6.2 에셋 제작

- `spr_pit_tile.png`: 구멍 테두리 타일(ㄱ자·직선 등) 타일셋.
- 구멍 **내부**는 검정 또는 깊이 느낌 스프라이트.

## 6.3 구현

**방법 A**: 구덩이 영역에 **Trigger** 콜라이더. `OnTriggerEnter2D`에서 플레이어 감지.

```csharp
using UnityEngine;

public class PitHazard2D : MonoBehaviour
{
    [SerializeField] private LayerMask playerLayer;

    private void OnTriggerEnter2D(Collider2D other)
    {
        if (((1 << other.gameObject.layer) & playerLayer) == 0) return;
        var hp = other.GetComponent<PlayerHealth>();
        if (hp == null) return;
        if (hp.CurrentHp >= hp.MaxHp) hp.SetHp(1); // 풀피 → 1
        else hp.TakeDamage(9999);
    }
}
```

`PlayerHealth`에 아래 메서드를 추가하거나, 풀피일 때만 `TakeDamage(maxHp-1)`로 1까지 깎는 식으로 대체.

```csharp
public void SetHp(int v) { _currentHp = Mathf.Clamp(v, 0, maxHp); }
```

## 6.4 연동

- 맵에서 구덩이는 **콜라이더만** 있고 타일은 시각용으로 구멍 모양 배치.
- 4-3 등 구덩이 없는 구간은 프리팹 배치 안 함.

## 7.1 기획 요약

검은 연기. 맞을 때마다 **저주 스택** (UI 상단). **3스택 즉사**. 휴식 스테이지에서 **재화로 치료**.

## 7.2 에셋 제작

- 파이프 스프라이트 + **연기** 파티클(검은 스모크) 또는 스프라이트 애니.
- UI: 저주 스택 아이콘 3칸([25](25-참조게임-UI-프리팹-디자인-레이아웃.md) HUD 확장).

## 7.3 구현

**저주 상태(싱글톤 또는 플레이어 컴포넌트)**

```csharp
using UnityEngine;
using UnityEngine.Events;

public class CurseStack : MonoBehaviour
{
    [SerializeField] private int maxStack = 3;
    public int Current { get; private set; }
    public UnityEvent<int> onStackChanged;

    public void AddStack(int n = 1)
    {
        Current = Mathf.Min(maxStack, Current + n);
        onStackChanged?.Invoke(Current);
        if (Current >= maxStack)
        {
            var ph = GetComponent<PlayerHealth>();
            if (ph != null) ph.TakeDamage(9999);
        }
    }

    public void ClearStacks()
    {
        Current = 0;
        onStackChanged?.Invoke(0);
    }
}
```

**파이프 구역**

```csharp
using UnityEngine;

public class PipeSmokeHazard2D : MonoBehaviour
{
    [SerializeField] private float tickInterval = 1f;
    [SerializeField] private LayerMask playerLayer;
    private float _t;
    private bool _playerInside;

    private void OnTriggerEnter2D(Collider2D o)
    {
        if (((1 << o.gameObject.layer) & playerLayer) != 0) _playerInside = true;
    }
    private void OnTriggerExit2D(Collider2D o)
    {
        if (((1 << o.gameObject.layer) & playerLayer) != 0) _playerInside = false;
    }

    private void Update()
    {
        if (!_playerInside) return;
        _t += Time.deltaTime;
        if (_t >= tickInterval)
        {
            _t = 0f;
            var c = FindFirstObjectByType<CurseStack>();
            if (c != null) c.AddStack(1);
        }
    }
}
```

## 7.4 연동

- **휴식 상점**: 버튼 클릭 시 `CurseStack.ClearStacks()` + 재화 차감([26 Phase 4](26-CSharp-코드복사-부록.md) 상점 흐름).
- UI: `onStackChanged`에 **이미지 3개** 알파/색 갱신 연결.
