## 구분하기: 내가 만드는 것 vs Unity(API)가 주는 것

| 구분 | 이번 단계 예시 | 설명 |
|------|----------------|------|
| **내가 만드는 것** | Tilemap 레이어 구성 결정, 어떤 타일을 어디에 배치할지, Ground 레이어 할당 | 맵 설계·배치 의도는 **우리 책임** |
| **Unity(API)가 주는 것** | `TilemapCollider2D`, `CompositeCollider2D`, `PlatformEffector2D`, `Tilemap`, `Grid` | 충돌 처리·타일 렌더링·단방향 발판은 **엔진**이 처리 |

**왜 엔진 것을 쓰나?**  
타일마다 충돌 경계를 직접 계산하거나 단방향 착지 로직을 직접 짜면 코드가 매우 복잡해진다. Unity의 Tilemap 관련 컴포넌트들은 이 문제를 설정만으로 해결한다. 우리가 결정할 것은 "어떻게 조합할지"다.

---

## 구현 개요

**이번 단계가 끝나면** 씬은 아래와 같고, 플레이어가 바닥에서 막히고 발판을 아래에서 뚫고 올라갈 수 있다.

```
씬 (Game)
├─ Grid
│   ├─ TilemapGround       ← 바닥·벽 (사방이 막힘, Layer: Ground)
│   ├─ TilemapPlatform     ← 단방향 발판 (아래에서 통과, Layer: Ground)
│   └─ TilemapBackground   ← 배경 장식 (충돌 없음, 선택)
├─ Player
└─ SpawnPoint              ← 빈 오브젝트, 플레이어 시작 위치 표시
```

**컴포넌트 역할 요약**

| 컴포넌트 | 부착 위치 | 역할 |
|----------|-----------|------|
| **TilemapCollider2D** | TilemapGround, TilemapPlatform | 타일 위치마다 충돌 영역 자동 생성 |
| **CompositeCollider2D** | TilemapGround | 인접한 타일 콜라이더를 하나로 합쳐 경계 턱 제거 |
| **PlatformEffector2D** | TilemapPlatform | 아래에서 통과·위에서만 착지하는 단방향 발판 처리 |

## TilemapGround — 바닥·벽

### 개요

- **역할**: 플레이어·적이 통과하지 못하는 **완전 차단** 영역. 바닥, 벽, 천장이 여기 해당된다.
- **부착**: Grid 자식 Tilemap 오브젝트.
- **흐름**: TilemapCollider2D가 타일마다 충돌 영역 생성 → CompositeCollider2D가 인접 타일을 하나로 합침 → 플레이어 Rigidbody2D가 이 콜라이더와 충돌해 막힘.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **Grid + Tilemap 구조** | Unity Tilemap은 Grid 아래에 있어야 한다. Grid가 "격자 규칙"을 정의하고, Tilemap이 그 위에 타일을 배치한다. |
| **TilemapGround / TilemapPlatform 분리** | 바닥은 사방이 막히고, 발판은 아래에서 통과된다. 이 두 물리 설정을 같은 Tilemap에서는 동시에 적용할 수 없다. |
| **CompositeCollider2D** | 타일 하나하나에 콜라이더가 각각 있으면 **타일 경계마다 미세한 턱**이 생긴다. 플레이어가 달릴 때 이 턱에 걸려 "뚝뚝" 걸리는 느낌이 난다. CompositeCollider2D는 인접한 콜라이더를 **하나의 매끄러운 콜라이더**로 합친다. |
| **Rigidbody2D (Static)** | CompositeCollider2D를 추가하면 자동으로 붙는다. Body Type을 Static으로 두면 Tilemap이 물리적으로 움직이지 않는다. |

### 사용하는 Unity 컴포넌트 (TilemapGround)

| 컴포넌트 | 역할 |
|----------|------|
| `Tilemap` | 타일 배치·렌더링 |
| `TilemapRenderer` | Tilemap과 세트. 타일을 화면에 그림 |
| `TilemapCollider2D` | 배치된 타일마다 충돌 영역 자동 생성. **Used By Composite** 체크 |
| `CompositeCollider2D` | 인접 타일 콜라이더를 하나로 병합. 경계 턱 제거 |
| `Rigidbody2D (Static)` | CompositeCollider2D가 요구하는 물리 컴포넌트. Body Type = Static |

### 설정 순서 (TilemapGround)

1. Hierarchy `+` → **2D Object → Tilemap → Rectangular**
   - 자동으로 `Grid > Tilemap` 구조 생성
2. Tilemap 이름 → `TilemapGround`
3. **Add Component**:
   - `TilemapCollider2D` → **Used By Composite** 체크
   - `CompositeCollider2D` → Rigidbody2D가 자동 추가됨, Body Type = **Static**
4. Layer → **Ground** (`Project Settings → Tags and Layers`에서 없으면 추가)

## TilemapPlatform — 단방향 발판

### 개요

- **역할**: 아래에서 점프해 올라설 수 있고, 위에서 발을 떼면 떨어지는 **단방향 발판**. 아래에서는 통과된다.
- **부착**: Grid 자식 Tilemap 오브젝트 (TilemapGround와 형제).
- **흐름**: PlatformEffector2D의 `Use One Way`가 충돌 방향을 판단해, 아래에서 올라오는 충돌은 무시하고 위에서 내려오는 충돌만 착지로 처리한다.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **PlatformEffector2D** | "아래에서 통과, 위에서만 착지"를 직접 코드로 구현하려면 매 프레임 충돌 방향을 계산해야 한다. `PlatformEffector2D`의 `Use One Way`를 켜면 이 로직이 자동으로 처리된다. |
| **CompositeCollider2D를 쓰지 않는 이유** | PlatformEffector2D와 CompositeCollider2D는 같이 쓸 수 없다. Effector가 콜라이더를 직접 제어해야 하기 때문이다. 따라서 TilemapCollider2D에 **Used By Effector** 만 체크한다. |
| **같은 Ground 레이어** | PlayerJump의 `OverlapCircle`이 Ground 레이어를 검사한다. TilemapGround와 TilemapPlatform 모두 Ground 레이어여야 발판 위에서도 착지가 인식된다. |

### 사용하는 Unity 컴포넌트 (TilemapPlatform)

| 컴포넌트 | 역할 |
|----------|------|
| `TilemapCollider2D` | 타일마다 충돌 영역 생성. **Used By Effector** 체크 (CompositeCollider2D 아님) |
| `PlatformEffector2D` | **Use One Way** = 아래에서 통과, 위에서만 착지 |

### 설정 순서 (TilemapPlatform)

1. Grid 우클릭 → **2D Object → Tilemap → Rectangular** → 이름 `TilemapPlatform`
2. **Add Component**:
   - `TilemapCollider2D` → **Used By Effector** 체크
   - `PlatformEffector2D` → **Use One Way** 체크
3. Layer → **Ground**

## TilemapBackground — 배경 장식 (선택)

### 개요

- **역할**: 충돌 없이 **배경만 그리는** Tilemap. 장식용 타일(풀, 벽면 무늬 등)을 여기에 배치한다.
- 필수가 아니다. 1차 프로토타입에서는 생략해도 된다.
- 콜라이더 컴포넌트를 추가하지 않는다 — 추가하면 플레이어가 배경에 막힌다.

## 단계별 실습 순서

### 0. 씬 준비

**작업 씬** 탭이 팀에서 정한 씬(예: `Game`)인지 확인한다.

### 1. Grid와 Tilemap 만들기

위 TilemapGround, TilemapPlatform 설정 순서를 순서대로 진행한다.

### 2. Tile Palette로 맵 그리기

1. `Window → 2D → Tile Palette` 열기
2. Tile Palette 상단 드롭다운에서 그릴 대상 Tilemap 선택
   - 바닥·벽 → `TilemapGround`
   - 발판 → `TilemapPlatform`
3. 타일 클릭 → 씬 뷰에서 클릭·드래그로 배치

**도구 단축키**

| 키 | 기능 |
|----|------|
| `B` | 브러시 (타일 배치) |
| `D` | 지우개 |
| `G` | 채우기 |
| `Shift+드래그` | 사각형 범위 선택 후 채우기 |

### 3. SpawnPoint 설정

1. Hierarchy → Create Empty → 이름 `SpawnPoint`
2. 원하는 스폰 위치(예: 바닥 왼쪽)로 이동
3. Player 오브젝트를 같은 위치에 배치

### 4. 플레이 테스트

1. Play
2. A/D 이동 — 바닥·벽에 막히는지 확인
3. Space 점프 — 발판 위로 올라서는지 확인
4. 발판 위에서 발을 떼면 떨어지는지 확인
5. 구멍에서 아래로 떨어지는지 확인

## 동작이 안 될 때 (순서대로 확인)

| # | 증상 / 확인 |
|---|-------------|
| 1 | **플레이어가 바닥을 통과한다** → TilemapGround에 TilemapCollider2D 없음, 또는 Layer가 Ground가 아님 |
| 2 | **달릴 때 걸리는 느낌** → TilemapCollider2D의 Used By Composite 미체크, 또는 CompositeCollider2D 없음 |
| 3 | **발판을 위에서도 통과한다** → PlatformEffector2D의 Use One Way 미체크 |
| 4 | **발판 위에서 착지 인식 안 됨** → TilemapPlatform Layer가 Ground가 아님 (PlayerJump의 OverlapCircle이 Ground 레이어만 검사) |
| 5 | **Tilemap 타일이 화면에 안 보인다** → TilemapRenderer가 없거나, Tile Palette에서 잘못된 Tilemap을 선택하고 그린 것 |

## 이번 단계에서 익히는 개념

- **Grid + Tilemap 계층** — Unity 2D 맵의 기본 구조
