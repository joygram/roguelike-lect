# 필요 에셋 목록

**README 전체 순서**: **8단계** (04 다음).

아트·기획·프로그래밍 담당자나 개인이 **직접** 보고, 우리 게임에 쓸 에셋을 요약표와 그래픽/사운드/데이터 표에 채우고, 02에서 만든 적·아이템 초안과 연결하는 실무용 문서입니다. 팀이면 아트가 그래픽 목록, 기획이 데이터 목록을 채운 뒤 06 역할 분담과 연동하세요.

Unity 2D 도트 액션 플랫포머 프로젝트에서 필요한 **에셋·데이터**를 기능별·역할별로 정리한 문서입니다.  
팀원별 담당을 정할 때 `06-팀원-역할-분담.md`와 함께 사용하세요.

**사용 지식·영역** ([01 §1.1](01-요구분석.md#11-이-프로젝트에서-사용하는-개발-영역지식-공통-기준)): **데이터 설계**(에셋 종류·크기·수량 정의, 데이터 에셋 목록, 기획 수치→에셋 매핑).

**넣을 위치(02·06·31과 동일)**: 이 목록의 **데이터 에셋·맵·적·장애물 프리팹**은 **`_Content/`**, **HUD·버튼용 그래픽**은 **`_UI/`** 에 붙일 프리팹과 연결, **코어에 쓸 스크립트**는 **`_Framework/`** — 역할 분담(06)과 폴더를 같이 맞춥니다.

## 2. 그래픽 에셋

에셋은 **2D 스프라이트 제작·사용**을 기준으로 합니다. **그림을 직접 그리고 Unity에 넣어 보는 과정**을 체험할 수 있고, 픽셀 단위로 캐릭터·맵을 구분하기 쉬우며, 2D 플랫포머에 맞는 작업량으로 맞추기 위해 **스프라이트**를 기준으로 했습니다. (자세한 제작·임포트 절차는 **02 문서 3.1절** 참고.)  
리소스가 없을 때는 **플레이스홀더로 작업 후 교체**하는 방식으로 진행합니다. → [06 문서 §1.6](06-팀원-역할-분담.md#16-리소스-없을-때-플레이스홀더--교체)

---

### 2.1 스프라이트 제작·사용 방식 (요약)

| 단계 | 할 일 |
|:----:|--------|
| 1 | 그림 프로그램에서 **투명 배경 PNG**로 그리기 → `Assets/Sprites/`에 넣기 |
| 2 | Unity에서 해당 PNG 선택 → Inspector **Texture Type**: **Sprite (2D and UI)** |
| 3 | **Pixels Per Unit(PPU)** 팀에서 정한 값으로 통일 (예: 32 또는 64) |
| 4 | 씬에서 Sprite Renderer로 사용 또는 Tilemap 타일로 사용 |

→ 상세 절차(그리기 도구, Sprite Mode Single/Multiple, 씬 배치)는 **02-준비사항-체크리스트.md** 3.1절 참고.

---

### 2.2 필요한 스프라이트 종류·크기 (우리 게임용)

| 용도 | 에셋 종류 | 권장 크기 | 수량 | 비고 |
|------|-----------|-----------|------|------|
| 플레이어 | PNG 스프라이트 | 32×32 ~ 64×64 | 1~4 (방향별) | Sprite Mode: Single 또는 Multiple(시트) |
| 적 | PNG | 32×32 | 종류별 1 이상 | 02 직접 채우기 표의 적 수만큼 |
| 플랫폼·바닥·벽 타일 | PNG | 16×16 또는 32×32 | 1~4 | 타일맵용, PPU와 맞추기 |
| 아이템 아이콘 | PNG | 16×16 ~ 32×32 | 종류별 1 | 포션, 코인 등 |
| UI (HP 바 등) | PNG 또는 UI Sprite | 자유 | 2~4 | HP, 버튼 |

**저작권**: 교육·팀 프로젝트용은 **직접 제작** 권장. 외부 사용 시 Unity Asset Store 무료, itch.io 교육용 명시 자료만 사용.

---

**스프라이트를 기준으로 한 이유**: **직접 그려 보고 임포트해 보는 과정**을 통해 제작 흐름을 익히고, 2D 로그라이크에 맞는 비주얼을 단계적으로 넣기 위함입니다. 리소스 없을 때는 [06 문서 §1.6](06-팀원-역할-분담.md#16-리소스-없을-때-플레이스홀더--교체) 플레이스홀더→교체 방식 참고.

## 3. 사운드 에셋 (**필수** + 확장 선택)

| 용도 | 포맷 | 수량 | 길이/비고 |
|------|------|------|-----------|
| BGM | MP3/OGG | **1 (필수)** | 30초~1분 루프 |
| 공격 효과음 | WAV/MP3 | **1 (필수)** | 짧은 클립 |
| 피격 효과음 | WAV/MP3 | **1 (필수)** | 짧은 클립 |
| 게임 오버·버튼·사망 등 | WAV/MP3 | **1 (필수)** | 공격·피격과 **겹치지 않는** 세 번째 효과음 |
| 아이템 획득 | WAV/MP3 | 선택 | 아이템 기능 넣을 때 |

Unity에서 사용 시 **Import Settings**에서 Loop, Compression 설정 확인.

## 4. 데이터 에셋 (게임 규칙)

기획·밸런스용 수치를 에셋으로 두면 프로그래머 없이도 수정 가능합니다.  
**현업에서 쓰는 데이터 처리 방식**과 **심화학습(개념·연습)** 은 **08 문서 1.5·1.6절**을 참고하세요.

### 4.1 ScriptableObject 예시 (권장)

| 에셋 타입 | 내용 예시 |
|-----------|-----------|
| PlayerStats | maxHp, moveSpeed, attackPower, attackCooldown |
| EnemyStats (종류별) | maxHp, moveSpeed, damage, scoreValue |
| DungeonSettings | roomCountMin/Max, roomSizeMin/Max, corridorWidth |
| ItemData (종류별) | itemType, healAmount, icon (Sprite 참조) |

### 4.2 테이블 형태 (참고용)

| 데이터명 | 예시 값 | 설명 |
|----------|---------|------|
| 플레이어 최대 HP | 100 | |
| 플레이어 이동 속도 | 5 | |
| 플레이어 공격력 | 10 | |
| 플레이어 공격 쿨다운 | 0.5초 | |
| 슬라임 HP | 20 | |
| 슬라임 공격력 | 5 | |
| 슬라임 이동 속도 | 2 | |
| 박쥐 HP | 10 | |
| 박쥐 공격력 | 3 | |
| 박쥐 이동 속도 | 4 | |
| 방 개수 (min~max) | 5~9 | |
| 방 크기 (가로·세로) | 4~8 | |

## 5. 씬·프리팹 (Unity 내 제작)

| 종류 | 필수 여부 | 설명 |
|------|-----------|------|
| Game 씬 | 필수 | 메인 플레이 씬 (카메라, 라이트, 빈 맵 또는 테스트 맵) |
| GameOver 씬 | 선택 | 게임 오버 전용. 없으면 Game 씬 내 UI로 처리 |
| MainMenu 씬 | 선택 | 타이틀 화면 |
| Player 프리팹 | 필수 | 플레이어 오브젝트 + 스크립트 |
| Enemy 프리팹 | 필수 | 몬스터 오브젝트 + 스크립트 (종류별로 1개 이상) |
| Wall / Floor 프리팹 | 필수 | 맵 생성 시 인스턴스로 배치 |
| (선택) Projectile 프리팹 | 선택 | 원거리 공격 시 |

### 5.1 씬·에셋 정확한 이름 제안

팀에서 따로 정하지 않았다면 아래 이름을 **그대로** 쓰면 됩니다. 02 문서 4.1절과 동일합니다.

| 구분 | 정확한 이름 (그대로 사용 가능) |
|------|-------------------------------|
| **씬** | `Game.unity`, `GameOver.unity`, `MainMenu.unity` |
| **프리팹** | `Player.prefab`, `Enemy_Slime.prefab`, `Enemy_Bat.prefab`, `Wall.prefab`, `Floor.prefab`, `Projectile.prefab` |
| **데이터 에셋** | `PlayerStats.asset`, `SlimeStats.asset`, `BatStats.asset`, `DungeonSettings.asset`, `RedPotion.asset` |
| **스프라이트 파일** | `player_idle.png`, `slime.png`, `bat.png`, `tile_wall.png`, `tile_floor.png`, `ui_hp_bar.png` |

이름 규칙: 영문, 공백 대신 밑줄(`_`), 소문자 권장. 팀에서 정한 이름은 한 곳에 적어 두고, 코드에서 `LoadScene("Game")`, `Instantiate(floorPrefab)` 등과 맞추면 됩니다.

## 6. 폴더 구조 (에셋 배치)

```
Assets/
├── Scenes/
├── Scripts/
├── Prefabs/
│   ├── Player.prefab
│   ├── Enemies/
│   ├── Map/
│   └── UI/
├── Sprites/          (2D 사용 시)
│   ├── Characters/
│   ├── Tiles/
│   └── UI/
├── Materials/
├── Data/             (ScriptableObject 등)
│   ├── PlayerStats.asset
│   ├── EnemyStats/
│   └── DungeonSettings.asset
└── Audio/
    ├── BGM/
    └── SFX/
```
