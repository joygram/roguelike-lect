## 구분하기: 내가 만드는 것 vs Unity(API)가 주는 것

| 구분 | 이번 단계 예시 | 설명 |
|------|----------------|------|
| **내가 만드는 것** | 적 수치 정의(ScriptableObject), "언제 데미지 판정을 내는지", 무적 시간 규칙, 히트박스 활성·비활성 타이밍 | 전투 규칙 설계는 **우리 책임** |
| **Unity(API)가 주는 것** | `OnCollisionEnter2D`, `OnTriggerEnter2D`, `Instantiate`, `Destroy`, `WaitForSeconds`, `Time.deltaTime` | 충돌 감지·오브젝트 생성/제거·시간 처리는 **엔진**이 제공 |

**왜 엔진 것을 쓰나?**  
충돌을 직접 계산(두 물체 위치 비교)하면 프레임 속도에 따라 판정이 달라질 수 있다. Unity의 물리 이벤트(`OnCollision`, `OnTrigger`)는 엔진이 매 프레임 계산해서 정확한 타이밍에 호출해 준다.

## EnemyStatsData — 적 수치 데이터

### 개요

- **역할**: 적 하나의 수치(HP·속도·데미지 등)를 **에셋 파일**로 분리한다. 슬라임·박쥐·고블린 등 종류별로 에셋을 하나씩 만들어 같은 `EnemyController` 스크립트에 연결한다.
- **부착**: 코드 파일이 아닌 **ScriptableObject 에셋**. Project 창에서 만들고 Inspector에서 수치를 입력한다.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **ScriptableObject** | 수치를 코드에 직접 쓰면(하드코딩) 적 종류가 늘 때마다 스크립트를 새로 만들어야 한다. ScriptableObject는 같은 `EnemyController` 스크립트로 **수치만 다른 여러 종류의 적**을 만들 수 있다. |
| **[CreateAssetMenu] 속성** | 이 속성 없이는 Project 창에서 에셋을 만들 수 없다. 속성을 붙이면 우클릭 메뉴에서 에셋 생성이 가능해진다. |

### SerializeField를 쓴 이유

EnemyStatsData의 모든 필드는 `public`이다. `SerializeField`와 마찬가지로 Inspector에서 편집 가능하다. ScriptableObject는 코드를 직접 참조하는 곳이 없고, **에셋 파일 자체가 데이터**이므로 public 필드가 자연스럽다.

### 코드 (EnemyStatsData.cs)

**파일**: `Assets/Scripts/EnemyStatsData.cs`

```csharp
using UnityEngine;

[CreateAssetMenu(menuName = "Game/EnemyStats")]   // [Unity API] 우클릭 메뉴에 등록
public class EnemyStatsData : ScriptableObject
{
    public string enemyName = "Slime";
    public int maxHp = 30;
    public float moveSpeed = 2f;
    public int contactDamage = 10;
    public int expValue = 5;
    public Sprite sprite;
}
```

**에셋 만들기**: `Assets/_Content/Data/` 우클릭 → **Create → Game → EnemyStats** → 이름 `EnemyStats_Slime` → Inspector에서 수치 입력.

## EnemyController — 적 이동·피격·사망

### 개요

- **역할**: 씬에 배치된 적 하나의 **이동(플레이어 추적)**, **피격 처리**, **사망(Destroy)**을 담당한다. 수치는 `EnemyStatsData`에서 읽는다.
- **부착**: Enemy 프리팹.
- **흐름**: `Start`에서 플레이어 Transform 탐색 → `Update`에서 매 프레임 플레이어 방향으로 이동 → `OnCollisionEnter2D`에서 플레이어 접촉 시 데미지 전달 → `TakeDamage`에서 HP 0이면 Destroy.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **Vector3.MoveTowards** | 목적지 방향으로 일정 속도로 이동. 속도를 `moveSpeed * Time.deltaTime`으로 계산해 프레임 속도와 무관하게 "초당 N 유닛"이 된다. |
| **MoveTowards의 한계** | Transform을 직접 바꾸므로 물리 엔진을 거치지 않아 얇은 벽을 빠르게 통과할 수 있다. 속도가 낮고 맵이 넓으면 체감하기 어렵다. 적이 벽을 뚫으면 `Rigidbody2D.velocity`로 바꾸는 것이 해결책이다. |
| **FindGameObjectWithTag("Player")** | 플레이어 Transform을 Inspector에서 연결하면 프리팹에 씬 오브젝트를 드래그할 수 없다. `Start`에서 Tag로 찾으면 프리팹이 어느 씬에 있어도 자동으로 연결된다. |
| **OnCollisionEnter2D** | 접촉 순간 한 번만 데미지를 주고 싶다. `OnCollisionStay2D`는 접촉 중 매 프레임 호출되어 데미지가 너무 자주 들어간다. |
| **RequireComponent** | 이 스크립트는 SpriteRenderer를 사용한다. 없으면 오류가 나므로 Unity가 미리 경고하도록 선언한다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `stats` | **씬에서 연결**. 슬라임 적에는 SlimeStats, 박쥐 적에는 BatStats를 각각 드래그로 연결한다. Inspector에서 교체만으로 다른 종류의 적이 된다. |

`_hp`, `_player`, `_sr` 는 코드에서만 쓰는 내부 상태. Inspector에서 노출할 필요 없다.

### 사용하는 Unity API (EnemyController)

| API | 역할 |
|-----|------|
| `Vector3.MoveTowards(현재, 목표, 최대거리)` | 목표 방향으로 최대거리만큼 이동. 초당 속도 = `speed * Time.deltaTime` |
| `Time.deltaTime` | 프레임 간 경과 시간. 이동 속도를 프레임 속도 무관하게 만듦 |
| `GameObject.FindGameObjectWithTag` | Tag로 씬에서 오브젝트 탐색 |
| `OnCollisionEnter2D` | 충돌 시작 순간 한 번 호출 |
| `CompareTag("Player")` | Tag 비교. 문자열 직접 비교보다 효율적 |
| `?.` (null 조건 연산자) | 컴포넌트가 없을 때 오류 없이 건너뜀 |
| `Destroy(gameObject)` | 씬에서 이 오브젝트 제거 |

### 코드 (EnemyController.cs)

**파일**: `Assets/Scripts/EnemyController.cs`. Enemy 프리팹에 부착. `stats` 슬롯에 EnemyStatsData 에셋 드래그.

```csharp
using UnityEngine;

[RequireComponent(typeof(SpriteRenderer))]   // [Unity API] SpriteRenderer 없으면 경고
public class EnemyController : MonoBehaviour
{
    public EnemyStatsData stats;   // [우리 멤버] Inspector에서 종류별 에셋 연결

    int _hp;
    Transform _player;
    SpriteRenderer _sr;

    void Start()   // [Unity가 호출]
    {
        _hp = stats.maxHp;
        _sr = GetComponent<SpriteRenderer>();   // [Unity API]
        if (stats.sprite != null) _sr.sprite = stats.sprite;

        var playerObj = GameObject.FindGameObjectWithTag("Player");   // [Unity API]
        if (playerObj != null) _player = playerObj.transform;
    }

    void Update()   // [Unity가 호출] 매 프레임 추적
    {
        if (_player == null) return;
        transform.position = Vector3.MoveTowards(
            transform.position, _player.position,
            stats.moveSpeed * Time.deltaTime   // [Unity API] 프레임 무관 속도
        );
    }

    public void TakeDamage(int amount)   // [우리 메소드] 외부(히트박스)에서 호출
    {
        _hp -= amount;
        if (_hp <= 0) Destroy(gameObject);   // [Unity API]
    }

    void OnCollisionEnter2D(Collision2D other)   // [Unity가 호출] 충돌 시작 순간
    {
        if (other.gameObject.CompareTag("Player"))   // [Unity API]
            other.gameObject.GetComponent<PlayerHealth>()?.TakeDamage(stats.contactDamage);
    }
}
```

**Enemy 프리팹 만들기**: Create Empty → 이름 `Enemy_Slime` → `SpriteRenderer`, `Rigidbody2D`(Gravity Scale 3, Freeze Z), `CapsuleCollider2D`, `EnemyController` 추가 → Tag `Enemy`, Layer `Enemy` → `Assets/_Content/Prefabs/`로 저장.

## PlayerHealth — 플레이어 HP·피격·무적

### 개요

- **역할**: 플레이어의 **현재 HP 관리**, **피격 시 HP 감소 + 무적 시간**, **HP 0 시 게임 오버 이벤트 발행**.
- **부착**: Player 오브젝트.
- **흐름**: `Start`에서 HP 초기화 + UI에 알림 → `TakeDamage` 호출 시 무적 중이면 무시, 아니면 HP 감소 + 이벤트 발행 + 무적 타이머 설정 → `Update`에서 타이머 감소.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **무적 타이머 (float)** | `bool isInvincible`만 두면 "언제 무적이 풀리는지" 별도로 처리해야 한다. `float _invincibleTimer`에 지속 시간을 넣고 매 프레임 `Time.deltaTime`으로 감소시키면 한 변수로 해결된다. |
| **CurrentHp 프로퍼티 (읽기 전용)** | 외부 코드가 `_hp`를 직접 수정하면 무적 타이머·이벤트 발행 로직이 우회된다. 프로퍼티로 읽기만 허용해 HP 변경은 반드시 `TakeDamage`를 통하게 한다. |
| **GameEvents로 이벤트 발행** | PlayerHealth가 UI를 직접 참조하면 UI 코드가 바뀔 때마다 PlayerHealth도 수정해야 한다. 이벤트로 "HP가 바뀌었다"는 신호만 보내면 UI는 이 이벤트를 구독해 알아서 갱신한다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `maxHp` | **기획/밸런스 수치**. 코드 수정 없이 Inspector에서 최대 HP를 조절할 수 있다. |

`_hp`, `_invincibleTimer`, `InvincibleDuration`은 내부 상태·고정 상수. Inspector 노출 불필요.

### 사용하는 Unity API (PlayerHealth)

| API | 역할 |
|-----|------|
| `Time.deltaTime` | 매 프레임 무적 타이머 감소에 사용. 프레임 속도 무관하게 실제 시간 경과 |
| `Mathf.Max(0, _hp - amount)` | HP가 0 아래로 내려가지 않도록 보정 |

### 코드 (PlayerHealth.cs)

**파일**: `Assets/Scripts/PlayerHealth.cs`. Player에 부착.

```csharp
using UnityEngine;

public class PlayerHealth : MonoBehaviour
{
    [SerializeField] int maxHp = 100;   // [우리 멤버]

    int _hp;
    float _invincibleTimer;
    const float InvincibleDuration = 0.5f;

    public int CurrentHp => _hp;   // [우리 멤버] 읽기 전용. 외부에서 직접 수정 차단
    public int MaxHp     => maxHp;

    void Start()   // [Unity가 호출]
    {
        _hp = maxHp;
        GameEvents.RaisePlayerHpChanged(_hp, maxHp);   // UI 초기화용 알림
    }

    void Update()   // [Unity가 호출]
    {
        if (_invincibleTimer > 0)
            _invincibleTimer -= Time.deltaTime;   // [Unity API] 실제 시간 감소
    }

    public void TakeDamage(int amount)   // [우리 메소드]
    {
        if (_invincibleTimer > 0) return;   // 무적 시간 중 피격 무시

        _hp = Mathf.Max(0, _hp - amount);   // [Unity API] 0 아래로 안 내려감
        _invincibleTimer = InvincibleDuration;

        GameEvents.RaisePlayerHpChanged(_hp, maxHp);   // HP 변화 → UI에 알림

        if (_hp <= 0) GameEvents.RaiseGameOver();   // 게임 오버 이벤트
    }
}
```

## PlayerAttackHitbox — 히트박스 충돌 판정

### 개요

- **역할**: 히트박스 콜라이더가 Enemy Tag 오브젝트와 겹칠 때 `EnemyController.TakeDamage`를 호출한다. 이펙트 프리팹도 여기서 생성한다.
- **부착**: Player 자식 `Hitbox` 오브젝트. `CapsuleCollider2D`(isTrigger = true)와 같이 붙인다.
- **흐름**: `OnTriggerEnter2D` → Enemy Tag 확인 → 데미지 전달 → 이펙트 Instantiate.

### 히트박스를 별도 자식 오브젝트로 만드는 이유

플레이어 몸체 콜라이더를 공격 판정으로 쓰면, 이동 중 적과 닿을 때마다 공격이 된다. 원하는 것은 "공격 버튼을 눌렀을 때만, 짧은 시간 동안만" 판정이 있는 것이다. 별도 자식 오브젝트(Hitbox)를 만들어 평상시 비활성화하고, 공격 시에만 잠깐 활성화한다.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **isTrigger = true** | 충돌 시 물리적으로 밀리지 않고 "겹침만 감지"한다. 공격 판정은 플레이어를 튕기거나 막으면 안 된다. |
| **OnTriggerEnter2D** | 겹침 시작 순간 한 번만 호출. `OnTriggerStay2D`는 겹쳐 있는 동안 매 프레임 호출되어 데미지가 너무 자주 들어간다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `damage` | **기획/밸런스 수치**. PlayerCombat에서 공격 시 이 값을 덮어쓰기도 한다. Inspector에서 기본값 확인 가능. |
| `hitFxPrefab` | **씬/에셋 연결**. 타격 이펙트 프리팹을 Inspector에서 드래그로 연결한다. |

### 코드 (PlayerAttackHitbox.cs)

**파일**: `Assets/Scripts/PlayerAttackHitbox.cs`. Player/HitboxPivot/Hitbox에 부착.

```csharp
using UnityEngine;

public class PlayerAttackHitbox : MonoBehaviour
{
    [SerializeField] public int damage = 10;
    [SerializeField] GameObject hitFxPrefab;

    void OnTriggerEnter2D(Collider2D other)   // [Unity가 호출] 겹침 시작 순간
    {
        if (!other.CompareTag("Enemy")) return;   // [Unity API]

        other.GetComponent<EnemyController>()?.TakeDamage(damage);   // [Unity API + 우리 메소드]

        if (hitFxPrefab != null)
            Instantiate(hitFxPrefab, other.transform.position, Quaternion.identity);   // [Unity API]
    }
}
```

**AutoDestroy.cs** (이펙트 프리팹용):

```csharp
using UnityEngine;

public class AutoDestroy : MonoBehaviour
{
    [SerializeField] float lifetime = 0.3f;
    void Start() { Destroy(gameObject, lifetime); }   // [Unity API] 지정 시간 후 제거
}
```

**이펙트 프리팹**: Create Empty → 이름 `HitFX` → SpriteRenderer + AutoDestroy 추가 → Prefabs에 저장 → Hitbox의 PlayerAttackHitbox `hitFxPrefab`에 드래그.

## PlayerCombat — 공격 입력·히트박스 타이밍

### 개요

- **역할**: 공격 버튼 입력 감지, 쿨다운 관리, **히트박스를 짧은 시간 동안 활성화**하는 코루틴 실행.
- **부착**: Player 오브젝트.
- **흐름**: `Update`에서 Fire1 입력 + 쿨다운 확인 → `StartCoroutine(ActivateHitbox)` → 히트박스 켜기 → `WaitForSeconds` → 히트박스 끄기.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **코루틴 (IEnumerator)** | "히트박스 켜기 → X초 기다리기 → 끄기" 순서를 `Update`에서 타이머 변수로 관리하면 변수가 많아진다. 코루틴은 시간 순서가 있는 로직을 한 함수 안에서 직관적으로 표현할 수 있다. |
| **Time.time >= _nextAttackTime** | 쿨다운을 `bool isCooling`으로 두면 코루틴으로 초기화해야 한다. `Time.time`(게임 시작 후 경과 초)과 비교하면 변수 하나로 해결된다. |
| **hitbox.enabled** | 오브젝트 자체를 SetActive하지 않고 **콜라이더만 켜고 끈다**. SetActive는 Start 재호출 등 부수 효과가 있다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `hitbox` | **씬에서 연결**. Hitbox 자식 오브젝트의 Collider2D를 Inspector에서 드래그 연결 |
| `hitboxLogic` | **씬에서 연결**. PlayerAttackHitbox 컴포넌트 참조. 공격마다 `damage` 값을 주입한다. |
| `attackDamage` | **기획/밸런스 수치** |
| `attackCooldown` | **기획/밸런스 수치** |
| `hitboxActiveDuration` | **기획 수치**. 히트박스가 켜져 있는 시간 |

### 사용하는 Unity API (PlayerCombat)

| API | 역할 |
|-----|------|
| `Input.GetButtonDown("Fire1")` | 마우스 왼쪽 클릭 또는 Ctrl. 누른 프레임에만 true |
| `Time.time` | 게임 시작 후 경과 시간(초). 쿨다운 비교에 사용 |
| `StartCoroutine` | 코루틴 시작 |
| `WaitForSeconds(초)` | 코루틴에서 지정 시간 동안 대기 |
| `Collider2D.enabled` | 콜라이더만 켜고 끔. SetActive와 달리 Start 재호출 없음 |

### 코드 (PlayerCombat.cs)

**파일**: `Assets/Scripts/PlayerCombat.cs`. Player에 부착. Inspector에서 hitbox·hitboxLogic 연결.

```csharp
using UnityEngine;
using System.Collections;

public class PlayerCombat : MonoBehaviour
{
    [SerializeField] Collider2D hitbox;
    [SerializeField] PlayerAttackHitbox hitboxLogic;
    [SerializeField] int attackDamage = 10;
    [SerializeField] float attackCooldown = 0.5f;
    [SerializeField] float hitboxActiveDuration = 0.15f;

    float _nextAttackTime;

    void Awake()   // [Unity가 호출]
    {
        hitbox.enabled = false;   // 시작 시 히트박스 비활성
    }

    void Update()   // [Unity가 호출]
    {
        if (Input.GetButtonDown("Fire1") && Time.time >= _nextAttackTime)   // [Unity API]
        {
            _nextAttackTime = Time.time + attackCooldown;
            hitboxLogic.damage = attackDamage;
            StartCoroutine(ActivateHitbox());   // [Unity API]
        }
    }

    IEnumerator ActivateHitbox()   // [우리 메소드] 코루틴
    {
        hitbox.enabled = true;
        yield return new WaitForSeconds(hitboxActiveDuration);   // [Unity API] X초 대기
        hitbox.enabled = false;
    }
}
```

## EnemySpawner — 적 스폰

### 개요

- **역할**: 씬 시작 시 지정된 스폰 포인트마다 Enemy 프리팹을 Instantiate한다.
- **부착**: 씬 빈 오브젝트.

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `enemyPrefab` | **씬에서 연결**. 어떤 적 프리팹을 스폰할지 Inspector에서 지정 |
| `stats` | **씬에서 연결**. 스폰되는 적에 어떤 수치를 줄지 |
| `spawnPoints` | **씬에서 연결**. 스폰 위치가 될 빈 오브젝트 배열 |

### 코드 (EnemySpawner.cs)

```csharp
using UnityEngine;

public class EnemySpawner : MonoBehaviour
{
    [SerializeField] GameObject enemyPrefab;
    [SerializeField] EnemyStatsData stats;
    [SerializeField] Transform[] spawnPoints;

    void Start()   // [Unity가 호출]
    {
        foreach (var point in spawnPoints)
        {
            var go = Instantiate(enemyPrefab, point.position, Quaternion.identity);   // [Unity API]
            go.GetComponent<EnemyController>().stats = stats;
        }
    }
}
```

씬에 빈 오브젝트 `EnemySpawner` 추가 → `EnemySpawner.cs` 부착 → Inspector에서 프리팹·stats·스폰포인트 연결.

## 단계별 실습 순서

1. **EnemyStatsData.cs** 작성 → `EnemyStats_Slime` 에셋 생성 (Project에서 수치 입력)
2. **EnemyController.cs** 작성 → Enemy_Slime 프리팹 만들고 stats 연결
3. **PlayerHealth.cs** 작성 → Player에 부착 (GameEvents 없으면 임시 Debug.Log 대체)
4. **HitboxPivot → Hitbox** 자식 구조 만들기 → PlayerAttackHitbox 부착
5. **AutoDestroy.cs** + HitFX 프리팹 만들기
6. **PlayerCombat.cs** 작성 → hitbox·hitboxLogic 연결
7. **EnemySpawner** 오브젝트 배치 → 스폰포인트 설정 후 Play 테스트

---

## 동작이 안 될 때 (순서대로 확인)

| # | 증상 / 확인 |
|---|-------------|
| 1 | **적이 스폰 안 됨** → EnemySpawner에 prefab·stats·spawnPoints 미연결 |
| 2 | **적이 플레이어를 못 찾음** → Player Tag가 `Player`로 설정 안 됨 |
| 3 | **공격해도 적 HP 안 깎임** → Hitbox Layer 충돌 매트릭스 미설정, 또는 Enemy Tag 미설정 |
| 4 | **히트박스가 항상 켜져 있음** → Awake에서 `hitbox.enabled = false` 누락 |
| 5 | **적에 닿아도 플레이어 HP 안 깎임** → Player Tag 미설정, 또는 PlayerHealth 컴포넌트 없음 |
| 6 | **GameEvents 오류** → GameEvents.cs가 아직 없다면 `Debug.Log`로 임시 대체 |

## 이번 단계에서 익히는 개념

- **ScriptableObject** — 수치를 코드에서 분리해 에셋으로 관리
- **OnTrigger vs OnCollision** — isTrigger 여부에 따른 이벤트 차이
- **코루틴** — 시간 순서가 있는 로직을 Update 없이 표현
- **이벤트 패턴** — HP가 바뀔 때 UI에 직접 접근하지 않고 신호만 보내기
