- **SerializeField**를 쓰는 이유 — 어떤 필드에 쓰고 어떤 필드에 안 쓰는가?

## 구분하기: 내가 만드는 것 vs Unity(API)가 주는 것

| 구분 | 이번 단계 예시 | 설명 |
|------|----------------|------|
| **내가 만드는 것** | 이동 속도·점프 힘 수치, **언제** 좌우/점프할지 판단, 땅에 닿았는지 **해석** | “몇으로 움직일지”“땅이면 점프 허용”은 **게임 규칙** = 우리 책임 |
| **Unity(API)가 주는 것** | `Rigidbody2D`, `Input`, `Physics2D.OverlapCircle`, `Transform`, `LateUpdate`, `Vector3.SmoothDamp` | 물리 적용·키 입력·원 안 충돌 검사·위치 보간은 **엔진**이 제공 |

**왜 엔진 것을 쓰나?**  
`Update()`에 우리 코드를 넣으면 **매 프레임** 입력을 읽을 수 있다. `Rigidbody2D`에 속도를 주면 엔진이 **충돌·중력**을 함께 처리해 준다. `Transform.position`만 매 프레임 바꾸면 벽에 박혀도 뚫리거나 점프가 어색해지기 쉽다.

## 구현 개요

**이번 단계가 끝나면** 씬은 대략 아래와 같고, Play하면 **A/D 이동·Space 점프·카메라 추적**이 된다.

```
씬 (예: Game 또는 TestMove)
├─ Main Camera          ← CameraFollow (Script), Target = Player
├─ Player               ← Rigidbody2D, CapsuleCollider2D, PlayerMove, PlayerJump
│   └─ GroundCheck      ← 빈 오브젝트, 발 아래 (0, -0.55, 0)
└─ TestFloor (또는 타일맵 바닥) ← BoxCollider2D, Layer = Ground
```

| 오브젝트 | 역할 |
|----------|------|
| **Player** | 물리 이동·점프의 주체 |
| **GroundCheck** | 발 밑 한 점 — 여기에 원을 그려 Ground와 겹치면 “착지” |
| **Main Camera** | 플레이어 위치를 부드럽게 따라감 |

**스크립트 역할 요약**

| 스크립트 | 부착 위치 | 역할 |
|----------|-----------|------|
| **PlayerMove** | Player | 좌우 입력을 받아 가로 속도만 갱신 |
| **PlayerJump** | Player | 땅 여부 검사 후 Space 입력 시 위로 속도 부여 |
| **CameraFollow** | Main Camera | 목표(Player) 위치로 카메라를 부드럽게 이동 |

**작업 씬**: 상단 씬 탭 이름을 팀에서 정한 메인 씬(예: `Game`)으로 통일해 두면 이후 전투·UI 단계와 맞추기 쉽다.

예제 코드 안의 주석은 아래처럼 읽는다.

## PlayerMove — 좌우 이동

### 개요

- **역할**: 키보드 좌우 입력(A/D 또는 화살표)을 받아, 플레이어의 **가로 속도만** 매 프레임 설정한다. 세로 속도(y)는 건드리지 않아 점프·낙하와 겹치지 않게 한다.
- **부착**: Player 오브젝트. Rigidbody2D와 같은 오브젝트에 있어야 한다.
- **흐름**: `Awake`에서 Rigidbody2D 참조 한 번 확보 → `Update`에서 `GetAxisRaw("Horizontal")`로 방향 읽기 → `linearVelocity`의 x만 갱신.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **GetAxisRaw** | `GetAxis`는 키를 누를 때 -1~1이 **서서히** 변한다. 액션/플랫포머는 **즉시** 방향 전환이 필요하다. |
| **가로만 velocity 설정** | `y`까지 덮어쓰면 점프 직후 이동키로 **세로 속도가 날아간다**. 가로만 바꾼다. |
| **Rigidbody로 이동** | `transform.position`만 바꾸면 콜라이더와 **끼임·관통**이 잘 난다. 2D 플랫포머는 보통 Rigidbody2D로 이동한다. |
| **Awake에서 GetComponent** | `Update`보다 먼저 한 번만 실행되므로, 매 프레임 `GetComponent`를 반복 호출하지 않는다. |

### SerializeField를 쓴 이유

| 필드 | SerializeField 쓴 이유 |
|------|-------------------------|
| `moveSpeed` | **기획/밸런스 수치**. 코드 수정 없이 Inspector에서 이동 속도를 바꿀 수 있다. 강좌·팀마다 “빠르게/천천히”가 다르므로 에디터에서 조절 가능하게 둔다. |

`_rb`는 씬에서 끌어다 넣을 대상이 아니라 **같은 오브젝트의 Rigidbody2D**이므로 `GetComponent`로만 채운다. SerializeField로 노출할 필요 없다.

### 사용하는 Unity API (PlayerMove)

| API | 역할 |
|-----|------|
| `Rigidbody2D.linearVelocity` | 속도 벡터. x만 바꿔 좌우, y는 점프 스크립트와 공유 (Unity 6+. 구버전은 `velocity`) |
| `Input.GetAxisRaw("Horizontal")` | 키보드 좌우 축, 가감속 없이 -1, 0, 1 |
| `GetComponent<Rigidbody2D>()` | 같은 오브젝트의 Rigidbody2D 참조 |

### 코드 (PlayerMove.cs)

**파일**: `Assets/Scripts/PlayerMove.cs` (없으면 생성). Player에 컴포넌트로 부착.

```csharp
using UnityEngine;

public class PlayerMove : MonoBehaviour
{
    [SerializeField] float moveSpeed = 6f;   // [우리 멤버] Inspector에서 조절

    Rigidbody2D _rb;   // [우리 멤버]

    void Awake()   // [Unity가 호출] 씬 로드 직후 1회
    {
        _rb = GetComponent<Rigidbody2D>();   // [Unity API]
    }

    void Update()   // [Unity가 호출] 매 프레임
    {
        float dir = Input.GetAxisRaw("Horizontal");   // [Unity API] A/← = -1, D/→ = 1
        _rb.linearVelocity = new Vector2(dir * moveSpeed, _rb.linearVelocity.y);
        // y는 건드리지 않음 → 점프 중 세로 속도 유지
    }
}
```

## PlayerJump — 점프와 착지 판정

### 개요

- **역할**: “지금 땅에 붙어 있는가?”를 **발 밑 한 점**에 작은 원을 그려 Ground 레이어와 겹치는지로 판단하고, 땅일 때만 Space 키 **누른 순간 한 번** 위로 속도를 준다. 공중에서는 점프 불가.
- **부착**: Player 오브젝트. `groundCheck`는 **자식 빈 오브젝트**(Transform만 있음)를 Inspector에서 연결한다.
- **흐름**: `Awake`에서 Rigidbody2D 참조 → `Update`에서 `OverlapCircle(groundCheck.position, 반지름, groundLayer)`로 착지 여부 갱신 → 땅이고 `GetButtonDown("Jump")`이면 `linearVelocity.y = jumpForce`.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **OverlapCircle** | `OnCollisionStay2D`만 쓰면 **타일 경계·경사**에서 “땅에 붙었다”가 프레임마다 끊길 수 있다. **발 밑 작은 원**으로 Ground 레이어만 검사하면 조건이 안정적이다. |
| **LayerMask (groundLayer)** | “땅”만 검사해, 적·아이템·트리거와 구분한다. 레이어를 바꾸지 않고 코드만 보면 “어떤 레이어가 땅인지” 알 수 있다. |
| **GetButtonDown("Jump")** | “누르고 있는 동안”이 아니라 **눌렀을 때 한 프레임만** true. 연타로 여러 번 점프가 들어가는 것을 막는다. |
| **속도 직접 설정 (y = jumpForce)** | Rigidbody2D에 **힘(AddForce)**을 주면 무게·프레임마다 누적되어 제어가 어렵다. 이 단계에서는 “점프 순간 속도 = 고정값”으로 단순하게 둔다. |

### SerializeField를 쓴 이유

| 필드 | SerializeField 쓴 이유 |
|------|-------------------------|
| `jumpForce` | **기획/밸런스 수치**. 점프 높이는 게임마다 다르므로 Inspector에서 조절 가능하게 둔다. |
| `groundLayer` | **씬/프로젝트 설정과 연결**. “어떤 레이어를 땅으로 볼지”는 프로젝트마다 다르고, 코드에 레이어 번호를 박지 않고 Inspector에서 LayerMask로 고른다. |
| `groundCheck` | **계층 구조와 연결**. 발 밑 위치는 Player 자식 오브젝트(GroundCheck)의 Transform이다. 씬에서 드래그로 지정해야 하므로 반드시 SerializeField. |
| `groundCheckRadius` | **튜닝**. 원이 너무 작으면 경사에서 “안 붙은 것처럼” 되고, 너무 크면 계단에서 “아직 공중인데 땅”으로 인식할 수 있다. 에디터에서 보면서 조절할 수 있게 둔다. |

### 사용하는 Unity API (PlayerJump)

| API | 역할 |
|-----|------|
| `Physics2D.OverlapCircle(위치, 반지름, 레이어마스크)` | 원 안에 해당 레이어 콜라이더가 있으면 Collider2D 반환(참), 없으면 null |
| `Input.GetButtonDown("Jump")` | Input Manager 기본값으로 **Space** 다운 프레임에만 true |
| `Rigidbody2D.linearVelocity` | 점프 시 x는 유지, y만 jumpForce로 설정 |

**Inspector**: `Ground Check` 칸에 Player 자식 **GroundCheck** 오브젝트 드래그. **Ground Layer**에는 Project Settings → Tags and Layers에서 만든 **Ground** 레이어를 선택. 바닥 타일/플랫폼에는 Layer = **Ground** 적용.

### 코드 (PlayerJump.cs)

**파일**: `Assets/Scripts/PlayerJump.cs`. Player에 컴포넌트로 부착.

```csharp
using UnityEngine;

public class PlayerJump : MonoBehaviour
{
    [SerializeField] float jumpForce = 12f;
    [SerializeField] LayerMask groundLayer;
    [SerializeField] Transform groundCheck;
    [SerializeField] float groundCheckRadius = 0.1f;

    Rigidbody2D _rb;
    bool _isGrounded;

    void Awake()
    {
        _rb = GetComponent<Rigidbody2D>();
    }

    void Update()
    {
        _isGrounded = Physics2D.OverlapCircle(
            groundCheck.position,
            groundCheckRadius,
            groundLayer
        );

        if (_isGrounded && Input.GetButtonDown("Jump"))
        {
            _rb.linearVelocity = new Vector2(_rb.linearVelocity.x, jumpForce);
        }
    }
}
```

## CameraFollow — 카메라 추적

### 개요

- **역할**: **목표(플레이어)**의 위치를 매 프레임 읽어, 카메라를 그 위치(xy)로 **부드럽게** 이동시킨다. z(카메라와 플레이어 거리)는 유지한다.
- **부착**: Main Camera. `target`에는 **Player**의 Transform을 Inspector에서 연결한다.
- **흐름**: `LateUpdate`에서 target의 position으로 목표점(dest) 계산 → `Vector3.SmoothDamp`로 현재 카메라 위치에서 dest까지 보간.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **LateUpdate** | 플레이어 위치는 보통 **Update**에서 바뀐다. 카메라를 같은 프레임의 **앞쪽**에서 움직이면 한 프레임 어긋난 위치를 따라갈 수 있다. **LateUpdate**는 모든 Update 뒤에 호출되므로, 그 프레임의 **최종 플레이어 위치**를 기준으로 카메라를 맞출 수 있다. |
| **SmoothDamp** | `transform.position = target.position`처럼 **따라붙이기**만 하면 카메라가 플레이어에 딱 붙어 움직여 어색하다. SmoothDamp로 **살짝 늦게 따라오는** 보간을 쓰면 자연스럽고, 급정지·급가속 시 카메라가 덜 튄다. |
| **z는 유지** | 2D에서 카메라와 플레이어의 **깊이(z)**는 보통 고정값(-10 등)이다. target.position을 그대로 쓰면 z가 0이 되어 화면이 깨질 수 있으므로, 목표는 (target.x, target.y, 기존 camera.z)로 둔다. |

### SerializeField를 쓴 이유

| 필드 | SerializeField 쓴 이유 |
|------|-------------------------|
| `target` | **씬 구조와 연결**. “어떤 오브젝트를 따라갈지”는 씬마다 다를 수 있고(테스트용 빈 오브젝트 등), Player 프리팹이 아니라 **카메라 쪽**에서 “누구를 따라갈지” 지정하는 것이 자연스럽다. Inspector에서 드래그로 연결한다. |
| `smoothTime` | **느낌 튜닝**. 0에 가까우면 거의 즉시 따라가고, 크면 더 늦게 따라온다. 코드 수정 없이 에디터에서 조절할 수 있게 둔다. |

`_velocity`는 SmoothDamp의 **ref** 인자로 쓰는 내부 상태이므로, Inspector에 노출할 필요가 없다. SerializeField를 쓰지 않는다.

### 사용하는 Unity API (CameraFollow)

| API | 역할 |
|-----|------|
| `Vector3.SmoothDamp(현재, 목표, ref 속도, smoothTime)` | 현재 위치에서 목표로 부드럽게 이동할 때 사용. ref 변수는 엔진이 내부적으로 갱신한다. |
| `LateUpdate()` | 모든 Update 실행 후 한 번 호출 — 플레이어 이동 반영 뒤 카메라 갱신에 적합 |
| `Transform.position` | 카메라/타깃의 월드 좌표. z만 카메라 쪽 값을 유지해 2D 거리 보존 |

### 코드 (CameraFollow.cs)

**파일**: `Assets/Scripts/CameraFollow.cs`. **Main Camera**에 컴포넌트로 부착. Inspector에서 **Target**에 Player 드래그.

```csharp
using UnityEngine;

public class CameraFollow : MonoBehaviour
{
    [SerializeField] Transform target;
    [SerializeField] float smoothTime = 0.2f;

    Vector3 _velocity = Vector3.zero;   // SmoothDamp용 ref 변수

    void LateUpdate()
    {
        Vector3 dest = new Vector3(
            target.position.x,
            target.position.y,
            transform.position.z
        );
        transform.position = Vector3.SmoothDamp(
            transform.position, dest, ref _velocity, smoothTime
        );
    }
}
```

## 단계별 실습 순서

### 0. Player 오브젝트 준비 (씬에 올리기)

이동·점프를 테스트하려면 **Player + 바닥**이 있어야 한다.

1. Hierarchy → **Create Empty** → 이름 `Player`
2. **Add Component**: `Rigidbody2D`, `CapsuleCollider2D`, `SpriteRenderer`(선택)
3. **Rigidbody2D**: Gravity Scale = 3, **Freeze Rotation Z** 체크
4. Player **자식**: Create Empty → `GroundCheck`, **Local Position** `(0, -0.55, 0)`
5. **PlayerMove.cs**, **PlayerJump.cs** 스크립트 생성 후 Player에 부착. Inspector에서 **Ground Check**에 GroundCheck 오브젝트, **Ground Layer**에 Ground 선택
6. Player **Tag** = `Player`, **Layer** = `Player` (전투 단계에서 `CompareTag("Player")` 등으로 구분)
7. **Main Camera**에 **CameraFollow.cs** 부착. **Target**에 Player 드래그

### 4. 테스트 바닥 · 프리팹

1. **2D Square** 또는 타일로 바닥 생성 → **BoxCollider2D** → Layer **Ground**
2. Player를 바닥 위에 두고 Play
3. 완료 후 Player를 `Assets/.../Prefabs/`로 드래그해 **Player.prefab** 저장

## 동작이 안 될 때 (순서대로 확인)

| # | 증상 / 확인 |
|---|-------------|
| 1 | **공중에서 무한 점프** → `groundLayer`가 비었거나 바닥이 Ground 레이어가 아님 |
| 2 | **점프가 전혀 안 됨** → `groundCheck` 미연결, 또는 위치가 발보다 위/반지름 너무 작음 |
| 3 | **좌우만 되고 안 떨어짐** → Rigidbody2D 없음 또는 Body Type이 Kinematic으로 잘못 설정 |
| 4 | **카메라가 안 따라옴** → `CameraFollow.target`에 Player 미연결, 카메라에 스크립트 없음 |
| 5 | **Unity 6 경고 (velocity)** → `linearVelocity` 사용. 구버전은 `velocity` |
