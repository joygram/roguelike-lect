## 구현 개요

**스크립트 역할 요약**

| 스크립트 | 부착 위치 | 역할 |
|----------|-----------|------|
| **AugmentData** | 에셋(ScriptableObject) | 증강 하나의 수치·아이콘·설명 |


## AugmentData — 증강 수치·아이콘 에셋

### 개요

- **역할**: 증강 **하나**의 ID·이름·설명·등급·아이콘·능력치 보너스(HP·이동속도·공격력·쿨감·피해감소)를 ScriptableObject로 담는다. 코드 수정 없이 에셋만 추가해 증강을 늘릴 수 있다.
- **부착**: 에셋. Create → Roguelike → Augment Data.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **ScriptableObject** | 증강마다 새 클래스를 만들면 파일이 늘어난다. SO 하나로 에셋만 추가해 증강 종류를 늘린다. |

### SerializeField를 쓴 이유

모든 필드는 에셋 데이터이므로 Inspector 편집용. public 또는 SerializeField.

### 사용하는 Unity API (AugmentData)

| API | 역할 |
|-----|------|
| `ScriptableObject` | 씬과 분리된 데이터 에셋. 게임 오브젝트에 붙이지 않음 |
| `[CreateAssetMenu]` | Project 우클릭 메뉴에 이 에셋 생성 항목 추가 |
| `[TextArea]` | Inspector에서 여러 줄 입력 가능한 텍스트 필드 |
| `Sprite` | 이미지 에셋 참조 타입 (UI 아이콘에 사용) |

```csharp
using UnityEngine;

/// <summary>증강 등급 정의</summary>
public enum AugmentRarity { Common, Rare, Epic }

/// <summary>
/// 증강 하나의 수치·설명·아이콘을 담는 데이터 에셋입니다.
/// 코드 수정 없이 에셋을 추가하는 것만으로 증강을 늘릴 수 있습니다.
/// </summary>
[CreateAssetMenu(fileName = "Augment", menuName = "Roguelike/Augment Data")]
public class AugmentData : ScriptableObject
{
    public string augmentId   = "aug_id";
    public string augmentName = "증강 이름";
    [TextArea] public string description = "효과 설명";
    public AugmentRarity rarity = AugmentRarity.Common;
    public Sprite icon;

    [Header("능력치 보너스 (0이면 적용 안 됨)")]
    public int   hpBonus            = 0;
    public float moveSpeedBonus     = 0f;   // 비율: 0.15 = +15%
    public int   attackDamageBonus  = 0;
    public float cooldownReduction  = 0f;   // 비율: 0.2 = 쿨타임 -20%
    public int   damageReduction    = 0;
}
```

## PlayerStats — 증강 누적 능력치

### 개요

- **역할**: **기본 능력치**에 적용된 증강 효과를 **누적**해 최종 MaxHp·MoveSpeed·AttackBonus·DamageReduction을 제공한다. DontDestroyOnLoad로 씬 전환 후에도 유지. ApplyAugment 호출 시 목록에 추가 후 RecalculateStats.
- **부착**: 씬 오브젝트(싱글톤). 메인 메뉴 또는 첫 스테이지 씬에 한 번 배치.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **List&lt;AugmentData&gt; 누적 + RecalculateStats** | 증강을 하나씩 적용할 때마다 목록에 추가하고, 전체를 다시 합산해 최종치를 낸다. 증강 제거가 없다면 단순 누적이면 된다. |
| **DontDestroyOnLoad** | 씬이 바뀌어도 증강이 유지되어야 하므로, PlayerStats 오브젝트가 파괴되지 않게 한다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `baseMaxHp` 등 | **기획/밸런스**. 런 시작 시 기본값. Inspector에서 조절. |

### 설계 개념 (참고)

증강 효과는 런(한 판) 동안 누적됩니다. 증강을 적용할 때마다 `_appliedAugments` 목록에 추가하고, 전체 목록을 기본값에 더해 최종 능력치를 재계산합니다.

```
기본 HP(100) + 강철심장(+20) + 또 강철심장(+20) = 최종 HP 140
```

`DontDestroyOnLoad`로 씬 전환 후에도 증강이 유지됩니다. 메인 메뉴로 돌아가면 `ResetAugments()`로 초기화합니다.

### 사용하는 Unity API

| API | 역할 |
|-----|------|
| `DontDestroyOnLoad(gameObject)` | 씬 전환 후에도 오브젝트가 파괴되지 않음 |
| `Awake()` | Start보다 먼저 실행. 싱글톤 초기화에 사용 |

```csharp
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 증강 효과가 누적 반영되는 플레이어 능력치 관리자.
/// PlayerHealth, PlayerCombat 등은 이 값을 읽어서 사용합니다.
/// </summary>
public class PlayerStats : MonoBehaviour
{
    public static PlayerStats Instance { get; private set; }

    [Header("기본 능력치 (런 시작 기준값)")]
    public int   baseMaxHp           = 100;
    public float baseMoveSpeed       = 5f;
    public int   baseAttackDamage    = 0;
    public int   baseDamageReduction = 0;

    // 최종 능력치 — 외부에서는 읽기만 가능
    public int   MaxHp           => _maxHp;
    public float MoveSpeed       => _moveSpeed;
    public int   AttackBonus     => _attackBonus;
    public int   DamageReduction => _damageReduction;

    private int   _maxHp;
    private float _moveSpeed;
    private int   _attackBonus;
    private int   _damageReduction;

    private List<AugmentData> _appliedAugments = new List<AugmentData>();

    void Awake()
    {
        if (Instance != null && Instance != this) { Destroy(gameObject); return; }
        Instance = this;
        DontDestroyOnLoad(gameObject);
        RecalculateStats();
    }

    /// <summary>증강을 하나 적용하고 능력치를 재계산합니다.</summary>
    public void ApplyAugment(AugmentData aug)
    {
        if (aug == null) return;
        _appliedAugments.Add(aug);
        RecalculateStats();
    }

    /// <summary>런 종료 시 증강 목록을 초기화하고 기본값으로 되돌립니다.</summary>
    public void ResetAugments()
    {
        _appliedAugments.Clear();
        RecalculateStats();
    }

    void RecalculateStats()
    {
        _maxHp           = baseMaxHp;
        _moveSpeed       = baseMoveSpeed;
        _attackBonus     = baseAttackDamage;
        _damageReduction = baseDamageReduction;

        foreach (var aug in _appliedAugments)
        {
            _maxHp           += aug.hpBonus;
            _moveSpeed       *= 1f + aug.moveSpeedBonus;
            _attackBonus     += aug.attackDamageBonus;
            _damageReduction += aug.damageReduction;
        }
    }
}
```

### PlayerHealth와 연결

10단계에서 PlayerHealth가 PlayerStats의 최대 HP와 피해 감소를 반영하도록 수정합니다.

```csharp
// PlayerHealth.cs — Start() 수정
void Start()
{
    if (PlayerStats.Instance != null)
        maxHp = PlayerStats.Instance.MaxHp;

    currentHp = maxHp;
    GameEvents.RaisePlayerHpChanged(currentHp, maxHp);
}

// TakeDamage — 피해 감소 적용
public void TakeDamage(int dmg)
{
    int reduction = PlayerStats.Instance?.DamageReduction ?? 0;
    int actualDmg = Mathf.Max(1, dmg - reduction);

    currentHp = Mathf.Max(0, currentHp - actualDmg);
    GameEvents.RaisePlayerHpChanged(currentHp, maxHp);

    if (currentHp <= 0) GameEvents.RaiseGameOver();
}
```

## AugmentManager — 증강 랜덤 3선택·적용

### 개요

- **역할**: **StageCleared** 구독 시 PrepareChoices()로 전체 목록에서 **중복 없이** 3개를 뽑아 UI에 전달. UI가 카드 선택 시 SelectAugment(aug)를 호출하면 PlayerStats.ApplyAugment를 실행한다. DontDestroyOnLoad.
- **부착**: 씬 오브젝트(싱글톤).

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **중복 없이 3개** | Random.Range로 3번 뽑으면 같은 증강이 두 번 나올 수 있다. 리스트에서 하나 뽑고 제거하는 것을 3번 반복하면 중복 없이 3개를 얻을 수 있다. |
| **StageCleared 구독** | AugmentSelectUI와 같이 "스테이지 클리어 시"만 반응하면 되므로, GameEvents 한 번으로 둘 다 동작시킬 수 있다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `allAugments` | **씬에서 연결**. 게임에 존재하는 모든 증강 에셋 배열. Inspector에서 드래그. |

### 설계 개념 (참고)

AugmentManager는 두 역할을 합니다.

1. **랜덤 3선택**: 전체 증강 목록에서 중복 없이 3개를 뽑아 UI에 전달
2. **선택 처리**: UI가 "이 카드 선택됨"을 알려오면 `PlayerStats.ApplyAugment` 호출

### 사용하는 Unity API

| API | 역할 |
|-----|------|
| `Random.Range(0, n)` | 0 이상 n 미만 정수 난수 |
| `GameEvents.StageCleared` | 스테이지 클리어 이벤트 구독 |

```csharp
using System.Collections.Generic;
using UnityEngine;

/// <summary>
/// 증강 목록에서 랜덤 3개를 선택하고, 선택 결과를 PlayerStats에 전달합니다.
/// </summary>
public class AugmentManager : MonoBehaviour
{
    public static AugmentManager Instance { get; private set; }

    [Tooltip("게임에 존재하는 모든 증강 에셋 목록")]
    [SerializeField] AugmentData[] allAugments;

    void Awake()
    {
        if (Instance != null && Instance != this) { Destroy(gameObject); return; }
        Instance = this;
        DontDestroyOnLoad(gameObject);
    }

    void OnEnable()  { GameEvents.StageCleared += OnStageCleared; }
    void OnDisable() { GameEvents.StageCleared -= OnStageCleared; }

    void OnStageCleared() => PrepareChoices();

    /// <summary>전체 목록에서 중복 없이 3개를 뽑아 반환합니다.</summary>
    public AugmentData[] PrepareChoices()
    {
        if (allAugments == null || allAugments.Length == 0)
            return new AugmentData[0];

        var pool   = new List<AugmentData>(allAugments);
        var chosen = new List<AugmentData>();
        int count  = Mathf.Min(3, pool.Count);

        for (int i = 0; i < count; i++)
        {
            int idx = Random.Range(0, pool.Count);
            chosen.Add(pool[idx]);
            pool.RemoveAt(idx);
        }
        return chosen.ToArray();
    }

    /// <summary>증강 UI 카드가 선택됐을 때 호출됩니다.</summary>
    public void SelectAugment(AugmentData aug)
    {
        PlayerStats.Instance?.ApplyAugment(aug);
    }
}
```

**에디터 배치**: 씬에 빈 오브젝트 `AugmentManager` 생성 → `AugmentManager.cs` 부착 → **All Augments** 배열에 증강 에셋 연결.

## AugmentSelectUI — 증강 선택 패널

### 개요

- **역할**: **StageCleared** 구독 시 AugmentManager.PrepareChoices()로 3개 받아 카드 슬롯에 Setup하고 패널을 연다. timeScale = 0으로 게임 일시정지. 카드 선택 시 SelectAugment 호출·패널 닫기·timeScale = 1·다음 씬 로드.
- **부착**: Canvas 아래 AugmentPanel.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **timeScale = 0** | 선택하는 동안 게임이 진행되지 않게 한다. UI는 unscaledTime이라 버튼은 동작한다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `panel` | **씬에서 연결**. 패널 GameObject. |
| `cardSlots` | **씬에서 연결**. AugmentCardUI 배열. |
| `nextSceneName` | **기획/씬**. 선택 후 로드할 씬 이름. |

### 사용하는 Unity API (AugmentSelectUI)

| API | 역할 |
|-----|------|
| `GameObject.SetActive(bool)` | 패널 활성/비활성 전환 |
| `Time.timeScale` | 0: 선택 중 게임 일시정지, 1: 선택 후 재개 |

```csharp
using UnityEngine;

/// <summary>
/// 증강 선택 패널 전체를 제어합니다.
/// AugmentManager에서 선택지를 받아 카드 UI에 전달하고,
/// 선택이 완료되면 패널을 닫고 다음 씬으로 이동합니다.
/// </summary>
public class AugmentSelectUI : MonoBehaviour
{
    [SerializeField] GameObject panel;
    [SerializeField] AugmentCardUI[] cardSlots;
    [SerializeField] string nextSceneName;

    void OnEnable()  { GameEvents.StageCleared += OnStageCleared; }
    void OnDisable() { GameEvents.StageCleared -= OnStageCleared; }

    void OnStageCleared()
    {
        AugmentData[] choices = AugmentManager.Instance?.PrepareChoices();
        if (choices == null || choices.Length == 0) return;

        for (int i = 0; i < cardSlots.Length; i++)
        {
            if (i < choices.Length)
                cardSlots[i].Setup(choices[i], this);
            else
                cardSlots[i].gameObject.SetActive(false);
        }

        if (panel != null) panel.SetActive(true);
        Time.timeScale = 0f;
    }

    public void OnCardSelected(AugmentData aug)
    {
        AugmentManager.Instance?.SelectAugment(aug);
        if (panel != null) panel.SetActive(false);
        Time.timeScale = 1f;

        if (!string.IsNullOrEmpty(nextSceneName))
            GameManager.Instance?.GoToScene(nextSceneName);
    }
}
```

**씬 구성:**

```
Canvas
  └─ AugmentPanel (비활성 상태로 시작)
       ├─ TitleText ("증강을 선택하세요")
       └─ CardContainer
            ├─ AugmentCard_0
            ├─ AugmentCard_1
            └─ AugmentCard_2
```

**에디터 연결**: `AugmentPanel`에 `AugmentSelectUI.cs` 부착 → Panel·CardSlots·NextSceneName 설정.

## Part 5. AugmentCardUI.cs

### 사용하는 Unity API

| API | 역할 |
|-----|------|
| `Image.sprite` | Image 컴포넌트에 스프라이트 할당 |
| `Image.color` | 이미지 색상 변경 (등급별 카드 색) |
| `TextMeshProUGUI.text` | TMPro 텍스트 내용 설정 |

```csharp
using UnityEngine;
using UnityEngine.UI;
using TMPro;

/// <summary>
/// 증강 카드 하나의 UI입니다.
/// 아이콘·이름·설명·등급 색상을 표시하고, 클릭 시 부모 UI에 알립니다.
/// </summary>
public class AugmentCardUI : MonoBehaviour
{
    [SerializeField] Image           iconImage;
    [SerializeField] TextMeshProUGUI nameText;
    [SerializeField] TextMeshProUGUI descText;
    [SerializeField] Image           cardBackground;

    [Header("등급별 카드 배경색")]
    [SerializeField] Color commonColor = new Color(0.6f, 0.6f, 0.6f);
    [SerializeField] Color rareColor   = new Color(0.2f, 0.7f, 0.3f);
    [SerializeField] Color epicColor   = new Color(0.8f, 0.2f, 0.2f);

    private AugmentData     _data;
    private AugmentSelectUI _parentUI;

    public void Setup(AugmentData data, AugmentSelectUI parentUI)
    {
        _data     = data;
        _parentUI = parentUI;
        gameObject.SetActive(true);

        if (iconImage != null) iconImage.sprite = data.icon;
        if (nameText  != null) nameText.text    = data.augmentName;
        if (descText  != null) descText.text    = data.description;

        if (cardBackground != null)
        {
            cardBackground.color = data.rarity switch
            {
                AugmentRarity.Common => commonColor,
                AugmentRarity.Rare   => rareColor,
                AugmentRarity.Epic   => epicColor,
                _                    => commonColor
            };
        }
```

## AugmentListUI — 획득 증강 목록 (TAB)

### 개요

- **역할**: **TAB** 키로 패널을 토글하고, 열릴 때 PlayerStats.AppliedAugments를 읽어 슬롯 프리팹을 Instantiate해 목록을 갱신한다. "지금까지 고른 증강"을 확인용으로 보여 준다.
- **부착**: Canvas 또는 전용 오브젝트.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **Refresh 시 기존 슬롯 Destroy 후 재생성** | 목록이 바뀔 때마다 기존 슬롯을 지우고 AppliedAugments 기준으로 다시 만든다. 간단하고, 증강 순서가 바뀌어도 그대로 반영된다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `panel` | **씬에서 연결**. 목록 패널 GameObject. |
| `content` | **씬에서 연결**. 슬롯이 추가될 부모 Transform. |
| `slotPrefab` | **씬에서 연결**. 아이콘+이름 한 행 프리팹. |

### 왜 필요한가 (참고)

증강을 여러 번 고른 후, 플레이어가 "지금까지 어떤 증강을 가지고 있지?"를 확인할 수 있어야 합니다. TAB 키를 누르면 팝업 패널이 열리고, 현재 `PlayerStats`가 보유한 증강 목록을 표시합니다.

### 사용하는 Unity API

| API | 역할 |
|-----|------|
| `Input.GetKeyDown(KeyCode.Tab)` | TAB 키 입력 감지 |
| `GameObject.SetActive(bool)` | 패널 표시·숨김 |
| `Instantiate(prefab, parent)` | 슬롯 프리팹을 목록에 동적 생성 |
| `Transform.parent` | 생성한 오브젝트를 Content 아래에 배치 |

### 씬 구조

```
AugmentListPanel (비활성 상태로 시작)
  ├─ Title (TextMeshPro — "획득한 증강")
  └─ Content  ← 동적으로 슬롯 추가
       └─ [런타임에 AugmentSlot 생성]
```

`AugmentSlot` 프리팹: 아이콘 Image + 이름 Text로 구성된 작은 행.

### AugmentListUI.cs

```csharp
using UnityEngine;
using TMPro;
using UnityEngine.UI;

/// <summary>
/// TAB 키로 열리는 획득 증강 목록 패널입니다.
/// PlayerStats.appliedAugments를 읽어 슬롯을 동적으로 생성합니다.
/// </summary>
public class AugmentListUI : MonoBehaviour
{
    [SerializeField] GameObject panel;
    [SerializeField] Transform content;      // 슬롯이 추가될 부모
    [SerializeField] GameObject slotPrefab;  // 아이콘 + 이름 슬롯 프리팹

    bool _visible;

    void Update()
    {
        if (Input.GetKeyDown(KeyCode.Tab))
            Toggle();
    }

    void Toggle()
    {
        _visible = !_visible;
        if (panel != null) panel.SetActive(_visible);

        if (_visible) Refresh();
    }

    void Refresh()
    {
        if (content == null || slotPrefab == null) return;

        // 기존 슬롯 모두 제거
        foreach (Transform child in content)
            Destroy(child.gameObject);

        if (PlayerStats.Instance == null) return;

        foreach (var aug in PlayerStats.Instance.AppliedAugments)
        {
            GameObject slot = Instantiate(slotPrefab, content);

            // 슬롯 프리팹의 첫 번째 Image = 아이콘, 첫 번째 TMP_Text = 이름
            var img = slot.GetComponentInChildren<Image>();
            if (img != null && aug.icon != null)
                img.sprite = aug.icon;

            var txt = slot.GetComponentInChildren<TMP_Text>();
            if (txt != null)
                txt.text = aug.augmentName;
        }
    }
}
```

## EndingUI — 엔딩·결과 화면

### 개요

- **역할**: **GameEvents.GameCleared** 구독. 보스 클리어 시 패널을 켜고 timeScale = 0. "메인 메뉴" 버튼 시 PlayerStats.ResetAugments·GoToMainMenu.
- **부착**: 보스 씬 Canvas 또는 엔딩 전용 패널.

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `panel` | **씬에서 연결**. 엔딩 패널 GameObject. |
| `clearText` | **씬에서 연결**. "클리어!" 등 텍스트. |

### 왜 필요한가 (참고)

보스를 클리어하면 게임이 끝납니다. 단순히 메인 메뉴로 돌아가는 것보다, "클리어했습니다" 화면을 보여주고 플레이어가 메뉴로 돌아갈지 선택하게 하는 것이 완성도를 높입니다.

엔딩 화면은 별도 씬(`Ending`)으로 만들거나, 보스 씬 위에 패널로 올릴 수 있습니다.

### 사용하는 Unity API

| API | 역할 |
|-----|------|
| `GameEvents.GameCleared` | 보스 클리어 이벤트 (P1이 BossHealth 사망 시 발행) |
| `GameObject.SetActive(bool)` | 패널 표시 |
| `Time.timeScale` | 클리어 연출 중 게임 일시 정지 |

### EndingUI.cs

```csharp
using UnityEngine;
using TMPro;

/// <summary>
/// 보스 클리어 후 엔딩 화면을 표시합니다.
/// GameEvents.GameCleared 이벤트를 받아서 패널을 활성화합니다.
/// </summary>
public class EndingUI : MonoBehaviour
{
    [SerializeField] GameObject panel;
    [SerializeField] TMP_Text clearText;

    void OnEnable()  { GameEvents.GameCleared += OnGameCleared; }
    void OnDisable() { GameEvents.GameCleared -= OnGameCleared; }

    void OnGameCleared()
    {
        Time.timeScale = 0f;
        if (panel != null) panel.SetActive(true);
        if (clearText != null) clearText.text = "클리어!";
    }

    // "메인 메뉴" 버튼 OnClick()에 연결
    public void OnMainMenuClicked()
    {
        Time.timeScale = 1f;
        PlayerStats.Instance?.ResetAugments();
        GameManager.Instance?.GoToMainMenu();
    }
}
```
