# 이펙트·몬스터·스킬·레벨 성장 — 가장 쉬운 구현 방안

**이펙트**, **몬스터**, **스킬**, **레벨 성장**을 각각 가장 쉽게 구현하는 방법만 정리한 문서입니다.  
**이 문서만 따라 하면** Phase 3(몬스터·전투)에 필요한 코드가 모두 나옵니다. 외부 코드 검색 없이 **문서 내용을 추가하면서** 단계적으로 동작·확장됩니다.

**사용 지식·영역** ([01 §1.1](01-요구분석.md#11-이-프로젝트에서-사용하는-개발-영역지식-공통-기준)): **알고리즘**(적 AI·추적, 공격/충돌 판정, 스킬 범위) · **데이터 설계**(EnemyStatsData, 스킬 수치·쿨다운) · **소프트웨어 설계**(프리팹·스크립트 역할, 이펙트 생성/제거 흐름) · **UI·사용자 경험**(이펙트·시각 피드백).

**코드 구분**: **[Unity]** = 유니티 엔진 제공, **[우리 구현]** = 우리가 작성하는 스크립트·함수. 자세한 내용은 16번 문서 참고.

## 1.2 방안 A: 프리팹 생성 후 제거 (가장 쉬움 ★)

**방식**: 이펙트용 프리팹을 `Instantiate` → 일정 시간 후 `Destroy`.

### (1) 준비

- **이펙트 프리팹**: 빈 오브젝트 + 자식으로 **Sprite 1장**(번쩍임·타격 이펙트 등) 또는 Particle System
- 프리팹에 **자동 제거 스크립트** 붙이기 (아래 참고).

### (2) 자동 제거 스크립트 (이펙트 프리팹에 붙임)

```csharp
// [우리 구현] SimpleEffectLife.cs — 이펙트 프리팹에 붙이는 스크립트
using UnityEngine;

public class SimpleEffectLife : MonoBehaviour   // [Unity] MonoBehaviour
{
    public float lifeTime = 0.5f;   // [C#] 우리가 정한 값 (초 후 제거)
    float timer;                     // [C#]

    void Update()                    // [Unity] 매 프레임 호출
    {
        timer += Time.deltaTime;     // [Unity] 경과 시간
        if (timer >= lifeTime)
            Destroy(gameObject);     // [Unity] 이 오브젝트 제거
    }
}
```

### (3) 사용하는 쪽 (공격·피격 시 한 줄)

```csharp
// [우리 구현] 공격/피격 코드에서 호출. [Unity] Instantiate = 프리팹을 씬에 하나 생성
Instantiate(hitEffectPrefab, hitPosition, Quaternion.identity);
```

- **생성 위치**: `hitPosition`에 공격이 닿은 위치 또는 캐릭터 위치 전달.
- **가장 쉬운 확장**: 이펙트 프리팹을 여러 종류 만들고, 상황별로 다른 프리팹만 `Instantiate`하면 됨.

## 2.2 방안 A: 1종 프리팹 + 데이터만 변경 (가장 쉬움 ★)

**방식**: 몬스터 프리팹은 **1종**만 두고, 스폰할 때 **EnemyStatsData(ScriptableObject)**만 바꿔 끼운다.  
체력·속도·데미지는 데이터에서 읽고, 외형은 나중에 스프라이트/머티리얼만 바꾸면 됨.

### (1) 데이터 (기존 EnemyStatsData 활용)

```csharp
// EnemyStatsData.cs (ScriptableObject)
[CreateAssetMenu(fileName = "EnemyStats", menuName = "Roguelike/Enemy Stats")]
public class EnemyStatsData : ScriptableObject
{
    public string enemyName = "슬라임";
    public int maxHp = 20;
    public float moveSpeed = 2f;
    public int damage = 5;
    public int scoreValue = 10;
    // (선택) 외형 구분용
    public Sprite sprite;
    public Color color = Color.red;
}
```

### (2) 몬스터 1개 동작 (AI·체력·데미지)

```csharp
// EnemyController.cs — 몬스터 프리팹에 1개만 붙임
using UnityEngine;

public class EnemyController : MonoBehaviour
{
    public EnemyStatsData stats;  // 스폰할 때 설정
    int currentHp;
    Transform player;

    void Start()
    {
        if (stats == null) return;
        currentHp = stats.maxHp;
        player = GameObject.FindGameObjectWithTag("Player")?.transform;
    }

    void Update()
    {
        if (player == null) return;
        // 가장 쉬운 AI: 플레이어 쪽으로 이동
        transform.position = Vector3.MoveTowards(
            transform.position, player.position, stats.moveSpeed * Time.deltaTime);
    }

    public void TakeDamage(int amount)
    {
        currentHp -= amount;
        if (currentHp <= 0) Destroy(gameObject);
    }

    void OnCollisionEnter2D(Collision2D other)  // 2D
    {
        if (other.gameObject.CompareTag("Player"))
            other.gameObject.SendMessage("TakeDamage", stats.damage, SendMessageOptions.DontRequireReceiver);  // 2D: Collision2D
    }
}
```

### (3) 스폰 시 종류만 바꾸기

```csharp
// 스폰할 때
GameObject enemy = Instantiate(enemyPrefab, spawnPos, Quaternion.identity);
enemy.GetComponent<EnemyController>().stats = slimeStats;  // 또는 batStats
```

- **슬라임/박쥐**: ScriptableObject만 SlimeStats, BatStats 두 개 만들면 됨. **같은 프리팹, 다른 데이터**로 동작.

## 3.2 방안 A: 단순 공격 확장 (가장 쉬움 ★)

**방식**: “스킬 = 쿨다운이 있는 공격 하나”. 기존 공격에 **쿨타임**과 **데미지 배율(또는 고정 데미지)**만 넣는다.

### (1) 플레이어 쪽 (공격 1개를 스킬처럼 사용)

```csharp
// 예: 플레이어가 Z키로 “스킬” 사용 — 기존 공격과 같은 로직, 데미지·쿨만 다름
public class PlayerCombat : MonoBehaviour
{
    public int normalAttackDamage = 10;
    public float normalCooldown = 0.5f;
    public int skillDamage = 30;           // 스킬은 데미지 3배
    public float skillCooldown = 2f;       // 스킬 쿨 2초
    float nextNormalTime, nextSkillTime;

    void Update()
    {
        if (Input.GetKeyDown(KeyCode.X)) TryAttack(normalAttackDamage, normalCooldown, ref nextNormalTime);
        if (Input.GetKeyDown(KeyCode.Z)) TryAttack(skillDamage, skillCooldown, ref nextSkillTime);
    }

    void TryAttack(int damage, float cooldown, ref float nextTime)
    {
        if (Time.time < nextTime) return;
        nextTime = Time.time + cooldown;
        // 여기서 실제 데미지 처리 (레이캐스트/충돌 등으로 적에게 damage 전달)
        DoAttack(damage);
    }

    void DoAttack(int damage) { /* 기존 공격 로직, damage만 파라미터로 */ }
}
```

- **가장 쉬운 스킬**: “Z키 = 데미지 큰 공격 + 쿨 2초” 한 가지만 넣어도 “스킬 1개” 구현 완료.

### (2) 스킬 2개 이상 (같은 패턴 반복)

- 스킬마다 `(데미지, 쿨, 키)`만 다르게 두고, `TryAttack(데미지, 쿨, ref 해당쿨타임)` 호출.
- 나중에 **SkillData ScriptableObject**로 데미지/쿨/키 묶어서 관리하면 “스킬 데이터 테이블”로 확장 가능.

## 4.2 방안 A: 경험치 + 레벨업 시 수치 증가 (가장 쉬움 ★)

**방식**:  
- 몬스터 처치 시 **경험치(EXP)** 부여.  
- EXP가 **필요 경험치**를 채우면 **레벨업** → 최대 HP·공격력 등 **고정 수치만 올림**.

### (1) 플레이어 쪽 변수

```csharp
// PlayerGrowth.cs (또는 기존 플레이어 스크립트에 필드 추가)
public class PlayerGrowth : MonoBehaviour
{
    public int level = 1;
    public int exp;
    public int[] expToNextLevel = { 10, 25, 50, 100, 200 };  // 레벨2~6 필요 경험치
    public int hpPerLevel = 10;    // 레벨업 시 maxHp 증가량
    public int attackPerLevel = 2;  // 레벨업 시 공격력 증가량

    PlayerStatsData stats;  // 또는 PlayerHealth 등
    int maxLevel = 6;

    public void AddExp(int amount)
    {
        exp += amount;
        while (level < maxLevel && exp >= expToNextLevel[level - 1])
        {
            exp -= expToNextLevel[level - 1];
            level++;
            OnLevelUp();
        }
    }

    void OnLevelUp()
    {
        // 수치만 올리기 (가장 쉬운 성장)
        stats.maxHp += hpPerLevel;
        stats.attackPower += attackPerLevel;
        // (선택) 풀 HP 회복
        // currentHp = stats.maxHp;
    }
}
```

### (2) 몬스터 처치 시 경험치 지급

```csharp
// EnemyController 사망 처리 부분에서
// FindObjectOfType<PlayerGrowth>()?.AddExp(stats.expValue);
// 또는 GameManager를 통해 PlayerGrowth.AddExp(stats.expValue) 호출
```

- **EnemyStatsData**에 `expValue = 5` 같은 필드만 추가하면 됨.

### (3) UI (가장 쉬운 표시)

- **레벨**: Text로 `"Lv." + level` 표시.  
- **경험치**: 슬라이더 또는 바 1개. `currentExp / expToNextLevel[level-1]` 비율로 채우기.
