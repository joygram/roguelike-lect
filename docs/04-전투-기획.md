# 04. 전투 — 기획

> **시스템**: 전투 시스템 | **담당**: P2 (적 설계) · P1 (플레이어 체력 수치) · 팀 전체 (밸런스 합의)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

맵에서 플레이어 혼자만 움직인다면 게임이 아니다. 적과의 상호작용이 생겨야 긴장감이 만들어진다. 전투 기획은 "어떤 적이 어떻게 행동하는가", "공격과 피격은 어떻게 판정되는가"를 결정한다.

## 전투 루프

게임의 전투는 다음 사이클로 반복된다.

```
플레이어 공격 입력
    ↓
히트박스 활성화 (짧은 시간)
    ↓
적과 겹치면 → 적 HP 감소
    ↓
적 HP 0 → 적 사망
    
적이 플레이어에 접촉
    ↓
플레이어 HP 감소
    ↓
플레이어 HP 0 → 게임 오버
```

이 흐름을 먼저 이해하고 수치를 채워야 한다.

---

## 플레이어 수치

| 항목 | 변수명 | 제안값 | 팀 결정 |
|------|--------|:------:|:-------:|
| 최대 HP | `maxHp` | 100 | |
| 공격 데미지 | `attackDamage` | 10 | |
| 공격 쿨다운 | `attackCooldown` | 0.5초 | |
| 피격 후 무적 시간 | `invincibleTime` | 0.5초 | |

무적 시간이 왜 필요한가: 적에게 닿는 순간 피격 처리가 일어나는데, 무적 시간 없이 적 위에 올라서면 매 프레임(초당 60번)마다 데미지가 들어간다. 0.5초 무적이면 그 사이 피하거나 공격할 기회가 생긴다.

## 공격 판정 방식

플레이어가 공격할 때 "어디까지가 공격 범위인가"를 판정하는 방법이 여러 가지다.

| 방식 | 설명 | 이 강좌 |
|------|------|:-------:|
| 접촉 판정 | 플레이어 몸에 닿으면 공격 | × (이동 중에도 공격됨) |
| 히트박스 (Trigger) | 별도 오브젝트를 공격 시에만 활성화 | ✅ |
| 투사체 | 총알 등 날아가는 오브젝트 | 선택 |

히트박스 방식을 선택하는 이유: 플레이어 몸 자체를 공격 콜라이더로 쓰면, 이동하는 동안 적에 닿을 때마다 공격이 발동된다. 히트박스는 공격 버튼을 눌렀을 때만 짧게 켜지는 별도 오브젝트이므로, "지금 공격 중인가"를 정확하게 제어할 수 있다.

## 적 AI 방식

| 방식 | 설명 | 구현 난이도 |
|------|------|:----------:|
| 플레이어 추적 | 항상 플레이어 방향으로 이동 | 낮음 |
| 순찰 + 감지 | 일정 구간 왕복하다가 플레이어 발견 시 추적 | 중간 |
| 패턴 AI | 정해진 패턴대로 행동 | 높음 |

처음에는 **플레이어 추적** 방식으로 시작한다. 단순하지만 충분히 위협적이다.

---

## 레이어 설정

물리 충돌 판정에서 "누가 누구와 충돌하는지"를 레이어로 제어한다.

| 레이어 | 용도 |
|--------|------|
| `Player` | 플레이어 오브젝트 |
| `Enemy` | 적 오브젝트 |
| `Ground` | 바닥·플랫폼 (이전 단계에서 설정) |
| `PlayerHitbox` | 플레이어 공격 히트박스 |

`Edit > Project Settings > Physics 2D > Layer Collision Matrix`에서 어떤 레이어가 어떤 레이어와 충돌할지 설정할 수 있다. 예를 들어 `PlayerHitbox`는 `Enemy`와만 충돌하도록 설정하면 바닥·플레이어 몸과의 오판정을 막을 수 있다.
