# 10 증강·게임 루프 — 기획

> **시스템**: 전투 시스템 · UI 시스템 | **담당**: P2 (증강 데이터 설계) · P3 (증강 선택·목록·메뉴·엔딩 UI 설계) · D1 (증강 아이콘 규격) · D2 (카드·메뉴·엔딩 이미지 규격)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

## 이 단계에서 만드는 것

로그라이크 게임의 핵심 재미인 **증강(Augment) 시스템**을 추가합니다.  
스테이지를 클리어할 때마다 3개 중 1개를 선택하는 증강 화면이 등장하고, 선택한 증강에 따라 이후 플레이가 달라집니다.

또한 전체 게임 루프 (메인 메뉴 → 게임 → 보스 클리어 → 엔딩 또는 재시작) 흐름을 완성합니다.

---

## 1. 증강 시스템이란

증강(Augment)은 **스테이지 클리어 보상**으로 주어지는 영구 능력 강화 효과입니다.

예시:

| 증강 이름 | 효과 |
|-----------|------|
| 강철 심장 | 최대 HP +20 |
| 신속 | 이동 속도 +15% |
| 날카로운 날 | 공격력 +5 |
| 빠른 손 | 기본 공격 쿨타임 -20% |
| 두꺼운 갑옷 | 받는 데미지 -2 |

---

## 2. 증강의 데이터 구조

증강 하나를 ScriptableObject(`AugmentData`)로 만들어, 코드를 수정하지 않고 에셋만 추가하면 새 증강을 늘릴 수 있습니다.

| AugmentData 필드 | 타입 | 설명 |
|------------------|------|------|
| `augmentId` | string | 고유 ID (예: "aug_steel_heart") |
| `augmentName` | string | 표시 이름 (예: "강철 심장") |
| `description` | string | 효과 설명 (예: "최대 HP가 20 증가합니다.") |
| `rarity` | Rarity | Common / Rare / Epic |
| `icon` | Sprite | UI에 표시할 아이콘 |
| `hpBonus` | int | HP 증가량 |
| `moveSpeedBonus` | float | 이동 속도 증가 비율 |
| `attackDamageBonus` | int | 공격력 증가량 |
| `cooldownReduction` | float | 쿨타임 감소 비율 |
| `damageReduction` | int | 피해 감소량 |

---

## 3. 증강 선택 화면 설계

스테이지 클리어 후 **3개 증강 카드가 나타나고**, 플레이어가 하나를 선택합니다.

```
┌──────────────────────────────────────────┐
│         스테이지 클리어!                   │
│                                          │
│  ┌────────┐  ┌────────┐  ┌────────┐    │
│  │ 강철심장│  │ 신속   │  │날카로운│    │
│  │  아이콘 │  │ 아이콘  │  │  날    │    │
│  │HP +20  │  │속도+15%│  │공격+5  │    │
│  │ [선택]  │  │ [선택]  │  │ [선택]  │    │
│  └────────┘  └────────┘  └────────┘    │
└──────────────────────────────────────────┘
```

- **카드 3장**: 전체 증강 목록에서 랜덤 3개 선택.
- **선택 후**: 증강 효과 즉시 적용 → 게임 재개 또는 다음 씬.

---

## 4. 게임 루프 흐름

완성된 게임 루프는 아래와 같습니다.

```
[메인 메뉴]
    ↓ (게임 시작 버튼)
[Stage_1_1]
    ↓ (출구 진입)
[Stage_1_2]
    ...
    ↓ (Stage_1_3 클리어)
[증강 선택 화면]
    ↓ (증강 선택)
[Stage_2_1]
    ...
    ↓ (Stage_Boss 클리어)
[엔딩/게임 오버 화면]
    ↓ (재시작 또는 메뉴)
[메인 메뉴]
```

증강 화면은 별도 씬 `AugmentSelect`로 만들거나, 현재 씬 위에 팝업 패널로 표시합니다.

---

## 5. 메인 메뉴 설계

게임 시작 전 첫 화면입니다.

| 구성 요소 | 설명 |
|-----------|------|
| 게임 타이틀 | 텍스트 또는 이미지 |
| 게임 시작 버튼 | → `Stage_1_1` 로드 |
| 종료 버튼 | → `Application.Quit()` |

---

## 6. 팀이 합의할 것

| 항목 | 결정 필요 |
|------|-----------|
| 증강 목록 | 최소 6개 권장 (3개 선택지 + 중복 없애기 위해) |
| 증강 화면 방식 | 별도 씬 / 현재 씬 팝업 |
| 증강 발생 시점 | 매 스테이지 클리어마다 / 특정 스테이지만 |
| 증강 효과 저장 | 런 동안 메모리 유지 (재시작 시 초기화) |
