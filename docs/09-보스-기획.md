## 이 단계에서 만드는 것

04 전투에서 만든 일반 적 AI를 기반으로, **체력 구간별로 패턴이 바뀌는 보스**를 설계합니다.  
보스는 로그라이크 게임의 핵심 재미 요소인 "긴장감 있는 큰 전투"를 담당합니다.

---

## 1. 보스가 일반 적과 다른 점

| 항목 | 일반 적 | 보스 |
|------|---------|------|
| 체력 | 적음 (예: 30) | 많음 (예: 200) |
| 공격 방식 | 접촉 또는 단순 돌진 | 여러 패턴 (탄막, 강타, 돌진) |
| 페이즈 | 없음 | 체력 구간마다 패턴 강화 |
| 씬 | 일반 스테이지 | 별도 보스 씬 (예: `Stage_Boss`) |
| 처치 후 | 점수 | 스테이지 클리어 → 다음 단계로 이동 |

---

## 2. 상태 머신(FSM)이란

보스는 "지금 뭘 하고 있나"에 따라 동작이 달라집니다.  
이 "지금 상태"를 코드로 관리하는 방식이 **상태 머신(Finite State Machine, FSM)** 입니다.

```
[Idle] → 잠깐 쉬다가 공격 선택
   ↓
[AttackA] → 탄막 발사
   ↓
[Idle] → 쿨타임 대기
   ↓
[AttackB] → 지면 강타
   ↓
[Idle] → ...
```

FSM을 쓰면 각 상태에서 하는 동작이 독립적이어서 버그가 생겨도 어느 상태에서 났는지 파악하기 쉽습니다.

---

## 3. 보스 기획 — 상태 목록

| 상태 | 설명 |
|------|------|
| **Idle** | 잠깐 멈추고 다음 공격을 선택 |
| **Move** | 플레이어 쪽으로 천천히 이동 |
| **AttackA** | 원형 탄막 — 여러 방향으로 투사체 발사 |
| **AttackB** | 지면 강타 — 보스 아래 범위에 충격파 |
| **AttackC** | 돌진 — 빠르게 플레이어 방향으로 이동 |
| **Dead** | 체력 0 → 클리어 이벤트 발행 |

---

## 4. 체력 구간 (페이즈) 설계

보스 체력이 줄어들수록 공격이 강해집니다.

| 페이즈 | HP 비율 | 변화 |
|--------|---------|------|
| **1페이즈** | 100% ~ 67% | 탄막 6발, 느린 이동 |
| **2페이즈** | 67% ~ 34% | 탄막 10발, 강타 추가 |
| **3페이즈** | 34% ~ 0% | 탄막 14발, 돌진 추가, 속도 상승 |

이 수치는 `BossData` ScriptableObject에 저장해, 코드를 수정하지 않고 Inspector에서 조정합니다.

---

## 5. 패턴 상세 설계

### AttackA — 원형 탄막

- 보스 중심에서 **N 방향**으로 동시에 투사체 발사.
- N = 페이즈에 따라 증가 (6 → 10 → 14).
- 투사체는 일정 속도로 직진, 화면 밖 또는 5초 후 삭제.
- **플레이어 대응**: 빈 방향을 찾아 이동하거나 점프로 피하기.

### AttackB — 지면 강타

- 보스가 잠깐 준비 자세 → 보스 발 아래 충격파 콜라이더 활성화.
- 충격파 범위는 좌우 약 2~3유닛.
- 활성 시간: 0.4초.
- **플레이어 대응**: 점프 또는 옆으로 피하기.

### AttackC — 돌진

- 3페이즈부터 추가.
- 플레이어 방향으로 0.6초간 빠르게 이동.
- 이동 중 플레이어에게 닿으면 데미지.
- **플레이어 대응**: 돌진 직전 반대 방향으로 이동.

## 6. 보스 씬 구성

보스전은 별도 씬 `Stage_Boss`에서 진행합니다.

| 구성 요소 | 설명 |
|-----------|------|
| 보스 오브젝트 | 씬 중앙 또는 위쪽에 배치 |
| 바닥 | 아레나형 — 넓은 단일 플랫폼 |
| 플레이어 스폰 | 씬 진입 시 바닥 왼쪽 또는 중앙 |
| 보스 HP 바 | Canvas에 별도 추가 (05 UI와 다른 보스 전용 UI) |
| 클리어 후 이동 | 보스 사망 이벤트 → 엔딩 또는 10 증강 씬 |

---

## 7. BossData 설계

| 필드 | 타입 | 설명 |
|------|------|------|
| `bossId` | string | "Boss_Final" |
| `maxHp` | int | 200 |
| `contactDamage` | int | 15 (보스에게 닿을 때) |
| `projectilePrefab` | GameObject | 탄막 프리팹 |
| `phases` | BossPhaseSettings[] | 페이즈별 설정 배열 |

**BossPhaseSettings** (페이즈 하나)

| 필드 | 설명 |
|------|------|
| `hpBelow` | 이 페이즈가 적용되는 HP 비율 (예: 1.0, 0.66, 0.33) |
| `moveSpeed` | 이동 속도 |
| `projectileCount` | 탄막 발사 수 |
| `attacks` | 이 페이즈에서 사용할 공격 목록 + 가중치 |

## 8. 팀이 합의할 것

| 항목 | 결정 필요 |
|------|-----------|
| 페이즈 수 | 2페이즈(간단) / 3페이즈(권장) |
| 구현할 공격 패턴 | AttackA, B 필수 / C 선택 |
| 보스 스프라이트 | 직접 제작 / 플레이스홀더 사용 |
| 보스 사망 후 | 증강 씬 / 엔딩 씬 / 메인 메뉴 |
