# 05. UI — 기획

> **시스템**: UI 시스템 | **담당**: P3 (UI 설계 주도) · P1 (이벤트 채널 설계) · D2 (UI 이미지 기준 합의)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

게임이 동작해도 플레이어가 자신의 상태를 모르면 판단을 할 수 없다. "내 HP가 얼마나 남았지?", "스킬 쿨타임이 남았나?", "게임이 끝난 건가?"를 화면에서 바로 알 수 있게 해주는 것이 UI의 역할이다.

이 단계(05)에서는 기본 HUD와 게임 오버를 만들고, 스킬·보스·증강 UI는 해당 기능을 구현하는 단계(07·09·10)에서 추가된다. 하지만 **전체 UI 구조는 지금 미리 설계**해두어야 나중에 뜯어고치지 않는다.

## Canvas 구조 설계

모든 UI는 하나의 Canvas 아래에 배치한다. **항상 보이는 것**과 **필요할 때만 보이는 것**을 분리해두면 관리가 쉽다.

```
Canvas
  ├─ HUD                          ← 항상 표시
  │    ├─ HpBar (배경 + 채우기)
  │    └─ SkillSlots (07단계에서 추가)
  │         ├─ SkillSlot_0
  │         ├─ SkillSlot_1
  │         └─ SkillSlot_2
  ├─ GameOverPanel   (비활성)     ← 사망 시 활성
  ├─ PausePanel      (비활성)     ← ESC 시 활성
  ├─ AugmentSelectPanel (비활성)  ← 10단계에서 추가
  └─ AugmentListPanel   (비활성)  ← 10단계에서 추가
```

보스 씬에는 별도로 `BossHpBar`가 포함된 Canvas를 추가한다 (09단계).

## HUD 설계

HUD(Heads-Up Display)는 게임 화면 위에 항상 떠있는 정보 표시 영역이다. 너무 크면 게임 화면을 가리고, 너무 작으면 안 보인다. 화면 모서리에 작게 배치하는 것이 일반적이다.

**HP 표시 방식 선택:**

| 방식 | 특징 |
|------|------|
| 슬라이더(Image Filled) | HP 비율이 바로 표시됨. 시각적으로 직관적 |
| 텍스트 (HP: 70/100) | 정확한 수치 표시. 구현 간단 |
| 하트 아이콘 | 귀여운 느낌. 정수 단위 HP에 적합 |

이 강좌에서는 **Image Filled** 방식의 HP 바를 사용한다. 빨간 막대가 HP에 비례해 줄어드는 방식이다.

**스킬 슬롯 설계 (07단계에서 구현):**

| 요소 | 설명 |
|------|------|
| 스킬 아이콘 | SkillData의 아이콘 이미지 |
| 키 표시 | X / Z / C 텍스트 |
| 쿨타임 링 | Image Filled(Radial360) — 쿨타임 동안 어두운 오버레이 |

## 게임 오버 화면 설계

게임 오버 시 보여줄 내용:
- 반투명 검은 오버레이 (게임 화면이 살짝 보이게)
- "GAME OVER" 텍스트
- "다시 시작" 버튼
- (선택) "메인 메뉴" 버튼

평상시에는 이 패널이 비활성화(`SetActive(false)`) 상태여야 한다. 플레이어가 죽으면 `GameManager`가 `Time.timeScale = 0`으로 멈추고, `GameOverPanelUI`가 패널을 활성화한다.

---

## 일시정지 패널 설계

| 요소 | 기능 |
|------|------|
| 반투명 배경 | 게임 화면이 뒤로 보임 |
| "일시정지" 텍스트 | 상태 표시 |
| "재개" 버튼 | 패널 닫기 + `Time.timeScale = 1` |
| "메인 메뉴" 버튼 | `GameManager.GoToMainMenu()` |

ESC 키를 누르면 `PausePanelUI`가 패널 표시 여부를 토글한다. 패널이 열릴 때 `Time.timeScale = 0`, 닫힐 때 `Time.timeScale = 1`.

## UI와 게임 로직의 분리

중요한 설계 원칙: **UI는 게임 로직을 직접 실행하지 않는다.**

예를 들어, 재시작 버튼은 씬을 다시 로드하는 역할을 한다. 이 로직이 UI 스크립트 안에 있으면 나중에 UI를 바꿀 때 로직도 같이 손봐야 한다. 대신 `GameManager`라는 별도 스크립트에 씬 로드 함수를 만들고, UI 버튼이 그 함수를 호출하는 방식이 유지보수가 쉽다.

HP 바도 마찬가지다. `PlayerHealth`의 HP 변화 이벤트를 받아서 화면에 표시하는 역할만 한다. HP를 직접 바꾸지 않는다.

```
[게임 로직]           [이벤트 채널]        [UI]
PlayerHealth    →  GameEvents.PlayerHpChanged  →  HpBarUI
PlayerHealth    →  GameEvents.GameOver         →  GameOverPanelUI
(ESC 입력)      →  (직접 감지)                →  PausePanelUI
```
