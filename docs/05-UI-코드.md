## 구분하기: 내가 만드는 것 vs Unity(API)가 주는 것

| 구분 | 이번 단계 예시 | 설명 |
|------|----------------|------|
| **내가 만드는 것** | 어떤 이벤트를 정의할지, 게임 오버 조건·재시작 흐름, HP 바 비율 계산 | 게임 흐름 규칙과 이벤트 설계는 **우리 책임** |
| **Unity(API)가 주는 것** | `Image.fillAmount`, `Time.timeScale`, `SceneManager.LoadScene`, `DontDestroyOnLoad`, `event Action` | 렌더링·시간 제어·씬 전환은 **엔진**이 처리 |

**왜 엔진 것을 쓰나?**  
씬 전환을 직접 구현하거나 시간을 멈추는 것은 엔진 내부를 건드려야 한다. `Time.timeScale`이나 `SceneManager`는 이를 한 줄로 해결한다. HP 바의 채우기 비율도 픽셀 단위 계산 없이 `fillAmount`에 0~1 값만 넣으면 된다.

## 구현 개요

이 단계에서는 두 가지를 함께 만든다.

1. **게임 흐름 로직** — `GameEvents`, `GameManager`: 게임 상태를 전달하는 뼈대
2. **UI** — HP 바, 게임 오버 패널, 일시정지 패널

**이 순서를 지키는 이유**: UI는 `GameEvents`를 구독해서 동작한다. 로직 파일 없이 UI만 만들면 컴파일이 되지 않는다.

```
[PlayerHealth] → GameEvents.RaiseGameOver()
                       ↓
        [GameManager] 수신 → Time.timeScale = 0
                       ↓
        [GameOverPanelUI] 수신 → 패널 활성화
```

**씬 구조**

```
씬 (Game)
├─ GameManager              ← GameManager (Script), DontDestroyOnLoad
├─ Canvas
│   ├─ HUD
│   │   ├─ HpBarBG          ← Image (배경)
│   │   └─ HpBarFill        ← Image (Filled, Horizontal)
│   ├─ GameOverPanel        ← 비활성 상태로 시작
│   │   ├─ GameOverText     ← TextMeshProUGUI
│   │   └─ RestartButton    ← Button
│   └─ PausePanel           ← 비활성 상태로 시작
│       ├─ PauseText
│       ├─ ResumeButton
│       └─ MainMenuButton
```

**스크립트 역할 요약**

| 스크립트 | 부착 위치 | 담당 | 역할 |
|----------|-----------|------|------|
| **GameEvents** | (파일만, 씬 불필요) | P1 | 이벤트 채널 모음 — static 클래스 |
| **GameManager** | 씬 빈 오브젝트 | P1 | 게임 흐름·씬 전환·싱글톤 |
| **HpBarUI** | HUD 오브젝트 | P3 | HP 변화 이벤트 구독 → fillAmount 갱신 |
| **GameOverPanelUI** | GameOverPanel | P3 | GameOver 이벤트 구독 → 패널 활성화 |
| **PausePanelUI** | Canvas 또는 빈 오브젝트 | P3 | ESC 입력 → 일시정지 토글 |

## GameEvents — 이벤트 채널

### 개요

- **역할**: 게임 전체에서 사용하는 이벤트 채널을 한 곳에 모은다. "HP가 바뀌었다", "게임 오버다"는 신호만 보내고, **누가 받는지는 알 필요 없다**.
- **부착**: 없음. **static 클래스**이므로 씬에 오브젝트가 없어도 어디서든 호출 가능하다.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **이벤트 방식** | 직접 참조 방식은 `PlayerHealth`가 `HpBarUI`를 알아야 한다. HP 바가 2개로 늘거나 없어지면 `PlayerHealth` 코드도 고쳐야 한다. 이벤트 방식은 `PlayerHealth`는 신호만 보내고, UI가 알아서 구독·해제한다. |
| **static 클래스** | 씬이 바뀌어도 `static`은 사라지지 않는다. GameManager 싱글톤 없이 어디서든 `GameEvents.RaiseXxx()`를 바로 호출할 수 있다. |
| **?.Invoke()** | 구독자가 없으면 `?.Invoke()`는 아무것도 안 한다. `event?.Invoke()` 없이 `event.Invoke()` 만 쓰면 구독자가 없을 때 NullReferenceException이 난다. |

### 사용하는 C# API

| API | 역할 |
|-----|------|
| `event Action` | 매개변수 없는 이벤트 채널 |
| `event Action<T>` | 매개변수 있는 이벤트 채널 |
| `?.Invoke()` | null 체크 후 발행 — 구독자 없어도 오류 안 남 |

### 코드 (GameEvents.cs)

**파일**: `Assets/Scripts/_Framework/GameEvents.cs`. 씬에 오브젝트 불필요.

```csharp
using System;

/// 발행: GameEvents.RaiseGameOver();
/// 구독: GameEvents.GameOver += 내함수;   (OnEnable에서)
/// 해제: GameEvents.GameOver -= 내함수;   (OnDisable에서)
public static class GameEvents
{
    // HP가 바뀔 때 — 현재 HP, 최대 HP 전달
    public static event Action<int, int> PlayerHpChanged;
    public static void RaisePlayerHpChanged(int current, int max)
        => PlayerHpChanged?.Invoke(current, max);

    // 플레이어 사망 → 게임 오버
    public static event Action GameOver;
    public static void RaiseGameOver() => GameOver?.Invoke();

    // 스테이지 모든 적 처치
    public static event Action StageCleared;
    public static void RaiseStageCleared() => StageCleared?.Invoke();

    // 스킬 사용 — 슬롯 인덱스, 쿨타임(초)
    public static event Action<int, float> SkillUsed;
    public static void RaiseSkillUsed(int slotIndex, float cooldown)
        => SkillUsed?.Invoke(slotIndex, cooldown);
}
```

## GameManager — 게임 흐름·싱글톤

### 개요

- **역할**: 씬 전환, `Time.timeScale` 제어, 게임 오버 처리를 총괄한다. **싱글톤**으로 씬이 바뀌어도 유지된다.
- **부착**: 씬 빈 오브젝트 `GameManager`. 씬에 **하나만** 있어야 한다.
- **흐름**: `Awake`에서 중복 제거 + `DontDestroyOnLoad` → `OnEnable`에서 GameOver 이벤트 구독 → `OnGameOver`에서 `timeScale = 0`.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **싱글톤 패턴** | 씬이 여러 개일 때 GameManager가 씬마다 새로 생기면 씬 전환 시 상태가 초기화된다. 하나만 유지되면 게임 전체에서 동일한 흐름 관리자를 쓸 수 있다. |
| **DontDestroyOnLoad** | 씬 전환 시 씬에 있는 오브젝트는 기본적으로 파괴된다. 이걸 붙이면 파괴되지 않는다. |
| **Time.timeScale = 0** | `Update`, `FixedUpdate`, 물리 연산이 모두 멈춘다. UI의 Canvas는 `unscaledTime`으로 업데이트되므로 버튼은 계속 동작한다. |
| **OnEnable/OnDisable 구독** | `Awake`에서 구독하면 오브젝트가 비활성화됐다 다시 활성화될 때 중복 구독이 생긴다. `OnEnable`/`OnDisable` 쌍으로 구독·해제하는 것이 안전하다. |

### SerializeField를 쓴 이유

SerializeField 필드 없음. GameManager는 씬 오브젝트나 에셋을 연결할 필요 없이 코드만으로 동작한다.

### 사용하는 Unity API (GameManager)

| API | 역할 |
|-----|------|
| `DontDestroyOnLoad(gameObject)` | 씬 전환 후에도 이 오브젝트 유지 |
| `SceneManager.LoadScene(name)` | 씬 이름으로 씬 전환 |
| `SceneManager.GetActiveScene().name` | 현재 씬 이름 반환 |
| `Time.timeScale` | 0 = 게임 정지, 1 = 정상 속도 |

### 코드 (GameManager.cs)

**파일**: `Assets/Scripts/_Framework/GameManager.cs`. 씬 빈 오브젝트에 부착.

```csharp
using UnityEngine;
using UnityEngine.SceneManagement;

public class GameManager : MonoBehaviour
{
    public static GameManager Instance { get; private set; }   // [우리 멤버] 싱글톤 참조

    void Awake()   // [Unity가 호출]
    {
        if (Instance != null && Instance != this) { Destroy(gameObject); return; }
        Instance = this;
        DontDestroyOnLoad(gameObject);   // [Unity API]
    }

    void OnEnable()  { GameEvents.GameOver += OnGameOver; }   // [Unity가 호출] 구독
    void OnDisable() { GameEvents.GameOver -= OnGameOver; }   // [Unity가 호출] 해제

    void OnGameOver() { Time.timeScale = 0f; }   // [Unity API] 게임 정지

    public void RestartGame()
    {
        Time.timeScale = 1f;
        SceneManager.LoadScene(SceneManager.GetActiveScene().name);   // [Unity API]
    }

    public void GoToMainMenu()
    {
        Time.timeScale = 1f;
        SceneManager.LoadScene("MainMenu");
    }
}
```

## HpBarUI — HP 바

### 개요

- **역할**: `GameEvents.PlayerHpChanged` 이벤트를 구독해, HP가 바뀔 때마다 `fillAmount`로 HP 바 이미지를 갱신한다.
- **부착**: HUD 오브젝트.
- **흐름**: `OnEnable`에서 구독 → `OnHpChanged`에서 `fillAmount = currentHp / maxHp` → `OnDisable`에서 해제.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **이벤트 구독** | `Update`로 매 프레임 HP를 읽으면 HP가 변하지 않아도 매 프레임 연산이 발생한다. 이벤트로 HP가 바뀔 때만 갱신하면 불필요한 연산이 없다. |
| **OnEnable/OnDisable 쌍** | HUD가 꺼진 상태에서 이벤트가 도착하면 null 참조 오류가 날 수 있다. `OnDisable`에서 구독을 해제하면 이를 막을 수 있다. |
| **Image.fillAmount** | HP 바를 직접 크기 조절(`RectTransform.sizeDelta`)하면 앵커 설정에 따라 위치도 같이 움직여 어색해진다. `fillAmount`는 이미지를 비율로 잘라 보여주므로 위치 변동 없이 자연스럽다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `fillImage` | **씬에서 연결**. HpBarFill 오브젝트의 Image 컴포넌트를 Inspector에서 드래그 연결 |

### 사용하는 Unity API (HpBarUI)

| API | 역할 |
|-----|------|
| `Image.fillAmount` | 0~1 비율로 이미지를 잘라서 표시 |
| `OnEnable` / `OnDisable` | 활성화·비활성화 시점에 이벤트 구독·해제 |

### 씬에서 HP 바 만들기

1. Canvas → Create Empty → 이름 `HUD`
2. HUD → **UI → Image** → 이름 `HpBarBG` (배경, 어두운 색)
3. HpBarBG → **UI → Image** → 이름 `HpBarFill`
   - **Image Type**: `Filled` / **Fill Method**: `Horizontal` / **Fill Origin**: `Left` / Color: 빨간색
   - Rect Transform: 부모와 같은 크기 (Stretch 앵커)
4. HUD에 `HpBarUI.cs` 부착 → **Fill Image** 슬롯에 `HpBarFill` 드래그

### 코드 (HpBarUI.cs)

**파일**: `Assets/Scripts/HpBarUI.cs`. HUD에 부착.

```csharp
using UnityEngine;
using UnityEngine.UI;

public class HpBarUI : MonoBehaviour
{
    [SerializeField] Image fillImage;   // [우리 멤버] Inspector에서 HpBarFill 연결

    void OnEnable()  { GameEvents.PlayerHpChanged += OnHpChanged; }   // [Unity가 호출] 구독
    void OnDisable() { GameEvents.PlayerHpChanged -= OnHpChanged; }   // [Unity가 호출] 해제

    void OnHpChanged(int currentHp, int maxHp)   // [우리 메소드]
    {
        if (fillImage == null || maxHp <= 0) return;
        fillImage.fillAmount = (float)currentHp / maxHp;   // [Unity API] int/int = int → 캐스팅 필수
    }
}
```

## GameOverPanelUI — 게임 오버 패널

### 개요

- **역할**: `GameEvents.GameOver` 이벤트를 받아 패널을 활성화한다. 재시작 버튼 클릭 시 GameManager에 위임한다.
- **부착**: GameOverPanel 오브젝트.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **게임 규칙을 모름** | 이 스크립트는 "게임 오버가 언제 일어나는지" 알 필요가 없다. GameEvents 신호가 오면 패널을 켜고, 버튼이 눌리면 GameManager에 맡기면 된다. |
| **재시작은 GameManager에 위임** | 씬 전환 코드를 UI에 두면 씬 이름이 바뀔 때 UI 코드를 수정해야 한다. GameManager에 두면 UI는 "재시작해 달라"고 요청만 한다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `panel` | **씬에서 연결**. GameOverPanel 오브젝트를 Inspector에서 드래그 연결. SetActive로 켜고 끄는 대상 |

### 씬에서 게임 오버 패널 만들기

1. Canvas → **UI → Panel** → 이름 `GameOverPanel`, **체크 해제**(비활성), Color `(0,0,0,0.7)`
2. 자식: TextMeshPro "GAME OVER", Button "다시 시작"
3. GameOverPanel에 `GameOverPanelUI.cs` 부착 → **Panel** 슬롯에 `GameOverPanel` 드래그
4. RestartButton → **On Click()** → `GameOverPanelUI.OnRestartClicked` 연결

### 코드 (GameOverPanelUI.cs)

```csharp
using UnityEngine;

public class GameOverPanelUI : MonoBehaviour
{
    [SerializeField] GameObject panel;

    void OnEnable()  { GameEvents.GameOver += ShowPanel; }
    void OnDisable() { GameEvents.GameOver -= ShowPanel; }

    void ShowPanel() { if (panel != null) panel.SetActive(true); }

    public void OnRestartClicked() { GameManager.Instance?.RestartGame(); }
}
```

> `Time.timeScale = 0`이어도 버튼이 동작하는 이유: Canvas는 `unscaledTime`으로 업데이트되어 게임이 정지된 상태에서도 UI 클릭이 정상 처리된다.

## PausePanelUI — 일시정지 패널

### 개요

- **역할**: ESC 키로 일시정지 패널을 토글. 열릴 때 `Time.timeScale = 0`, 닫힐 때 `Time.timeScale = 1`.
- **부착**: Canvas 또는 씬 빈 오브젝트.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **timeScale 토글** | 일시정지는 게임 시간이 멈추는 것. `timeScale = 0`이면 `Update`·물리·애니메이션 모두 멈춘다. UI만 `unscaledTime`으로 동작한다. |
| **bool _paused** | 토글 상태를 표현. `panel.activeSelf`로 대신할 수 있지만, 명시적인 상태 변수가 가독성이 좋다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `panel` | **씬에서 연결**. PausePanel 오브젝트를 Inspector에서 드래그 연결 |

### 씬에서 일시정지 패널 만들기

1. Canvas → **UI → Panel** → 이름 `PausePanel`, **체크 해제**, Color `(0,0,0,0.7)`
2. 자식: TextMeshPro "일시정지", Button "재개", Button "메인 메뉴"
3. `PausePanelUI.cs` 부착 → **Panel** 슬롯에 `PausePanel` 드래그
4. "재개" 버튼 → `PausePanelUI.OnResumeClicked` / "메인 메뉴" 버튼 → `PausePanelUI.OnMainMenuClicked`

### 코드 (PausePanelUI.cs)

```csharp
using UnityEngine;

public class PausePanelUI : MonoBehaviour
{
    [SerializeField] GameObject panel;
    bool _paused;

    void Update()   // [Unity가 호출]
    {
        if (Input.GetKeyDown(KeyCode.Escape)) Toggle();   // [Unity API]
    }

    void Toggle()
    {
        _paused = !_paused;
        if (panel != null) panel.SetActive(_paused);
        Time.timeScale = _paused ? 0f : 1f;   // [Unity API]
    }

    public void OnResumeClicked()
    {
        _paused = false;
        if (panel != null) panel.SetActive(false);
        Time.timeScale = 1f;
    }

    public void OnMainMenuClicked()
    {
        Time.timeScale = 1f;
        GameManager.Instance?.GoToMainMenu();
    }
}
```

## 단계별 실습 순서

1. **GameEvents.cs** 작성 (P1) → 이 파일 없이는 04 단계의 PlayerHealth가 컴파일 안 됨
2. **GameManager.cs** 작성 (P1) → 씬에 빈 오브젝트 배치·부착
3. **Canvas + CanvasScaler** 설정
4. **HP 바** 만들기 → HpBarUI.cs 작성·부착·연결
5. **GameOverPanel** 만들기 → GameOverPanelUI.cs 작성·부착·연결·버튼 OnClick 연결
6. **PausePanel** 만들기 → PausePanelUI.cs 작성·부착·연결·버튼 OnClick 연결
7. Play 테스트 — 피격 → HP 바 감소, HP 0 → 게임 오버 패널, ESC → 일시정지 확인

## 동작이 안 될 때 (순서대로 확인)

| # | 증상 / 확인 |
|---|-------------|
| 1 | **HP 바가 갱신 안 됨** → HpBarUI의 fillImage 미연결, 또는 GameEvents 연결 안 됨 |
| 2 | **게임 오버 패널이 안 나타남** → GameOverPanelUI의 panel 슬롯 미연결, 또는 OnEnable에서 구독 안 됨 |
| 3 | **재시작이 안 됨** → GameManager 씬에 없음, 또는 RestartButton OnClick 미연결 |
| 4 | **ESC 눌러도 일시정지 안 됨** → PausePanelUI 스크립트 없거나 panel 미연결 |
| 5 | **timeScale=0에서 버튼이 안 눌림** → Canvas가 없거나 Render Mode 확인 (Screen Space - Overlay여야 함) |
| 6 | **GameManager가 씬 전환 후 두 개가 됨** → Awake의 중복 제거 로직 확인 |

## 이번 단계에서 익히는 개념

- **이벤트 패턴** — 로직이 UI를 몰라도 되는 구조
- **static 클래스** — 씬이 바뀌어도 사라지지 않는 이벤트 채널
- **OnEnable/OnDisable 구독·해제** — 비활성 상태에서 이벤트가 도착해 오류 나는 것 방지
- **싱글톤** — 하나의 GameManager가 게임 전체에서 유지되는 방식
- **fillAmount** — 비율 기반 이미지 채우기
