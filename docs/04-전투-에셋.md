> **시스템**: 전투 시스템 | **에셋 담당**: D1 (적 스프라이트 · 타격 이펙트)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

전투에는 적 스프라이트와 타격 이펙트가 필요하다. 특히 이펙트는 "공격이 맞았다"는 피드백을 시각적으로 전달하는 역할로, 있고 없고에 따라 타격감이 크게 달라진다. 사운드와 함께 쓰면 그 효과는 배가 된다.

## 적 스프라이트

### 크기 결정 기준

적 캐릭터의 크기는 플레이어와의 시각적 관계로 결정한다.

- 기본 적(슬라임 등): 플레이어와 같거나 약간 작게 → **32×32 px**
- 중간 적: 플레이어보다 약간 크게 → **48×48 px** 또는 **64×64 px**
- 보스: 위협감을 주기 위해 플레이어의 2~4배 → **64×64 px ~ 128×128 px**

크기가 커질수록 씬에서 더 많은 공간을 차지한다. PPU 32 기준 32px = 1 unit이므로, 128px 보스는 4×4 unit 크기가 된다.

### 다수 종류 설계

적이 여러 종류라면 스프라이트 크기·형태로 강함을 암시할 수 있다.

| 적 종류 | 크기 | 형태 특징 |
|--------|------|----------|
| 슬라임 (기본) | 32×32 px | 둥글고 낮게 |
| 박쥐 | 32×32 px | 좌우로 넓게 |
| 고블린 | 32×48 px | 세로로 길게, 사람 형태 |
| 보스 | 96×96 px | 압도적 크기 |

### 스펙

| 항목 | 값 | 이유 |
|------|---|------|
| 파일 형식 | PNG | 투명 배경 필수 |
| 기본 크기 | 32×32 px | PPU 32 기준 1 unit (플레이어와 동일) |
| Pixels Per Unit | 32 | 팀 기준과 동일 |
| Filter Mode | Point (no filter) | 도트 그래픽 선명도 유지 |

### 저장 위치

```
Assets/Sprites/Enemy/
  enemy_slime.png
  enemy_bat.png
  enemy_boss.png
```

## 타격 이펙트

### 이펙트가 하는 역할

이펙트는 단순한 장식이 아니다. 플레이어에게 "지금 내 공격이 유효했는지"를 즉각 알려주는 **피드백 신호**다. 이펙트 없이 공격하면 적의 HP가 줄어드는 것을 HP 바가 없는 이상 알 수 없다.

게임 디자인에서 피드백 3종 세트가 있다:
- **시각**: 타격 이펙트 (이번 단계)
- **청각**: 타격음 (06 사운드 단계)
- **촉각**: 화면 흔들림, 히트스톱 (선택)

이 셋이 동시에 발생할 때 타격감이 가장 강하다.

### 이펙트 크기 설계

이펙트는 적 캐릭터보다 **같거나 작게** 만든다. 이펙트가 너무 크면:
- 화면이 어수선해져 적의 위치를 놓치게 된다
- 다수의 적이 있을 때 이펙트가 겹쳐 가시성이 떨어진다

| 이펙트 종류 | 권장 크기 | 이유 |
|-----------|:--------:|------|
| 일반 타격 | 16×16 px | 적(32px)보다 작게, 인식은 가능하게 |
| 강타격 | 32×32 px | 강한 공격은 크게 |
| 적 사망 | 32×32 px | 마지막 인상이므로 크게 |

### 이펙트 수명 설계

이펙트는 생성 직후 자동으로 사라져야 한다. 수명이 너무 길면 화면에 이펙트가 쌓이고, 너무 짧으면 인식이 안 된다.

| 수명 | 느낌 |
|------|------|
| 0.1초 미만 | 너무 빠름, 인식 어려움 |
| 0.2~0.4초 | 타격감 좋음 (권장) |
| 0.5초 이상 | 느림, 다음 공격 시 겹침 |

수명 코드(`Destroy(gameObject, lifetime)`)는 구현 단계에서 다룬다.

### 스프라이트 시트 vs 단일 이미지

- **단일 이미지**: 번쩍이는 별, 원 등 정적 이펙트. 구현이 단순하다.
- **스프라이트 시트**: 여러 프레임을 하나의 파일에 담아 순서대로 재생. 애니메이션 이펙트에 사용한다. Unity의 `Sprite Editor`에서 슬라이싱해서 사용한다.

처음에는 **단일 이미지**로 시작하고, 이후 애니메이션이 필요해지면 스프라이트 시트로 교체한다.

### 스펙

| 항목 | 값 | 이유 |
|------|---|------|
| 파일 형식 | PNG | 투명 배경 필수 |
| 크기 | 16×16 px (일반 타격) | 적(32px)보다 작게 |
| Filter Mode | Point (no filter) | |

### 저장 위치

```
Assets/Sprites/Effects/
  hit_fx.png            ← 일반 타격 이펙트
  hit_fx_strong.png     ← 강타격 이펙트 (선택)
  enemy_die_fx.png      ← 적 사망 이펙트 (선택)
```

이 스프라이트는 구현 단계에서 프리팹으로 만들어 공격 명중 시 생성(`Instantiate`)하고, 수명이 지나면 자동 삭제(`Destroy`)한다.
