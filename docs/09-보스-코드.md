## 이 파일에서 사용하는 핵심 개념

### FSM(Finite State Machine, 유한 상태 기계)

보스는 한 번에 하나의 **상태(State)**에만 있습니다.

```
Idle → PickAttack → AttackA (방사 탄막)
                  → AttackB (강타)
                  → AttackC (돌진)
     → Idle → PickAttack → ...
```

각 상태는 코루틴으로 구현합니다.  
코루틴은 "해당 공격 동작이 끝날 때까지 기다렸다가" 다음 상태로 넘어가는 방식으로 작성합니다.  
상태 이름은 문자열로 관리되며, `BossData` SO에 설정된 공격 목록에서 가중치 랜덤으로 선택합니다.

### UnityEvent

`UnityEvent`는 Inspector에서 시청자를 연결할 수 있는 이벤트입니다.  
`BossHpBarUI`를 Inspector에서 `onHpChanged`에 연결하면 코드 수정 없이 UI가 HP 변화를 받습니다.

## BossData — 보스 설정 에셋

### 개요

- **역할**: 보스 하나의 **maxHp·contactDamage·탄막 프리팹·페이즈별 공격 슬롯(이름·가중치·쿨)**을 ScriptableObject로 담는다. BossHealth·BossAI2D가 이 에셋을 참조한다.
- **부착**: 에셋. Project에서 Create → Game → Boss Data.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **BossPhaseSettings[]** | HP 비율에 따라 다른 공격 세트·이동 속도·탄막 개수를 데이터로 넣으면, 코드 수정 없이 난이도·패턴 조정이 가능하다. |
| **BossAttackSlot.stateName** | FSM 상태 이름(문자열)과 매칭해 코루틴에서 switch(stateName)으로 분기한다. |

### SerializeField를 쓴 이유

BossData·BossPhaseSettings·BossAttackSlot의 필드는 에셋/Inspector 편집용이므로 public 또는 SerializeField.

### 사용하는 Unity API (BossData)

| API | 역할 |
|-----|------|
| `[System.Serializable]` | C# 구조체를 Inspector에서 편집 가능하게 만듦 |
| `[Range(min, max)]` | Inspector에서 슬라이더로 입력 범위 제한 |
| `[CreateAssetMenu]` | Project 우클릭으로 에셋 생성 |
| `LayerMask` | 특정 레이어만 대상으로 하는 마스크 타입 |

```csharp
using UnityEngine;

[System.Serializable]
public class BossAttackSlot
{
    [Tooltip("FSM 상태 이름과 동일하게 입력 (예: AttackA, AttackB, Move)")]
    public string stateName = "AttackA";
    [Tooltip("이 공격이 랜덤 선택될 확률 비중")]
    public float weight = 1f;
    [Tooltip("이 공격 끝난 뒤 Idle 대기 시간(초)")]
    public float cooldownAfter = 2f;
}

[System.Serializable]
public class BossPhaseSettings
{
    [Tooltip("이 페이즈는 HP 비율이 hpBelow 이하일 때 적용됩니다. 1.0 = 항상 적용(1페이즈)")]
    [Range(0f, 1f)]
    public float hpBelow = 1f;
    public BossAttackSlot[] attacks;
    public float moveSpeed = 2f;
    public int projectileCount = 6;
    public float projectileSpeed = 5f;
}

[CreateAssetMenu(fileName = "BossData", menuName = "Game/Boss Data")]
public class BossData : ScriptableObject
{
    public string bossId = "Boss_Final";
    public int maxHp = 200;
    public int contactDamage = 15;
    public GameObject projectilePrefab;
    public LayerMask playerLayer;

    [Tooltip("인덱스 0 = 1페이즈(hpBelow 가장 큰 것), 마지막 = 3페이즈")]
    public BossPhaseSettings[] phases;
}
```

## BossHealth — 체력·페이즈·사망 이벤트

### 개요

- **역할**: 보스의 **현재 HP**, **TakeDamage** 호출 시 감소·페이즈 갱신·**onHpChanged**/onPhaseChanged/onDeath 발행. BossHpBarUI는 onHpChanged를 구독해 fillAmount를 갱신한다.
- **부착**: 보스 오브젝트.
- **흐름**: TakeDamage → HP 감소 → RefreshPhaseIndex → 이벤트 발행 → HP 0이면 onDeath.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **UnityEvent&lt;int&gt;** | 보스 HP 바는 보스 씬에만 있고, GameEvents에 보스 전용 이벤트를 늘리기보다는 BossHealth에서 **Inspector로 연결 가능한** 이벤트를 두면 보스 씬만 독립적으로 테스트하기 쉽다. |
| **페이즈 = hpBelow 구간** | HP 비율이 특정 값 이하가 되면 페이즈 인덱스 증가. phases 배열을 hpBelow 내림차순으로 두면 한 번의 비율 비교로 현재 페이즈를 구할 수 있다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `data` | **씬에서 연결**. 이 보스용 BossData 에셋을 Inspector에서 지정한다. |

### 사용하는 Unity API (BossHealth)

| API | 역할 |
|-----|------|
| `UnityEvent<T>` | Inspector에서 연결하거나 코드로 구독하는 이벤트. `?.Invoke(value)` 로 발행 |
| `UnityEvent<T>.AddListener` | 코드에서 이벤트 수신자 등록 |
| `Mathf.Max(a, b)` | 두 값 중 큰 값 반환 (HP가 음수가 되지 않도록) |

### 페이즈 전환 로직

HP 비율이 페이즈 임계값 이하가 되면 `_phaseIndex`가 증가합니다.  
`phases` 배열은 `hpBelow`가 높은 것(1페이즈)에서 낮은 것(3페이즈) 순으로 정렬합니다.

```csharp
using UnityEngine;
using UnityEngine.Events;

public class BossHealth : MonoBehaviour
{
    [SerializeField] BossData data;

    private int _hp;
    public int CurrentHp => _hp;
    public int MaxHp => data != null ? data.maxHp : 100;

    // Inspector에서 연결하거나 코드로 구독
    public UnityEvent<int> onHpChanged;        // 현재 HP 전달
    public UnityEvent<int> onPhaseChanged;     // 페이즈 번호 전달 (1, 2, 3)
    public UnityEvent onDeath;

    private int _phaseIndex;

    void Start()
    {
        _hp = MaxHp;
        _phaseIndex = 0;
        onHpChanged?.Invoke(_hp);
    }

    public void TakeDamage(int dmg)
    {
        if (_hp <= 0) return;
        _hp = Mathf.Max(0, _hp - dmg);
        onHpChanged?.Invoke(_hp);

        int prevPhase = _phaseIndex;
        RefreshPhaseIndex();
        if (_phaseIndex != prevPhase)
            onPhaseChanged?.Invoke(_phaseIndex + 1); // 1-based

        if (_hp <= 0)
            onDeath?.Invoke();
    }

    private void RefreshPhaseIndex()
    {
        if (data == null || data.phases == null || data.phases.Length == 0) return;
        float ratio = (float)_hp / Mathf.Max(1, MaxHp);
        // phases는 hpBelow 내림차순 (1.0 → 0.66 → 0.33)
        // ratio가 만족하는 가장 낮은 인덱스(가장 강한 페이즈) 선택
        _phaseIndex = 0;
        for (int i = 0; i < data.phases.Length; i++)
            if (ratio <= data.phases[i].hpBelow) _phaseIndex = i;
    }

    public BossPhaseSettings GetCurrentPhase()
    {
        if (data == null || data.phases == null || data.phases.Length == 0) return null;
        RefreshPhaseIndex();
        return data.phases[_phaseIndex];
    }
}
```

## BossProjectile — 탄막 투사체

### 개요

- **역할**: 생성 시 **Init(방향, playerLayer, 속도)**로 초기화하고, Update에서 직진, **Player** 레이어와 Trigger 충돌 시 데미지 후 Destroy. lifeTime 후 자동 Destroy로 화면 밖 탄막이 씬에 남지 않게 한다.
- **부착**: 탄막 프리팹. BossAI2D가 Instantiate 후 Init 호출.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **Init()** | 탄막은 방향·속도가 공격마다 다르다. 프리팹에 고정값을 넣지 않고, 생성한 쪽에서 인자로 넘기면 하나의 프리팹으로 여러 패턴을 쓸 수 있다. |
| **레이어 비트 연산** | `(1 << other.gameObject.layer) & _playerLayer`로 "이 오브젝트가 Player 레이어인가"를 판별한다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `damage` | **기획/밸런스 수치**. |
| `lifeTime` | **튜닝**. 화면 밖으로 나간 탄막이 무한히 남지 않도록 초 단위로 제거 시간. |

### 설계 개념 (참고)

투사체는 생성 시 방향과 속도를 외부에서 받아 `Init()`으로 초기화합니다.  
`Update`에서 매 프레임 직진하고, 플레이어 레이어에 닿으면 데미지를 주고 삭제됩니다.  
`lifeTime` 후 자동 삭제로 화면 밖으로 나간 탄막이 씬에 남지 않습니다.

### 사용하는 Unity API

| API | 역할 |
|-----|------|
| `Transform.position` | 매 프레임 직접 이동 (Rigidbody2D Kinematic이므로 물리 적용 안 됨) |
| `Time.deltaTime` | 프레임 간 시간 간격. 곱하면 프레임 속도에 무관한 이동 속도가 됨 |
| `OnTriggerEnter2D` | Is Trigger 콜라이더에 충돌할 때 호출 |
| `(1 << layer) & layerMask` | 비트 연산으로 레이어 마스크 확인 |

```csharp
using UnityEngine;

public class BossProjectile : MonoBehaviour
{
    [SerializeField] int damage = 10;
    [SerializeField] float lifeTime = 5f;

    private Vector2 _dir;
    private float _speed;
    private LayerMask _playerLayer;

    public void Init(Vector2 direction, LayerMask playerLayer, float speed)
    {
        _dir = direction.normalized;
        _playerLayer = playerLayer;
        _speed = speed;
        Destroy(gameObject, lifeTime);
    }

    void Update()
    {
        transform.position += (Vector3)(_dir * _speed * Time.deltaTime);
    }

    void OnTriggerEnter2D(Collider2D other)
    {
        if (((1 << other.gameObject.layer) & _playerLayer) == 0) return;
        var ph = other.GetComponent<PlayerHealth>();
        if (ph != null) ph.TakeDamage(damage);
        Destroy(gameObject);
    }
}
```

**프리팹 설정**: 스프라이트 + `CircleCollider2D` (Is Trigger) + `Rigidbody2D` (Kinematic) + `BossProjectile`.

## BossSlamHitbox — 강타 충격파 판정

### 개요

- **역할**: 강타 공격 **동작 중에만** Collider를 켜서 그 순간에만 플레이어에게 데미지를 준다. 평소에는 enabled = false로 두어 보스가 이동할 때 부당하게 피격되지 않게 한다.
- **부착**: 보스 자식 SlamHitbox(BoxCollider2D Is Trigger + 본 스크립트).

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **평소 비활성** | 항상 켜두면 보스가 걸어가기만 해도 플레이어가 맞는다. BossAI2D의 AttackSlam 코루틴에서 짧은 시간만 enabled = true로 둔다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `damage` | **기획/밸런스 수치**. |
| `playerLayer` | **프로젝트 설정**. Player 레이어만 피격 대상으로 한다. |

### 사용하는 Unity API (BossSlamHitbox)

| API | 역할 |
|-----|------|
| `OnTriggerEnter2D` | Trigger 콜라이더에 충돌 시 호출 |
| `Collider2D.enabled` | false로 끄면 충돌 판정 없음. 강타 모션 중에만 켜서 판정 |

강타 공격 시에만 Collider를 활성화하는 이유: 항상 켜두면 보스가 이동할 때도 충격파 판정이 남아 플레이어가 부당하게 피격됩니다.

```csharp
using UnityEngine;

public class BossSlamHitbox : MonoBehaviour
{
    [SerializeField] int damage = 25;
    [SerializeField] LayerMask playerLayer;

    void OnTriggerEnter2D(Collider2D other)
    {
        if (((1 << other.gameObject.layer) & playerLayer) == 0) return;
        var ph = other.GetComponent<PlayerHealth>();
        if (ph != null) ph.TakeDamage(damage);
    }
}
```

**배치**: 보스 오브젝트 자식으로 `SlamHitbox` 빈 오브젝트를 만들고, `BoxCollider2D` (Is Trigger) + `BossSlamHitbox` 부착. 기본 상태는 Collider `enabled = false`.

## BossAI2D — FSM 보스 AI

### 개요

- **역할**: **MainLoop** 코루틴으로 무한 루프하며, 현재 페이즈에서 가중치 랜덤으로 공격을 고른 뒤 **RunState(상태 이름)**로 AttackA/B/C·Move 등을 실행한다. onDeath 시 코루틴 중단·StageCleared 발행.
- **부착**: 보스 오브젝트. RequireComponent(BossHealth).

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **FSM + 코루틴** | "공격 A 실행 → 끝날 때까지 대기 → 쿨 → 다음 공격 선택"처럼 **시간 순서가 있는** 동작을 코루틴으로 쓰면 yield로 흐름이 읽기 쉽다. Update에서 상태 변수로만 하면 복잡해진다. |
| **가중치 랜덤** | weight가 큰 공격이 더 자주 선택되도록 해, 난이도·패턴 조절을 데이터로 할 수 있게 한다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `data` | **씬에서 연결**. BossData 에셋. |
| `player` | **씬에서 연결**. 추적·돌진 방향용. 비면 Tag "Player"로 찾음. |
| `firePoints` | **씬에서 연결**. 탄막 발사 위치. 비면 보스 위치에서 발사. |

### 사용하는 Unity API (BossAI2D)

| API | 역할 |
|-----|------|
| `StartCoroutine(IEnumerator)` | 코루틴 시작 |
| `StopAllCoroutines()` | 실행 중인 모든 코루틴 중단 (사망 시 AI 루프 정지) |
| `yield return new WaitForSeconds(t)` | t초 대기 후 다음 줄 실행 |
| `yield return StartCoroutine(...)` | 내부 코루틴이 완료될 때까지 기다림 |
| `Vector2.MoveTowards` | 현재 위치에서 목표 방향으로 일정 속도로 이동 |
| `Quaternion.Euler` | 각도를 회전값(Quaternion)으로 변환 (탄막 발사 방향 계산) |
| `[RequireComponent(typeof(T))]` | 이 스크립트를 붙이려면 T 컴포넌트가 반드시 있어야 함을 강제 |

### 가중치 랜덤 선택 원리

공격마다 `weight`(가중치)를 합산하고, 0~총합 사이 랜덤 값을 뽑아 해당 슬롯을 선택합니다.

```
AttackA(weight=3), AttackB(weight=1), AttackC(weight=2) → 총합 6
랜덤 값 0~3 → AttackA, 3~4 → AttackB, 4~6 → AttackC
```

가중치가 높은 공격이 더 자주 선택됩니다.

가장 핵심인 보스 AI입니다.  
매 루프마다 현재 페이즈에서 공격을 가중치 랜덤으로 선택하고 실행합니다.

```csharp
using System.Collections;
using UnityEngine;

[RequireComponent(typeof(BossHealth))]
public class BossAI2D : MonoBehaviour
{
    [SerializeField] BossData data;
    [SerializeField] Transform player;
    [SerializeField] Transform[] firePoints; // 비우면 보스 위치에서 발사

    private BossHealth _health;

    void Awake()
    {
        _health = GetComponent<BossHealth>();

        // Player 태그로 자동 검색 (Inspector에서 직접 할당이 더 안전)
        if (player == null)
        {
            var go = GameObject.FindGameObjectWithTag("Player");
            if (go != null) player = go.transform;
        }
    }

    void Start()
    {
        _health.onDeath.AddListener(OnBossDead);
        StartCoroutine(MainLoop());
    }

    void OnBossDead()
    {
        StopAllCoroutines();
        // 보스 사망 → 스테이지 클리어 이벤트 발행
        GameEvents.RaiseStageCleared();
        // 사망 연출 후 다음 씬 이동은 GameManager 또는 별도 ClearHandler에서 처리
    }

    IEnumerator MainLoop()
    {
        yield return new WaitForSeconds(0.5f); // 등장 연출용 짧은 대기

        while (_health.CurrentHp > 0)
        {
            BossPhaseSettings phase = _health.GetCurrentPhase();
            if (phase == null || phase.attacks == null || phase.attacks.Length == 0)
            {
                yield return new WaitForSeconds(1f);
                continue;
            }

            BossAttackSlot slot = PickAttack(phase.attacks);
            yield return RunState(slot.stateName, phase);
            yield return new WaitForSeconds(slot.cooldownAfter);
        }
    }

    // 가중치 기반 랜덤 선택
    BossAttackSlot PickAttack(BossAttackSlot[] slots)
    {
        float sum = 0f;
        foreach (var s in slots) sum += s.weight;

        float r = Random.Range(0f, sum);
        foreach (var s in slots)
        {
            r -= s.weight;
            if (r <= 0f) return s;
        }
        return slots[0];
    }

    IEnumerator RunState(string stateName, BossPhaseSettings phase)
    {
        switch (stateName)
        {
            case "AttackA": yield return AttackRadialBurst(phase); break;
            case "AttackB": yield return AttackSlam(phase); break;
            case "AttackC": yield return AttackChargeDash(phase); break;
            case "Move":    yield return MoveTowardPlayer(phase.moveSpeed, 1.5f); break;
            default:        yield return new WaitForSeconds(0.8f); break;
        }
    }

    // AttackA: N방향 원형 탄막
    IEnumerator AttackRadialBurst(BossPhaseSettings phase)
    {
        int n = phase.projectileCount;
        for (int i = 0; i < n; i++)
        {
            float angle = (360f / n) * i * Mathf.Deg2Rad;
            Vector2 dir = new Vector2(Mathf.Cos(angle), Mathf.Sin(angle));
            SpawnProjectile(dir, phase.projectileSpeed);
        }
        yield return new WaitForSeconds(0.5f);
    }

    void SpawnProjectile(Vector2 dir, float speed)
    {
        if (data.projectilePrefab == null) return;
        Vector3 spawnPos = (firePoints != null && firePoints.Length > 0)
            ? firePoints[Random.Range(0, firePoints.Length)].position
            : transform.position;
        var go = Instantiate(data.projectilePrefab, spawnPos, Quaternion.identity);
        var proj = go.GetComponent<BossProjectile>();
        if (proj != null) proj.Init(dir, data.playerLayer, speed);
    }

    // AttackB: 지면 강타 (SlamHitbox 활성화)
    IEnumerator AttackSlam(BossPhaseSettings phase)
    {
        // 준비 대기
        yield return new WaitForSeconds(0.3f);

        var hitbox = transform.Find("SlamHitbox")?.GetComponent<Collider2D>();
        if (hitbox != null) hitbox.enabled = true;

        yield return new WaitForSeconds(0.4f);

        if (hitbox != null) hitbox.enabled = false;
    }

    // AttackC: 돌진 (3페이즈)
    IEnumerator AttackChargeDash(BossPhaseSettings phase)
    {
        if (player == null) yield break;
        float dir = Mathf.Sign(player.position.x - transform.position.x);
        float elapsed = 0f;
        while (elapsed < 0.6f)
        {
            transform.position += Vector3.right * dir * phase.moveSpeed * 3f * Time.deltaTime;
            elapsed += Time.deltaTime;
            yield return null;
        }
    }

    // Move: 플레이어 방향으로 이동
    IEnumerator MoveTowardPlayer(float speed, float duration)
    {
        float elapsed = 0f;
        while (elapsed < duration && player != null)
        {
            float dir = Mathf.Sign(player.position.x - transform.position.x);
            transform.position += Vector3.right * dir * speed * Time.deltaTime;
            elapsed += Time.deltaTime;
            yield return null;
        }
    }
}
```

## BossHpBarUI — 보스 HP 바

### 개요

- **역할**: **BossHealth.onHpChanged**를 구독해 HP가 바뀔 때마다 fillAmount를 갱신한다. GameEvents가 아니라 **UnityEvent**를 쓰므로 보스 씬에서만 BossHealth와 Inspector로 연결하면 된다.
- **부착**: Canvas 아래 보스 HP 바용 Image(Filled)가 붙은 오브젝트.

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `fillImage` | **씬에서 연결**. Filled 타입 Image. |
| `bossHealth` | **씬에서 연결**. 보스 오브젝트의 BossHealth. |

### 코드 (BossHpBarUI.cs)

보스 HP 바를 Canvas에 만들고 아래 스크립트로 연결한다.

```csharp
using UnityEngine;
using UnityEngine.UI;

public class BossHpBarUI : MonoBehaviour
{
    [SerializeField] Image fillImage;
    [SerializeField] BossHealth bossHealth;

    void OnEnable()
    {
        if (bossHealth != null)
            bossHealth.onHpChanged.AddListener(OnHpChanged);
    }

    void OnDisable()
    {
        if (bossHealth != null)
            bossHealth.onHpChanged.RemoveListener(OnHpChanged);
    }

    void OnHpChanged(int currentHp)
    {
        if (fillImage != null && bossHealth != null)
            fillImage.fillAmount = (float)currentHp / bossHealth.MaxHp;
    }
}
```

**배치**: Canvas → HP Bar Image (Image Type: Filled) → `BossHpBarUI.cs` 부착 → Inspector에서 `BossHealth` 슬롯에 보스 오브젝트 연결.

## 단계별 실습 순서

1. `Stage_Boss` 씬에 바닥 타일맵 배치.
2. 빈 오브젝트 `Boss_CurseOrb` 생성.
   - `SpriteRenderer` (보스 스프라이트)
   - `BoxCollider2D`
   - `Rigidbody2D` → Body Type: Static (AI가 transform 직접 이동)
   - `BossHealth` → `BossData_Final` 에셋 연결
   - `BossAI2D` → 동일 `BossData_Final`, Player 슬롯에 Player 오브젝트 연결
   - `BossBodyContact` (선택): 보스에 닿으면 데미지
3. `Boss_CurseOrb` 자식에 빈 오브젝트 `SlamHitbox`:
   - `BoxCollider2D` → Is Trigger, 기본 enabled false
   - `BossSlamHitbox` → Layer Mask에 Player 레이어 설정
4. Canvas에 보스 HP 바 추가 → `BossHpBarUI` 연결.
5. `BossData.onDeath` Inspector 이벤트 → 없으면 `BossAI2D.OnBossDead`에서 `GameEvents.RaiseStageCleared()` 호출.
