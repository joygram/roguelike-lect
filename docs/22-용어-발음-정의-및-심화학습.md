# 용어·영어 발음·정의 및 어려운 개념 심화 학습

문서에서 나오는 **영어 용어**에 **한글 표기**, **영어 발음**(한글로 적은 읽는 법), **정의**를 모아 두었습니다.  
**어려운 개념**은 별도 **심화 학습** 섹션에서 자세히 설명합니다.

💡 **독려**: 용어가 많아서 복잡해 보여도, **한 번에 다 외울 필요는 없어요**. 필요할 때 이 표만 찾아 보면 됩니다. "ScriptableObject가 뭐지?" → 표에서 찾고, 더 궁금하면 심화 학습으로. 즐겁게 상상한 게임을 만드는 데 쓰는 **도구 이름**이라고 생각하면 됩니다.

## 1. 게임·기획 관련 용어

| 영어 | 한글 표기 | 발음 (한글로 읽는 법) | 정의 |
|------|-----------|------------------------|------|
| **Roguelike** | 로그라이크 | 로우그라이크 | 매번 맵이 바뀌고, 죽으면 처음부터 다시 시작하는 식의 게임 장르. |
| **HP** | 에이치피 / 체력 | 에이치 피 | Hit Point. 캐릭터의 생명력(체력). 0이 되면 죽음. |
| **BGM** | 비지엠 | 비 지 엠 (Background Music) | 배경 음악. 게임 중 계속 흐르는 음악. |
| **SFX** | 에스에프엑스 | 에스 에프 엑스 (Sound Effects) | 효과음. 공격·피격·획득 등 짧은 소리. |
| **GDD** | 지디디 | 지 디 디 (Game Design Document) | 게임 기획서. 게임이 뭔지, 뭘 만드는지 적어 둔 문서. |
| **UI** | 유아이 | 유 아이 (User Interface) | 사용자 인터페이스. 화면에 보이는 버튼·글자·바 등. |
| **HUD** | 허드 | 허드 (Heads-Up Display) | 화면에 계속 보이는 정보(HP 바, 점수 등). |
| **Phase** | 페이즈 | 페이즈 | 단계. 개발을 나눈 단위(Phase 0, 1, 2…). |
| **MVP** | 엠비피 | 엠 비 피 (Minimum Viable Product) | 최소한으로 동작하는 제품. 꼭 필요한 기능만 넣은 버전. |

## 2. Unity·엔진 관련 용어

| 영어 | 한글 표기 | 발음 (한글로 읽는 법) | 정의 |
|------|-----------|------------------------|------|
| **Unity** | 유니티 | 유니티 | 게임을 만들 때 쓰는 엔진(도구). 우리가 쓰는 프로그램. |
| **Asset** | 에셋 | 앳셋 | 프로젝트 안의 자원. 이미지, 소리, 스크립트, 프리팹 등. |
| **Prefab** | 프리팹 | 프리팹 | 미리 만들어 둔 오브젝트 뭉치. 복사해서 씬에 여러 번 넣을 수 있음. |
| **Scene** | 씬 | 씬 | 장면 하나. 예: 게임 플레이 씬, 게임 오버 씬. |
| **Inspector** | 인스펙터 | 인스펙터 | Unity 에디터에서 오브젝트를 고르면 오른쪽에 보이는 속성 창. |
| **GameObject** | 게임오브젝트 | 게임 오브젝트 | 씬에 있는 “무언가” 하나. 플레이어, 벽, 이펙트 모두 GameObject. |
| **Transform** | 트랜스폼 | 트랜스폼 | 오브젝트의 위치·회전·크기. `transform.position` 등. |
| **Component** | 컴포넌트 | 컴포넌넌트 | 오브젝트에 “붙는” 부품. 스크립트, Collider, Rigidbody 등. |
| **Instantiate** | 인스턴시에이트 | 인스턴시에이트 | 프리팹을 **복사해서** 씬에 하나 만드는 것. |
| **Destroy** | 디스트로이 | 디스트로이 | 오브젝트를 씬에서 **없애는** 것. |
| **ScriptableObject** | 스크립터블오브젝트 | 스크립터블 앱젝트 | “데이터만 담는 에셋”의 부모. 숫자·설정을 에셋으로 저장할 때 씀. ▶ [심화 1](#심화-1-scriptableobject) |
| **MonoBehaviour** | 모노비헤이비어 | 모노 비헤이비어 | “씬에 붙는 스크립트”의 부모. Start, Update 같은 걸 쓸 수 있게 해 줌. ▶ [심화 2](#심화-2-monombehaviour-모노비헤이비어) |
| **API** | 에이피아이 | 에이 피 아이 (Application Programming Interface) | “엔진이 우리에게 제공하는 기능(이름·함수)”의 모음. Unity API = Unity가 제공하는 함수·클래스. ▶ [심화 3](#심화-3-api) |
| **Rigidbody** | 리지드바디 | 리지드 바디 | 물리 엔진으로 움직이게 하는 컴포넌트. 중력·충돌 적용. |
| **Collider** | 콜라이더 | 콜라이더 | 충돌을 감지하는 “몸통” 영역. |
| **Trigger** | 트리거 | 트리거 | “겹치기만 해도” 감지되는 영역. 통과 가능. |
| **Vector3** | 벡터쓰리 | 벡터 쓰리 | x, y, z 세 개의 숫자. 위치나 방향을 나타낼 때 씀. |
| **Quaternion** | 쿼터니언 | 쿼터니언 | 회전을 나타내는 값. `Quaternion.identity` = 회전 없음. |
| **deltaTime** | 델타타임 | 델타 타임 | “지난 프레임에 걸린 시간”(초). 부드러운 이동에 씀. ▶ [심화 4](#심화-4-deltatime과-프레임) |
| **Cooldown** | 쿨다운 | 쿨다운 | 스킬·공격을 한 번 쓴 뒤, 다시 쓸 수 있을 때까지의 대기 시간. ▶ [심화 5](#심화-5-쿨다운-cooldown) |

## 4. 어려운 개념 — 심화 학습

아래는 처음 보면 헷갈릴 수 있는 개념을 **정의**와 **추가 설명**으로 정리한 것입니다.

---

### 심화 1: ScriptableObject (스크립터블 오브젝트)

**정의**  
**ScriptableObject**는 “씬에 안 붙고, **데이터만** 에셋 파일로 저장되는 형식”입니다.  
플레이어 HP, 몬스터 공격력, 던전 방 개수 같은 **숫자·설정**을 하나의 에셋(.asset)으로 만들 수 있습니다.

**왜 쓰나**  
- 코드를 안 바꿔도 **Unity Inspector에서 숫자만 바꿔서** 밸런스 조정 가능.  
- 같은 형식(예: EnemyStatsData)으로 **몬스터 종류별 에셋**을 여러 개 만들 수 있음.

**쉽게 기억**  
“**씬에 있는 오브젝트**가 아니라 **데이터 한 묶음**을 파일로 저장한 것”이라고 보면 됩니다.

---

### 심화 2: MonoBehaviour (모노비헤이비어) {#심화-2-monombehaviour-모노비헤이비어}

**정의**  
**MonoBehaviour**는 “**씬에 있는 오브젝트에 붙는** 스크립트의 부모 클래스”입니다.  
우리가 만드는 `PlayerController`, `PlayerHealth` 같은 스크립트는 보통 `MonoBehaviour`를 **상속**합니다.

**그래서 할 수 있는 것**  
- `Start()` — 이 오브젝트가 켜질 때 한 번 실행.  
- `Update()` — 매 프레임 한 번 실행.  
- `transform`, `gameObject` — 이 오브젝트의 위치, 자기 자신을 가리킴.

**쉽게 기억**  
“**씬에 붙어서** 매 프레임·이벤트에 반응하는 스크립트”는 MonoBehaviour를 씁니다.  
데이터만 담는 ScriptableObject는 **MonoBehaviour를 쓰지 않습니다**.

---

### 심화 3: API (에이피아이)

**정의**  
**API**(Application Programming Interface)는 “**다른 쪽(여기서는 Unity 엔진)**이 우리에게 제공하는 **기능의 이름·함수**”를 말합니다.  
우리는 그 **이름을 불러서** 사용만 하면 됩니다. 안쪽이 어떻게 돌아가는지는 몰라도 됩니다.

**예**  
- `Instantiate()`, `Destroy()` — Unity가 제공하는 API. 우리는 이름만 부르면 됨.  
- `Time.deltaTime` — Unity가 제공하는 값.  
- **Unity API** = Unity 엔진이 제공하는 이런 이름·함수·클래스 전부.

**쉽게 기억**  
“**엔진이 미리 만들어 둔 기능**을 우리가 **이름으로 호출**하는 것”이 API 사용입니다.

---

### 심화 4: deltaTime (델타 타임)과 프레임

**정의**  
- **프레임**: 화면이 한 번 갱신되는 단위. 1초에 60번 갱신되면 “60 FPS”.  
- **deltaTime**: “**방금 지나간 한 프레임**에 걸린 시간”(초).  
  - 60 FPS면 대략 1/60 ≈ 0.0167초.  
  - 컴퓨터가 느리면 한 프레임이 더 길어지니까 `deltaTime`이 커짐.

**왜 쓰나**  
이동할 때 “**초당 5미터**”처럼 만들려면:  
`위치 += 방향 * 속도 * Time.deltaTime`  
이렇게 하면 **컴퓨터가 빠르든 느리든** 비슷한 속도로 움직입니다.  
(매 프레임 “고정 거리”만큼만 움직이면, 느린 PC에서는 뚝뚝 끊겨 보임.)

**쉽게 기억**  
“**한 프레임 동안 얼마나 시간이 지났는지**”를 알려 주는 값. 이동·타이머에 곱해 주면 됩니다.

---

### 심화 5: 쿨다운 (Cooldown)

**정의**  
**쿨다운**은 “스킬·공격을 **한 번 쓴 뒤**, 다시 쓸 수 있을 때까지의 **기다리는 시간**”입니다.

**구현 흐름**  
1. “마지막으로 쓴 시간”을 변수에 저장.  
2. 스킬을 쓰려 할 때: “지금 시간 - 마지막 사용 시간”이 **쿨다운 시간보다 크거나 같으면** 사용 가능.  
3. 사용했으면 “마지막으로 쓴 시간 = 지금 시간”으로 갱신.  
- `Time.time`(게임 시작 후 흐른 초)을 쓰면 편합니다.

**쉽게 기억**  
“한 번 누르면 **몇 초 동안은 다시 못 누르게**” 하는 그 **몇 초**가 쿨다운입니다.
