# 01. 기획 합의 — 무엇을 만들지 결정하기

코드를 짜기 전에 팀이 만들 게임에 대해 합의해야 한다. 기획 없이 코딩을 시작하면 중간에 "이 기능은 넣기로 했나요?"라는 질문이 끊임없이 생기고, 작업이 뒤엉킨다.

이 단계에서 결정하는 것들은 이후 모든 단계의 기준이 된다. 수치, 키 조작, 승패 조건 등이 여기서 정해지면 코드 담당과 에셋 담당이 각자 진행할 수 있다.

---

## 전체 흐름

```
00 준비 → [01 기획] → 02 이동·점프 → 03 맵 → 04 전투 → 05 UI → 06 사운드·빌드
```

## 게임 정의

만들 게임을 한 문장으로 정의해보자. 막연하게 "로그라이크 게임"이라고 해두면 팀원마다 다르게 상상한다. 아래 항목을 채우면 같은 그림을 그릴 수 있다.

| 항목 | 작성 |
|------|------|
| 장르 | 2D 도트 액션 플랫포머 |
| 핵심 루프 | 스테이지를 이동하며 적을 처치하고, 게임 오버 시 처음부터 다시 시작한다 |
| 플랫폼 | PC (Windows) |
| 엔진 | Unity 6.3 LTS |
| 개발 기간 | *(팀이 결정)* |

"핵심 루프"는 플레이어가 반복적으로 하는 행동의 사이클이다. 이 한 문장이 게임의 모든 기능이 왜 필요한지를 설명해준다.

## 기능 범위 결정

만들 수 있는 것과 만들어야 하는 것을 구분한다. 처음부터 욕심내면 아무것도 완성하지 못한다.

아래 목록에서 팀이 개발 기간 안에 완성할 수 있는 것을 확인한다. **필수** 항목만 완성해도 플레이 가능한 게임이 나온다.

| 기능 | 설명 | 중요도 | 구현 단계 |
|------|------|:------:|----------|
| 좌우 이동 | 키보드로 캐릭터를 좌우로 움직인다 | 필수 | 02 |
| 점프·착지 | 점프 후 중력으로 떨어진다, 플랫폼에 선다 | 필수 | 02 |
| 카메라 추적 | 플레이어를 카메라가 따라간다 | 필수 | 02 |
| 플랫폼 맵 | 바닥·벽·발판이 배치된 스테이지가 있다 | 필수 | 03 |
| 적 배치 | 스테이지에 적이 등장한다 | 필수 | 04 |
| 근접 공격 | 플레이어가 적을 공격해서 처치할 수 있다 | 필수 | 04 |
| HP·피격 | 피격 시 HP가 줄어든다 | 필수 | 04 |

| 재시작 | 게임 오버 후 다시 시작할 수 있다 | 필수 | 05 |
| HP 바 | 남은 HP가 화면에 표시된다 | 필수 | 05 |
| BGM | 게임 중 배경음악이 재생된다 | 필수 | 06 |

| EXP·레벨업 | 적을 처치하면 경험치를 얻고 레벨이 오른다 | 선택 | 04 |
| 증강 선택 | 스테이지 클리어 후 능력을 선택한다 | 선택 | 이후 |
| 보스 | 특수한 적이 등장한다 | 선택 | 이후 |
| 랜덤 맵 | 매판 맵이 자동 생성된다 | 선택 | 이후 |

## 조작 방식 결정

키 설정은 나중에 바꿀 수 있지만, 지금 정해두지 않으면 코드마다 키가 다르게 들어가게 된다. 한 번 정하고 공유한다.

| 행동 | 키 |
|------|----|
| 왼쪽 이동 | `A` 또는 `←` |
| 오른쪽 이동 | `D` 또는 `→` |
| 점프 | `Space` |
| 공격 | `J` 또는 마우스 왼쪽 클릭 |
| 재시작 | 버튼 클릭 |

Unity는 입력을 `Input.GetAxisRaw("Horizontal")`처럼 이름으로 처리한다. 구체적인 키 매핑은 `Edit > Project Settings > Input Manager`에서 확인·수정할 수 있다.

## 승리·패배 조건

게임이 언제 끝나는지를 명확히 정해야 한다.

| 조건 | 내용 |
|------|------|
| 패배 | 플레이어 HP가 0이 되면 게임 오버 |
| 재시작 | 게임 오버 화면에서 버튼을 눌러 처음부터 다시 시작 |
| 승리 | *(팀 결정 — 예: 보스 처치, 특정 지점 도달, 또는 생존이 목표라 별도 승리 없음)* |

## 역할 분담

한 사람이 모든 것을 하면 병렬로 진행하기 어렵다. 각 단계의 세 파일(기획·에셋·코드)이 독립적으로 진행될 수 있도록 역할을 나눈다.

| 역할 | 주요 작업 | 주로 다루는 영역 |
|------|---------|----------------|
| 기획 | 수치 설계, 레벨 설계, 기능 명세 | 문서, `_Content/Data/` |
| 에셋 | 스프라이트 제작, 오디오 수집 | `Sprites/`, `Audio/` |
| 코드 | 스크립트 작성, 씬·프리팹 구성 | `_Framework/`, `Scripts/` |

1인 개발이라면 모든 역할을 순서대로 진행하면 된다. 각 단계의 기획 → 에셋 → 코드 순서를 따른다.

### 에셋이 아직 없을 때

에셋이 완성되기 전에도 코드 작업은 진행할 수 있다. Unity에서 `SpriteRenderer`의 스프라이트 슬롯에 임시로 흰 사각형을 넣어두고, 에셋이 완성되면 슬롯만 교체하면 된다. 코드는 바뀌지 않는다.
