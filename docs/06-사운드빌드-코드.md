## 구분하기: 내가 만드는 것 vs Unity(API)가 주는 것

| 구분 | 이번 단계 예시 | 설명 |
|------|----------------|------|
| **내가 만드는 것** | 어떤 이벤트에서 어떤 사운드를 재생할지, BGM 루프 여부, 볼륨 비율 | 사운드 설계·이벤트 매핑은 **우리 책임** |
| **Unity(API)가 주는 것** | `AudioSource`, `AudioClip`, `PlayOneShot`, `AudioSource.Play()`, `loop`, `volume` | 실제 소리 재생·믹싱은 **엔진**이 처리 |

**왜 엔진 것을 쓰나?**  
소리 파일을 직접 스트리밍하거나 믹싱하는 것은 OS·드라이버 수준의 작업이다. Unity의 `AudioSource`는 이를 컴포넌트 하나로 해결한다. 우리가 할 일은 "언제 어떤 클립을 재생할지"만 결정하는 것이다.

## 구현 개요

```
씬 (Game)
└─ AudioManager
    ├─ BGMSource   ← AudioSource (BGM 전용)
    └─ SFXSource   ← AudioSource (효과음 전용)
```

**AudioManager**를 하나의 싱글톤으로 두고, 모든 사운드 재생을 여기서 처리한다. 각 이벤트 스크립트(PlayerCombat, PlayerHealth 등)는 `AudioManager.Instance.PlaySFX(클립)`만 호출하면 된다.

**스크립트 역할 요약**

| 스크립트 | 부착 위치 | 역할 |
|----------|-----------|------|
| **AudioManager** | AudioManager 오브젝트 | BGM 재생·루프, 효과음 PlayOneShot 호출 |

## AudioManager — BGM + 효과음 통합 관리

### 개요

- **역할**: BGM 재생·정지, 효과음 `PlayOneShot` 호출을 한 곳에서 담당한다. 게임 전체에서 `AudioManager.Instance.PlaySFX(클립)`으로 효과음을 호출한다.
- **부착**: 씬 빈 오브젝트 `AudioManager`. 자식으로 `BGMSource`, `SFXSource` 빈 오브젝트를 두고 각각에 `AudioSource` 컴포넌트를 부착한다.
- **흐름**: `Awake`에서 싱글톤 설정 → `Start`에서 BGM 클립 할당·루프·재생 → 각 이벤트 스크립트에서 `PlaySFX(clip)` 호출.

### BGM과 효과음을 AudioSource 두 개로 분리하는 이유

BGM과 효과음을 같은 `AudioSource`에서 재생하면 독립 제어가 불가능하다. BGM 볼륨을 낮추면 효과음도 같이 낮아지고, `AudioSource.Play()`로 BGM을 끊으면 재생 중인 효과음도 끊긴다. 별도 `AudioSource`를 두면 BGM·SFX 볼륨·재생·정지를 각각 독립적으로 제어할 수 있다.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **BGM: AudioSource.Play()** | BGM은 하나의 클립이 계속 루프된다. `Play()`는 현재 클립을 처음부터 재생한다. `loop = true`로 설정하면 끝나면 자동으로 다시 시작한다. |
| **SFX: PlayOneShot(clip)** | `Play()`는 이전 재생을 끊고 새로 시작한다. 적을 연속 공격하면 타격음이 겹쳐 나와야 하는데 `Play()`를 쓰면 이전 타격음이 끊긴다. `PlayOneShot()`은 기존 재생을 유지하면서 새 소리를 추가로 시작한다. |
| **싱글톤** | 모든 이벤트 스크립트가 AudioManager를 찾아야 한다. 싱글톤으로 두면 `AudioManager.Instance.PlaySFX()`로 어디서든 바로 접근할 수 있다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `bgmSource` | **씬에서 연결**. BGMSource 오브젝트의 AudioSource를 Inspector에서 드래그 연결 |
| `bgmClip` | **씬에서 연결**. 재생할 BGM 오디오 파일을 드래그 연결. 파일이 바뀌어도 코드 수정 불필요 |
| `bgmVolume` | **기획/밸런스 수치**. `[Range(0,1)]`으로 슬라이더 제공. 코드 없이 볼륨 조절 |
| `sfxSource` | **씬에서 연결**. SFXSource 오브젝트의 AudioSource 연결 |
| `sfxVolume` | **기획/밸런스 수치**. 효과음 전체 볼륨 |
| `sfxAttack`, `sfxHit`, … | **씬에서 연결**. 이벤트별 오디오 파일 드래그 연결. 파일 교체 시 코드 수정 불필요 |

`[Header("BGM")]`과 `[Header("SFX")]`는 Inspector에서 필드 그룹을 구분하는 라벨이다. 기능은 없고 **가독성**을 높인다.

### 사용하는 Unity API (AudioManager)

| API | 역할 |
|-----|------|
| `AudioSource.clip` | 재생할 AudioClip 설정 |
| `AudioSource.loop` | true로 설정 시 끝나면 자동 반복 |
| `AudioSource.volume` | 0~1 볼륨 |
| `AudioSource.Play()` | clip을 처음부터 재생 (이전 재생 중단) |
| `AudioSource.PlayOneShot(clip, volume)` | 기존 재생을 유지하면서 clip을 추가 재생 |
| `[Range(0f, 1f)]` | Inspector에서 float 필드를 슬라이더로 표시 |
| `[Header("…")]` | Inspector 필드 그룹 라벨 |

### 코드 (AudioManager.cs)

**파일**: `Assets/Scripts/AudioManager.cs`. AudioManager 오브젝트에 부착.

```csharp
using UnityEngine;

public class AudioManager : MonoBehaviour
{
    public static AudioManager Instance { get; private set; }   // [우리 멤버] 싱글톤

    [Header("BGM")]
    [SerializeField] AudioSource bgmSource;
    [SerializeField] AudioClip   bgmClip;
    [SerializeField] [Range(0f, 1f)] float bgmVolume = 0.5f;

    [Header("SFX")]
    [SerializeField] AudioSource sfxSource;
    [SerializeField] [Range(0f, 1f)] float sfxVolume = 1f;

    // 이벤트별 효과음 클립 — Inspector에서 오디오 파일 드래그 연결
    public AudioClip sfxAttack;
    public AudioClip sfxHit;
    public AudioClip sfxPlayerHurt;
    public AudioClip sfxGameOver;
    public AudioClip sfxEnemyDie;
    public AudioClip sfxJump;

    void Awake()   // [Unity가 호출]
    {
        if (Instance != null) { Destroy(gameObject); return; }
        Instance = this;
    }

    void Start()   // [Unity가 호출]
    {
        if (bgmClip == null) return;
        bgmSource.clip   = bgmClip;
        bgmSource.loop   = true;
        bgmSource.volume = bgmVolume;
        bgmSource.Play();   // [Unity API]
    }

    public void PlaySFX(AudioClip clip)   // [우리 메소드] 외부에서 호출
    {
        if (clip == null) return;
        sfxSource.PlayOneShot(clip, sfxVolume);   // [Unity API] 기존 재생 유지하며 추가 재생
    }
}
```

### 씬에 설정하기

1. Hierarchy → Create Empty → 이름 `AudioManager`
2. 자식 빈 오브젝트 두 개: `BGMSource`, `SFXSource`
3. 각각에 **AudioSource** 컴포넌트 추가 (BGMSource는 **Play On Awake** 체크 해제)
4. AudioManager 오브젝트에 `AudioManager.cs` 부착
5. Inspector:
   - `bgmSource` → BGMSource 드래그
   - `sfxSource` → SFXSource 드래그
   - `bgmClip` → BGM 오디오 파일 드래그
   - SFX 슬롯들 → 각 효과음 파일 드래그

## 이벤트 스크립트에 사운드 연결

각 이벤트가 발생하는 스크립트에서 `AudioManager.Instance?.PlaySFX()`를 호출한다.

**`?.`를 쓰는 이유**: `AudioManager.Instance`가 null이면(씬에 없거나 아직 초기화 전이면) `?.`는 오류 없이 건너뛴다. 개발 중에는 씬 구성이 완전하지 않을 수 있으므로 방어적으로 처리한다.

### 공격음 — PlayerCombat.cs에 추가

```csharp
// Update()의 Fire1 입력 처리 안에 추가
AudioManager.Instance?.PlaySFX(AudioManager.Instance.sfxAttack);
```

### 점프음 — PlayerJump.cs에 추가

```csharp
// if (_isGrounded && Input.GetButtonDown("Jump")) 블록 안에 추가
AudioManager.Instance?.PlaySFX(AudioManager.Instance.sfxJump);
```

### 피격음·게임오버음 — PlayerHealth.cs에 추가

```csharp
// TakeDamage()에 추가
AudioManager.Instance?.PlaySFX(AudioManager.Instance.sfxPlayerHurt);

// HP 0 처리 블록에 추가
AudioManager.Instance?.PlaySFX(AudioManager.Instance.sfxGameOver);
```

### 타격음 — PlayerAttackHitbox.cs에 추가

```csharp
// OnTriggerEnter2D()에 추가
AudioManager.Instance?.PlaySFX(AudioManager.Instance.sfxHit);
```

### 적 사망음 — EnemyController.cs에 추가

```csharp
// TakeDamage()의 HP 0 처리에 추가
AudioManager.Instance?.PlaySFX(AudioManager.Instance.sfxEnemyDie);
```

## 빌드

### Build Settings 확인

`File → Build Settings`:

1. **씬 목록**: `MainMenu`(index 0), `Game`(index 1) 순서로 있어야 한다. 없으면 씬 파일 드래그 추가
2. **Platform**: `Windows, Mac, Linux` 선택
3. **Architecture**: `x86_64`

### Player Settings

`Player Settings`:
- **Company Name**: 팀 이름
- **Product Name**: 게임 제목
- **Default Screen Width / Height**: `1280 / 720`

### 빌드 실행

1. **Build And Run** 클릭 → 저장 폴더 선택 (예: `Builds/`)
2. `.exe` + `_Data` 폴더가 함께 있어야 실행된다. `.exe`만 복사하면 실행 안 됨

---

## 동작이 안 될 때 (순서대로 확인)

| # | 증상 / 확인 |
|---|-------------|
| 1 | **BGM이 재생 안 됨** → bgmClip 미연결, 또는 BGMSource의 Play On Awake 체크됨 (Start에서 Play하므로 체크 해제 필요) |
| 2 | **효과음이 재생 안 됨** → sfxSource 미연결, 또는 해당 sfx 클립 미연결 |
| 3 | **효과음이 겹쳐 나오지 않음** → `Play()` 대신 `PlayOneShot()` 사용 여부 확인 |
| 4 | **AudioManager.Instance가 null** → 씬에 AudioManager 오브젝트 없음 |
| 5 | **빌드 실행이 안 됨** → Build Settings에 씬 추가 여부, `_Data` 폴더가 `.exe`와 같은 폴더인지 확인 |
