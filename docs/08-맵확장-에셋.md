## 1. 타일 스프라이트 제작 이론

### 픽셀 단위 통일의 중요성

타일 크기가 통일되지 않으면 맵에 틈이 생기거나 배율이 달라 보입니다.

| 항목 | 규칙 |
|------|------|
| 타일 크기 | 모든 타일을 **같은 픽셀 크기**로 제작 (예: 32×32) |
| Pixels Per Unit | 타일 크기와 동일한 값 (32px 타일 → PPU 32) |
| 이유 | PPU가 32면 32픽셀짜리 스프라이트가 Unity에서 정확히 1유닛 크기가 됨. 타일맵 셀 크기와 일치해 빈틈 없이 채워짐 |

### 타일 종류별 규격

| 타일 | 크기 | 투명 배경 | 비고 |
|------|------|-----------|------|
| 바닥 타일 | 32×32 | 불필요 | 빈틈 없이 채워야 해서 불투명 권장 |
| 벽 타일 | 32×32 | 불필요 | 위와 동일 |
| 플랫폼 | 32×16 또는 32×32 | 아래 투명 가능 | 플레이어가 위에서만 서야 함 |
| 배경 | 자유 | 필요 | 게임 플레이에 영향 없음 |

## 2. 장애물 스프라이트 제작

### 2.1 컨베이어 벨트

**크기**: 가로 96×32 (3칸 너비) 또는 팀 단위로 통일  
**디자인 포인트**: 화살표 방향을 명확히 표현 (좌향 화살표 ◀ 또는 우향 ▶)

| 구성 | 내용 |
|------|------|
| 기본 이미지 | 벨트 패턴 (체인 또는 롤러 질감) |
| 화살표 | 벨트 위에 방향 화살표 |
| 색상 | 어두운 회색 + 노란 경고선 (공장 분위기) |

**제작 Tip**: Aseprite나 Photoshop에서 32×32 타일 3개를 이어붙이거나, 단일 32×32 한 장으로 타일처럼 반복해도 됩니다.

### 2.2 사라지는 발판

**크기**: 64×16 또는 96×16  
**상태 이미지**: 2장 필요 (일반, 깜빡임)

| 상태 | 이미지 |
|------|--------|
| 일반 (Normal) | 보통 밝기 |
| 경고 (Warning) | 빨간빛 또는 반투명 |

Sprite Animation 없이 `SpriteRenderer.color`를 조절해 경고 효과를 낼 수도 있습니다.

### 2.3 출구 오브젝트

**크기**: 32×64 (문) 또는 64×64 (계단/포털)  
**활성/비활성 2개**: 모든 적 처치 전(잠긴 문), 처치 후(열린 문)

## 3. Unity 임포트 설정

### 3.1 타일 스프라이트 (타일맵용)

1. PNG를 `Assets/Sprites/Tiles/` 폴더에 드래그.
2. Texture Type: **Sprite (2D and UI)**.
3. Pixels Per Unit: **32** (팀 통일값).
4. Mesh Type: **Full Rect**.
5. Filter Mode: **Point (no filter)** (픽셀아트의 경우).
6. **Apply**.

타일맵에 추가하려면 Tile Palette 창에 드래그합니다.

### 3.2 장애물 스프라이트

1. PNG를 `Assets/Sprites/Obstacles/` 폴더에 드래그.
2. Texture Type: **Sprite (2D and UI)**.
3. Pixels Per Unit: 32.
4. Mesh Type: **Full Rect**.
5. **Apply**.

## 4. LevelSettingsData 에셋 만들기

### 전제 조건

`LevelSettingsData.cs` 스크립트가 있어야 에셋을 만들 수 있습니다.  
08-코드 문서에서 스크립트를 먼저 만들고 돌아오세요.

### 에셋 생성

1. `Assets/Data/Levels/` 폴더 만들기.
2. 폴더 안에서 우클릭 → **Create → Roguelike → Level Settings**.
3. 이름: `Level_Stage_1_1`.

### 에셋 채우기 예시

**`Level_Stage_1_1`**

| 필드 | 값 |
|------|-----|
| Stage Id | Stage_1_1 |
| Stage Name | 공장 1층 |
| Has Conveyor | false |
| Has Falling Platform | false |
| Next Scene Name | Stage_1_2 |

**`Level_Stage_1_2`**

| 필드 | 값 |
|------|-----|
| Stage Id | Stage_1_2 |
| Stage Name | 공장 1층 - 2 |
| Has Conveyor | true |
| Has Falling Platform | false |
| Next Scene Name | Stage_1_3 |

## 5. 스테이지 씬 설정

방식 A(씬 단위 이동)를 선택했다면 각 스테이지마다 씬을 만듭니다.

1. **File → New Scene** → 이름 `Stage_1_1`로 저장.
2. 03 맵에서 만든 타일맵 구성을 복사하거나 새로 배치.
3. **File → Build Settings → Add Open Scenes**로 빌드에 포함.
4. Stage_1_2, Stage_1_3도 동일하게 생성.

> **씬 이름 규칙**: `Stage_1_1` 형태로 통일합니다. LevelSettingsData의 `nextSceneName` 필드 값과 일치해야 합니다.

## 6. 장애물 프리팹 만들기

### 컨베이어 프리팹

1. Hierarchy → **Create Empty** → 이름 `Conveyor`.
2. 자식 → `SpriteRenderer` + 컨베이어 스프라이트 할당.
3. 루트에 **BoxCollider2D** 크기를 스프라이트에 맞게 조정.
4. 루트에 **Rigidbody2D** → Body Type: **Static**.
5. `ConveyorBelt.cs` 스크립트 추가 (08-코드에서 만들 예정).
6. 프리팹으로 저장: `Assets/Prefabs/Obstacles/Conveyor.prefab`.

### 사라지는 발판 프리팹

1. Hierarchy → **Create Empty** → 이름 `FallingPlatform`.
2. 자식 → `SpriteRenderer` + 발판 스프라이트.
3. 루트에 **BoxCollider2D**.
4. 루트에 **Rigidbody2D** → Body Type: **Static**.
5. `FallingPlatform.cs` 스크립트 추가.
6. 프리팹 저장: `Assets/Prefabs/Obstacles/FallingPlatform.prefab`.

### 출구 프리팹

1. Hierarchy → **Create Empty** → 이름 `StageExit`.
2. 자식 → `SpriteRenderer` (잠긴 문 스프라이트).
3. 루트에 **BoxCollider2D** → **Is Trigger** 체크.
4. `StageExit.cs` 스크립트 추가.
5. Inspector에서 `Next Scene Name` 필드에 "Stage_1_2" 등 입력.
6. 프리팹 저장.
