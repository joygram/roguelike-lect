# Part 1. 데이터 테이블 — 가장 손쉽게 구성하는 방안

## 1.1 방안 비교 (난이도·추천)

| 방안 | 난이도 | 장점 | 단점 | 추천 |
|------|--------|------|------|:----:|
| **A. Inspector 기본값** | ★☆☆ | 코드만으로 끝, 파일 없음 | 수치 변경 시 스크립트 수정 필요 | 처음 시작용 |
| **B. ScriptableObject** | ★★☆ | Unity에서 클릭만으로 수치 수정, 에셋 재사용 | 에셋 파일 생성 필요 | **가장 추천** |
| **C. JSON/CSV** | ★★☆ | 엑셀·외부 도구로 편집 가능 | 로드 코드·경로 관리 필요 | 대량 데이터 시 |

**초등학생·팀 프로젝트**: 먼저 **B. ScriptableObject**를 쓰는 것을 권장합니다.  
코딩 없이 Unity Inspector에서 숫자만 바꿔도 밸런스 조정이 가능합니다.

😄 **한줄 조크**: "데이터 테이블"이라고 하면 어렵게 느껴지는데, 결국 **적 HP 몇으로 할까?** 같은 걸 정리해 두는 거예요. 상상한 숫자를 넣어 보면 됩니다.

## 1.2 방안 A: Inspector 기본값 (가장 단순)

스크립트에 `[SerializeField]` 또는 `public` 변수를 두고, Inspector에서만 수정합니다.  
**별도 데이터 파일 없음.**

```csharp
// [우리 구현] PlayerStats.cs — 플레이어 오브젝트에 붙이는 스크립트
using UnityEngine;   // [Unity] 네임스페이스

public class PlayerStats : MonoBehaviour   // [Unity] MonoBehaviour = 씬에 붙는 스크립트 부모
{
    [Header("체력")]   // [Unity] Inspector에 "체력" 구역 표시
    public int maxHp = 100;      // [C#] 숫자 타입
    public int currentHp = 100;

    [Header("이동·공격")]
    public float moveSpeed = 5f;
    public int attackPower = 10;
    public float attackCooldown = 0.5f;
}
```

- **[Unity]**: `MonoBehaviour`, `[Header]` — 엔진이 제공. **[우리 구현]**: `PlayerStats` 클래스와 그 안의 변수 — 우리가 작성.
- **구성 난이도**: 매우 쉬움 (변수 선언만 하면 됨)
- **데이터 테이블 역할**: “플레이어 1종”에 대한 한 세트. 적은 적(Enemy) 스크립트에 동일하게 필드 추가.

## 1.3 방안 B: ScriptableObject (가장 추천 ★)

**데이터만 담는 에셋**을 만들어 두고, 여러 오브젝트에서 참조합니다.  
Unity 메뉴로 에셋 생성 → Inspector에서 수치 수정만 하면 됩니다.

### (1) 데이터 클래스 정의 (한 번만 작성)

**[우리 구현]** — 아래 클래스들은 우리가 만드는 "데이터 형식"입니다. Unity 메뉴로 에셋을 만들 때 이 형식을 사용합니다.

```csharp
// [우리 구현] PlayerStatsData.cs — 에셋으로 저장할 플레이어 수치
using UnityEngine;

[CreateAssetMenu(fileName = "PlayerStats", menuName = "Roguelike/Player Stats")]  // [Unity] Create 메뉴에 항목 추가
public class PlayerStatsData : ScriptableObject   // [Unity] ScriptableObject = 에셋 데이터의 부모
{
    public int maxHp = 100;      // [C#] 우리가 정하는 필드
    public float moveSpeed = 5f;
    public int attackPower = 10;
    public float attackCooldown = 0.5f;
}
```

```csharp
// Assets/Scripts/Data/EnemyStatsData.cs
using UnityEngine;

[CreateAssetMenu(fileName = "EnemyStats", menuName = "Roguelike/Enemy Stats")]
public class EnemyStatsData : ScriptableObject
{
    public string enemyName = "슬라임";
    public int maxHp = 20;
    public float moveSpeed = 2f;
    public int damage = 5;
    public int scoreValue = 10;
}
```

```csharp
// Assets/Scripts/Data/DungeonSettingsData.cs
using UnityEngine;

[CreateAssetMenu(fileName = "DungeonSettings", menuName = "Roguelike/Dungeon Settings")]
public class DungeonSettingsData : ScriptableObject
{
    public int roomCountMin = 5;
    public int roomCountMax = 9;
    public int roomSizeMin = 4;
    public int roomSizeMax = 8;
    public int corridorWidth = 1;
}
```

### (2) 에셋 만드는 방법 (손쉬운 구성)

1. Project 창에서 **우클릭** → **Create** → **Platformer** 또는 **Roguelike** → **Player Stats** (또는 Enemy Stats, Level Settings)
2. 이름 지정 후 `Assets/Data/` 같은 폴더에 저장
3. 적 종류별로 **Enemy Stats**를 여러 개 만들기 (예: `SlimeStats`, `BatStats`)

### (3) 스크립트에서 사용

**[우리 구현]** `PlayerHealth`는 우리가 작성하는 스크립트. **[Unity]** `Start()`는 엔진이 정해진 타이밍에 호출합니다.

```csharp
// [우리 구현] PlayerHealth.cs — 플레이어 체력을 다루는 스크립트
public class PlayerHealth : MonoBehaviour
{
    public PlayerStatsData stats;   // [우리 구현] 우리가 만든 데이터 에셋 참조 (Inspector에서 드래그)

    void Start()   // [Unity] 씬이 시작되고 이 오브젝트가 켜질 때 한 번 호출됨
    {
        currentHp = stats.maxHp;    // [우리 구현] 우리가 정한 필드 사용
    }
}
```

- **데이터 테이블 역할**: 플레이어 1종, 적 N종, 스테이지/레벨 설정 1종을 “에셋 = 테이블 행”처럼 관리.

## 1.5 현업에서 사용하는 데이터 처리 방식 (참고)

위에서 쓴 **Inspector 기본값**, **ScriptableObject**, **JSON/CSV** 말고도, 현업에서는 아래 같은 방식을 쓰는 경우가 있습니다.  
이 문서에서는 **ScriptableObject**만 따라 하면 되고, “나중에 이런 것도 있다” 정도만 알아 두면 됩니다.

| 방식 | 설명 | 언제 쓰는지 |
|------|------|-------------|
| **ScriptableObject** | Unity 에셋으로 데이터 저장, Inspector에서 수정. 이 문서의 기준 방식. | 소규모~중규모, 기획·밸런스가 에디터에서 수정할 때 |
| **JSON / XML / CSV** | 텍스트 파일로 수치 저장, 코드로 로드. 엑셀·Google 스프레드시트로 편집 후 내보내기 가능. | 대량 데이터, 기획툴·번역툴과 연동할 때 |
| **데이터 테이블 툴** | 사내에서 “몬스터 테이블”, “스킬 테이블”을 편집하는 전용 툴을 만들고, 그 결과를 JSON/바이너리로 빌드에 넣음. | 대규모, 기획이 많고 데이터 종류가 많을 때 |
| **Addressables / 에셋번들** | 에셋을 **번들로 묶어 두었다가** 실행 중에 필요한 것만 로드. 메모리·다운로드 최적화에 사용. | 모바일·대용량 에셋, DLC·패치로 리소스만 추가할 때 |

**ScriptableObject를 쓰는 이유**: 위 방식들도 있지만, **Unity만으로** 기획·입문자가 “데이터 한 줄 추가 = 에셋 하나 생성”으로 바로 쓸 수 있고, 코드 수정 없이 Inspector에서 숫자만 바꿔도 되기 때문입니다.

데이터 처리를 **조금 더 이해**하고 싶을 때, 아래 **개념**과 **연습**을 해 보면 좋습니다.

| 개념 | 뜻 |
|------|-----|

**사용 지식·영역**: **데이터 설계**(맵 데이터 구조·그리드 정의) + **알고리즘**(그리드 채우기, 방/복도 생성, 칸 값→배치 위치 계산) + **소프트웨어 설계**(배치 로직·프리팹 역할).

## 2.1 방안 비교 (난이도·추천)

| 방안 | 난이도 | 장점 | 단점 | 추천 |
|------|--------|------|------|:----:|
| **가. 2D int 배열 + 프리팹 배치** | ★★☆ | 개념 단순, 2D에 적합 | 배치 코드 직접 작성 | **가장 추천** |
| **나. 2D Tilemap** | ★★☆ | Unity 타일 에디터, 2D에 최적 | 2D 전용, 타일 에셋 필요 | 2D 픽셀일 때 |
| **다. 미리 만든 방 프리팹 배치** | ★★★ | 연출 자유로움 | 방 종류·연결 로직 복잡 | 여유 있을 때 |

**가장 쉽게 구현하려면**: **가. 2D int 배열 + 프리팹 배치**를 권장합니다.  
“칸(그리드)”을 숫자로 나타내고, 숫자에 따라 벽/바닥 프리팹만 배치하면 됩니다.

---

## 2.2 가장 쉬운 맵: 2D 배열 + 프리팹 배치

### (1) 맵을 숫자로 표현 (데이터 테이블 역할)

한 칸을 하나의 숫자로 표현합니다.

```
0 = 빈칸(밖)  1 = 바닥  2 = 벽
```

예시 맵 (5×5):

```
2 2 2 2 2
2 1 1 1 2
2 1 1 1 2
2 1 1 1 2
2 2 2 2 2
```

- **구현 난이도**: 배열만 이해하면 됨. 랜덤 던전은 “방 영역”을 정한 뒤 해당 영역을 1·2로 채우는 방식으로 확장하면 됩니다.

### (2) 맵 데이터 구조 (C#)

**[우리 구현]** 맵의 "칸마다 숫자"를 담는 클래스와 메서드. **[C#]** `int[,]`, `new`는 C# 문법입니다.

```csharp
// [우리 구현] SimpleMapData — 맵 그리드 데이터 (가로×세로)
public class SimpleMapData
{
    public int width = 20;    // [C#] 맵 가로 칸 수
    public int height = 20;   // [C#] 맵 세로 칸 수
    public int[,] grid;       // [C#] 2차원 배열. grid[x,y] = 0(빈칸), 1(바닥), 2(벽)

    public void CreateEmpty()  // [우리 구현] 우리가 호출하는 함수
    {
        grid = new int[width, height];   // [C#] 배열 생성
    }
}
```

### (3) 맵 생성 — 단순한 “방 하나” 예시 (가장 쉬운 구현)

먼저 “사각형 방 하나”만 만들어 보는 것이 가장 쉽습니다.

```csharp
// 의사 코드 (가장 쉬운 맵 생성)
void MakeSingleRoom(SimpleMapData map, int roomX, int roomY, int roomW, int roomH)
{
    for (int x = roomX; x < roomX + roomW; x++)
    for (int y = roomY; y < roomY + roomH; y++)
    {
        bool isEdge = (x == roomX || x == roomX + roomW - 1 ||
                       y == roomY || y == roomY + roomH - 1);
        map.grid[x, y] = isEdge ? 2 : 1;  // 가장자리=벽, 나머지=바닥
    }
}
```

- **랜덤 던전**으로 확장할 때: 방 개수만 정한 뒤, 각 방의 (x, y, w, h)를 랜덤으로 잡고 위 함수로 채우고, 방과 방 사이를 복도(1로 채운 줄)로 연결하면 됩니다.  
  자세한 알고리즘은 07 문서 Phase 2 참고.

### (4) 맵 그리기 — 숫자 → 프리팹 배치 (가장 쉬운 그리기)

- **벽 프리팹**: Wall (Quad 또는 스프라이트)
- **바닥 프리팹**: Floor (Plane 또는 Quad)

한 칸의 **월드 크기**를 정합니다 (예: 1칸 = 1유닛).

```csharp
// [우리 구현] 그리드 숫자에 따라 벽/바닥 배치하는 로직
float cellSize = 1f;   // [C#] 한 칸 크기 (우리가 정한 값)

for (int x = 0; x < map.width; x++)
for (int y = 0; y < map.height; y++)
{
    int cell = map.grid[x, y];
    Vector3 pos = new Vector3(x * cellSize, 0, y * cellSize);   // [Unity] Vector3

    if (cell == 1) Instantiate(floorPrefab, pos, Quaternion.identity);   // [Unity] 프리팹 복사해 씬에 생성
    if (cell == 2) Instantiate(wallPrefab,  pos, Quaternion.identity);    // [Unity] Quaternion.identity = 회전 없음
}
```

- **[Unity]**: `Instantiate`, `Vector3`, `Quaternion.identity` — 엔진 제공. **[우리 구현]**: `map`, `cellSize`, 반복문으로 배치하는 흐름 — 우리가 작성.
- **2D**: `Vector3(x, y, 0)` 처럼 Z를 0으로 두고, 프리팹은 Sprite 또는 Quad로 합니다.

이렇게 하면 **데이터(배열) → 맵 그리기**가 같은 구조로 유지되어, “가장 쉽게” 맵을 구현할 수 있습니다.

🎮 **독려**: 0, 1, 2 숫자로 맵을 만든다고 하면 뭔가 복잡해 보이지만, **"여기는 벽, 여기는 바닥"**이라고 상상한 걸 숫자로 옮긴 거예요. 그걸 화면에 펼치면 스테이지 맵이 됩니다.

## 2.3 맵 구현 단계 요약 (가장 쉬운 순서)

| 단계 | 할 일 |
|------|--------|
| 1 | `int[,] grid`와 칸 의미 정하기 (0=빈칸, 1=바닥, 2=벽) |
| 2 | “방 하나” 채우기 (사각형 영역에 1·2 넣기) |
| 3 | Wall/Floor 프리팹 만들고, 위 이중 for문으로 배치 |
| 4 | (선택) 방 여러 개 + 복도 연결해서 랜덤 던전으로 확장 |

---

## 2.4 2D Tilemap을 쓸 때 (대안)

- **장점**: Unity 에디터에서 타일을 그려 넣기 좋고, 2D 게임에 맞음.
- **가장 쉬운 설정**:  
  1) 2D 프로젝트에서 **Tilemap** 생성,  
  2) **Tile Palette**에 바닥·벽 타일 올리고,  
  3) 랜덤 던전은 **스크립트에서 Tilemap.SetTile(position, tile)** 로 칸마다 타일만 바꿔 주는 방식으로 구현 가능.

데이터는 “칸 좌표 + 타일 종류”만 관리하면 되고, “배열 + 프리팹”과 같은 사고방식입니다.
