# 06. 사운드·빌드 — 기획

> **시스템**: 오디오·빌드 | **담당**: P3 (AudioManager 설계) · D2 (BGM/SFX 수집·정리)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

소리가 없는 게임은 완성되지 않은 느낌을 준다. BGM과 효과음은 시각 정보를 보완하고, 플레이어의 행동에 즉각적인 피드백을 제공한다. "내 공격이 맞았나?"를 눈으로만 판단하는 것과 효과음으로 확인하는 것은 게임 느낌이 완전히 다르다.

---

## 전체 흐름

```
00 준비 → 01 기획 → 02 이동·점프 → 03 맵 → 04 전투 → 05 UI → [06 사운드·빌드]
                                                                  ↑ 기획 (지금)
```

---

## BGM 계획

BGM(Background Music)은 게임 플레이 중 계속 반복되는 배경 음악이다.

| 항목 | 결정 |
|------|------|
| 분위기 | *(팀 결정 — 예: 긴장감 있는 전자음악, 레트로 8비트)* |
| 루프 여부 | 루프 (반복 재생) |
| 씬당 필요한 곡 수 | Game 씬: 1곡 필수 / MainMenu 씬: 1곡 (선택) |

---

## 효과음(SFX) 목록

| 효과음 | 재생 시점 | 중요도 |
|--------|----------|:------:|
| 공격음 | 플레이어 공격 버튼을 눌렀을 때 | 필수 |
| 타격음 | 공격이 적에게 맞았을 때 | 필수 |
| 피격음 | 플레이어가 데미지를 받았을 때 | 필수 |
| 게임 오버음 | 플레이어가 사망했을 때 | 필수 |
| 적 사망음 | 적이 제거될 때 | 권장 |
| 점프음 | 점프할 때 | 권장 |
| 버튼 클릭음 | UI 버튼 클릭 시 | 권장 |

BGM 없이 효과음만 있어도 게임 느낌은 크게 달라진다. 반대로 효과음 없이 BGM만 있으면 타격감이 부족하다.

---

## 음량 기준

나중에 코드에서 조정하게 되지만, 시작점을 정해둔다.

| 항목 | 권장 볼륨 (0~1) |
|------|:--------------:|
| BGM | 0.4~0.5 |
| 효과음 | 0.8~1.0 |

BGM은 효과음보다 조용해야 한다. BGM이 너무 크면 게임 소리가 묻힌다.

---

## 빌드 계획

이 강좌의 최종 산출물은 PC에서 실행 가능한 `.exe` 파일이다.

| 항목 | 설정 |
|------|------|
| 플랫폼 | Windows x86_64 |
| 기본 해상도 | 1280×720 |
| 전체화면 | 선택 가능 |

빌드 전에 해결해야 할 것:
- Console 창에 빨간 오류(Error) 없음
- 모든 씬이 Build Settings에 등록되어 있음
- 필수 기능이 모두 동작하는지 플레이 테스트 완료
