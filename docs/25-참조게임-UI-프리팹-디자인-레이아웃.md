# Part 1. 참조 게임 기준 디자인 방향

## 1.1 참조 게임과 적용 포인트

| 참조 | 적용 포인트 (UI·디자인) |
|------|--------------------------|
| **스컬** | 로그라이크 증강 선택 흐름, 런 중 HUD(체력·재화), 어두운 톤·간결한 UI |
| **Violent Santa** | 호러·공장 분위기, 위협감 있는 색감(빨강·검정)·경고성 UI |
| **산나비** | 비주얼 스타일 참고, 자체 제작 시 톤·구도 참조 |

→ **이미지 에셋**: 문서·참조 이미지는 **레퍼런스용**이며, 실제 에셋은 **자체 제작**을 전제로 합니다.

## 1.2 톤·컬러 (선택)

| 용도 | 권장 톤 | 비고 |
|------|---------|------|
| HUD 배경 | 반투명 검정/다크 그레이 | 게임 화면 가리지 않도록 |
| 체력 바 채우기 | 빨강/주황 | 위험 시 인지 용이 |
| 재화·증강 일반 | 밝은 강조색(골드·흰색) | 가독성 |
| 증강 등급 | 회색(일반), 초록(고급), 빨강(신화), 노랑(???) | 기획서 증강목록과 동일 |
| 버튼 | 공장·호러 테마에 맞는 단색 또는 소량 포인트 | 클릭 영역 명확히 |

# Part 2. 전체 디자인 레이아웃

## 2.1 해상도·캔버스

| 항목 | 값 | 비고 |
|------|-----|------|
| 기준 해상도 | 1920×1080 | 팀 합의 후 고정 권장 |
| Canvas Scaler | Scale With Screen Size | Reference Resolution 1920×1080 |
| Match | 0.5 | 가로·세로 비율에 따른 UI 비율 유지 |
| 안전 영역 | 화면 가장자리에서 안쪽 5~10% | 버튼·중요 텍스트가 잘리지 않도록 |

## 2.2 화면별 레이아웃 개요

- **인게임 HUD**: 좌측 상단 고정. 체력·공격력·이동속도·공격속도·재화.
- **증강 선택(3중 1택)**: 스테이지 클리어 후 전용 패널. 카드 3장 배치.
- **일시정지(ESC)**: 전체 화면 반투명 + 음량·나가기 등.
- **증강 확인(TAB)**: 팝업 또는 패널. 획득한 증강 목록.
- **게임 오버**: 전체 화면. 문구 + 재시작(및 나가기).
- **휴식 스테이지**: 상점 UI(재화 소비·증강 구매 등).

# Part 3. UI 프리팹 구성요소 (전체 목록)

아래는 **게임 제작에 필요한 UI 구성요소**를 **프리팹 단위**로 정리한 목록입니다. 각 프리팹은 씬에 배치하거나 코드에서 Instantiate/SetActive로 제어합니다.

## 3.1 HUD (인게임 상시 표시)

| 프리팹/오브젝트명 | 구성 요소 | 표시 내용 | 데이터 연동(24번 문서 키·클래스) |
|-------------------|-----------|-----------|----------------------------------|
| **HUD_Stats** | 컨테이너(빈 오브젝트) | — | — |
| └ HUD_HP_Bar | Image(배경) + Image(Filled) | 현재/최대 체력 | PlayerStatsData.currentHp, maxHp → fillAmount |
| └ HUD_HP_Text | Text (선택) | "50/50" | currentHp, maxHp |
| └ HUD_AttackPower | Text | 공격력 숫자 | PlayerStatsData.attackPower |
| └ HUD_MoveSpeed | Text | 이동속도 | PlayerStatsData.moveSpeed |
| └ HUD_AttackSpeed | Text | 공격속도(초당 1회 등) | attackCooldown 역산 또는 1/cooldown |
| └ HUD_Currency | Text | 재화(사탕 등) | 런타임 재화 변수(예: GameState.currency) |

**레이아웃**: 좌측 상단, 세로 또는 2줄로 정렬. HUD_Stats 자체를 Anchor Preset Left-Top으로 고정.

## 3.2 증강 선택 (3중 1택) — 스테이지 클리어 후

| 프리팹/오브젝트명 | 구성 요소 | 표시 내용 | 데이터 연동 |
|-------------------|-----------|-----------|-------------|
| **Panel_AugmentSelect** | 전체 패널(Image 반투명) | 배경 | — |
| └ Title | Text | "증강을 선택하세요" 등 | GameStrings.AugmentSelect_Title |
| └ Card_1, Card_2, Card_3 | AugmentCard 프리팹 각 1개 | 증강 아이콘·이름·설명·등급 색 | AugmentData (id, name, description, rarity) |
| └ Button_Confirm (선택) | Button | 선택 확정 | 선택된 AugmentData 적용 후 씬 전환 |

**AugmentCard (1장) 구성**

| 자식 요소 | 타입 | 표시/역할 | 데이터 연동 |
|-----------|------|-----------|-------------|
| Icon | Image | 증강 아이콘 스프라이트 | AugmentData.icon 또는 iconId |
| Name | Text | 증강 이름 | AugmentData.augmentName |
| Description | Text | 설명 | AugmentData.description |
| RarityIndicator | Image/Color | 등급 색(회색/초록/빨강/노랑) | AugmentData.rarity |

**레이아웃**: 화면 중앙. 카드 3장 가로 배치(좌·중·우). 카드 간격·크기는 팀 디자인에 맞게.

# Part 4. 컨텐츠 선택 디자인 (3중 1택 상세)

## 4.1 플로우

1. 스테이지 클리어 조건 충족 → **Panel_AugmentSelect** 표시(SetActive(true)).
2. **AugmentData** 3개를 랜덤 또는 풀에서 선택해 Card_1~3에 바인딩.
3. 플레이어가 카드 1장 클릭 → 선택 확정.
4. 선택한 **AugmentData**를 **GameState.collectedAugments**에 추가, 스탯 반영(ScriptableObject 또는 런타임 수치).
5. 패널 비활성화 후 다음 씬 또는 다음 스테이지로 진행.

## 4.2 증강 데이터 구조 (데이터 연동용)

코드·데이터와 연동하려면 아래 필드를 갖는 **AugmentData**(ScriptableObject 또는 테이블)를 두는 것을 권장합니다.

| 한글 | 영문 (코드에 사용) | 타입 | 비고 |
|------|---------------------|------|------|
| 증강 ID | augmentId | string | 고유 식별 |
| 증강 이름 | augmentName | string | UI 표시 |
| 설명 | description | string | UI 표시 |
| 등급 | rarity | enum (Normal/Rare/Mythic/Unknown) | 회색/초록/빨강/노랑 |
| 아이콘 | icon | Sprite 또는 iconId | UI Image |
| 적용 수치(선택) | statModifiers | 구조체 또는 JSON | 공격력+2 등 |

→ 기획서 "증강목록"이 채워지면 위 구조에 맞춰 에셋·테이블을 추가합니다.

## 4.3 UI 스트링 키 (GameStrings 보강)

24번 문서 Part 2.5·4.2와 맞추어, 아래 키를 추가해 두면 문구 변경 시 코드 수정 없이 대응할 수 있습니다.

| 한글(기획) | 영문 키 (코드에 사용) | 표시 문구 예시 |
|------------|------------------------|----------------|
| 증강 선택 제목 | AugmentSelect_Title | "증강을 선택하세요" |
| 획득한 증강 제목 | AugmentList_Title | "획득한 증강" |
| 일시정지 제목 | Pause_Title | "일시정지" |
| 나가기 버튼 | Button_Quit | "나가기" |
| 게임 오버 | GameOver_Message | "게임 오버" |
| 재시작 버튼 | Button_Restart | "재시작" |

# Part 6. 데이터 연동

구성요소가 **데이터와 연동해 동작**하려면 아래 매핑을 유지합니다.

| UI 구성요소 | 데이터 소스 | 연동 방법 |
|-------------|-------------|-----------|
| HUD 체력 바·텍스트 | PlayerStatsData (currentHp, maxHp) | 플레이어 HP 변경 시 fillAmount·Text 갱신 |
| HUD 공격력·이동·공격속도 | PlayerStatsData | 스탯 변경 시(증강 적용 포함) Text 갱신 |
| HUD 재화 | GameState.currency (또는 동일 역할 변수) | 재화 증감 시 Text 갱신 |
| 증강 카드 3장 | AugmentData 3개(랜덤 선택) | 카드별 Icon/Name/Description/Rarity 바인딩 |
| 증강 목록(TAB) | GameState.collectedAugments | 리스트/그리드에 AugmentData 나열 |
| 게임 오버·재시작·나가기 | GameStrings + 씬 매니저 | 문구 = 상수, 버튼 = 씬 로드/종료 |
| 일시정지 볼륨 | AudioManager + 저장 설정 | Slider value ↔ volume, 저장 시 로드 |
| 상점 재화·상품 | GameState.currency, ShopItemData | 가격 표시·구매 시 차감·Augment 적용 |

→ **24번 문서**의 「예시 — UI·표시 항목」, Part 2.5·Part 3와 맞춰 변수명·스트링 키를 통일하면 기획서와 코드·UI가 일치합니다.
