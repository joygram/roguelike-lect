# 03. 맵 — 에셋

> **시스템**: 환경·맵 | **에셋 담당**: D2 (기본 타일셋 · 타일맵 배치)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

맵 에셋은 스테이지를 구성하는 타일 스프라이트다. 단순히 이미지를 만드는 것이 아니라, Unity Tilemap 시스템과 맞물리는 규격을 이해하고 제작해야 씬에서 정확하게 배치된다.

---

## 전체 흐름

```
00 준비 → 01 기획 → 02 이동·점프 → [03 맵] → 04 전투 → 05 UI → 06 사운드·빌드
                                       ↑ 에셋 (지금)
```

---

## 타일 크기 이론

Tilemap은 씬을 격자(Grid)로 나누어 각 칸에 타일을 배치한다. 이 격자의 한 칸 크기를 **Cell Size**라고 하며, 기본값은 `1 unit × 1 unit`이다.

PPU = 32 기준에서 1 unit = 32px이므로, 타일 스프라이트를 **32×32 px**로 만들면 격자 한 칸에 딱 맞게 들어간다.

```
Grid Cell Size = 1 unit
PPU = 32  →  1 unit = 32 px
따라서 타일 크기 = 32×32 px
```

타일을 64×64 px로 만들면 격자 칸보다 2배 크게 표시되어 타일이 서로 겹치거나 어긋난다. **Tilemap에서는 타일 크기가 PPU와 반드시 일치해야 한다.**

---

## 타일 종류와 용도

맵에 필요한 타일 종류를 먼저 파악하고 제작한다. 역할이 다른 타일은 시각적으로도 구분되어야 플레이어가 맵을 읽을 수 있다.

| 종류 | 역할 | 디자인 방향 |
|------|------|------------|
| **바닥** | 플레이어·적이 밟는 기본 지면 | 가장 흔하게 보이므로 눈에 피로하지 않은 중립 색상 |
| **벽** | 좌우·위 이동 차단 | 바닥보다 어둡게, 무거운 느낌 |
| **플랫폼** | 아래에서 올라설 수 있는 발판 | 바닥보다 밝게, 올라설 수 있음을 암시 |
| **배경** | 충돌 없는 장식 | 가장 어둡게, 전경과 구분되도록 |

배경 타일이 전경 타일(바닥·벽·플랫폼)과 구분이 안 되면 플레이어가 어디가 밟히는지 알 수 없다. **명도 차이를 의도적으로 크게** 만드는 것이 중요하다.

---

## Tile Asset이란?

Unity Tilemap에서는 스프라이트 이미지를 직접 쓰지 않는다. 스프라이트를 **Tile Asset**(.tile 파일)으로 감싸야 Tile Palette에서 사용할 수 있다.

```
스프라이트(.png)  →  Tile Asset(.tile)  →  Tile Palette에 등록  →  씬에 배치
```

Tile Asset은 스프라이트 이미지 외에 충돌 설정, 스프라이트 오프셋 등 Tilemap에서 필요한 메타데이터를 함께 담는다. Tile Palette 창에 스프라이트를 드래그하면 Unity가 자동으로 Tile Asset을 생성해준다.

---

## 타일 스프라이트 스펙

| 항목 | 값 | 이유 |
|------|---|------|
| 파일 형식 | PNG | 투명 배경 지원, 픽셀 정확도 유지 |
| 크기 | **32×32 px** | Tilemap Cell Size(1 unit)와 일치 |
| Pixels Per Unit | 32 | 팀 규약과 동일 |
| Texture Type | Sprite (2D and UI) | |
| Filter Mode | Point (no filter) | 타일 경계가 선명해야 격자가 깔끔하게 보임 |
| Sprite Mode | Single | 타일 하나 = 이미지 파일 하나 |

---

## 파일 저장 위치

```
Assets/Sprites/Map/
  tile_floor.png        ← 바닥 타일 (필수)
  tile_wall.png         ← 벽 타일 (필수)
  tile_platform.png     ← 단방향 발판 타일 (필수)
  tile_bg.png           ← 배경 장식 타일 (선택)
```

---

## Tile Palette 만들기

1. `Window > 2D > Tile Palette` 열기
2. 상단 → `Create New Palette` → 이름: `MapPalette`
3. 저장 위치: `Assets/Sprites/Map/`
4. 타일 스프라이트들을 Tile Palette 창 안으로 드래그
5. Unity가 자동으로 `.tile` 파일을 같은 폴더에 생성한다

---

## 플레이스홀더로 진행하기

스프라이트가 없어도 Tilemap은 기본 도형으로 구성할 수 있다.

1. `Create > Sprites > Square`로 흰 사각형 생성
2. 역할별로 이름 지정 (`ph_floor`, `ph_wall`, `ph_platform`)
3. Tile Palette에 드래그해서 등록
4. `SpriteRenderer > Color`로 역할별 색상 구분 (바닥=회색, 벽=짙은 회색, 플랫폼=밝은 회색)

에셋이 완성되면 Tile Palette에서 해당 Tile Asset의 스프라이트만 교체하면, 씬에 배치된 모든 같은 타일이 한 번에 바뀐다.
