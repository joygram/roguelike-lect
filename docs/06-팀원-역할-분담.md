# 팀원별 역할 분담

**README 전체 순서**: **9단계** (05 다음). [27](27-데이터-코드-에셋-병렬개발-및-커밋.md)와 같은 단계 묶음.

팀 리더나 개인이 **직접** 보고, 팀 규모에 맞게 역할 표를 채우고, 각 역할별 할 일·산출물을 확인하는 실무용 문서입니다. 1인 개발이면 여러 역할을 겸임한 뒤, 2절 표에서 "오늘 할 일"만 골라 진행하면 됩니다.

로그라이크 프로젝트를 **팀 단위**로 진행할 때 역할을 나누는 방법과  
각 역할별 **해야 할 일·산출물**을 정리한 문서입니다.

**동시 작업·합치기**: 기획(데이터)·프로그래밍·아트가 **각자 진행해도 합쳐지게** 하려면 [27-데이터-코드-에셋-병렬개발-및-커밋.md](27-데이터-코드-에셋-병렬개발-및-커밋.md)를 담당 지정과 함께 공유하세요. (공속·이속·데미지·이펙트·물리 분리, Git 브랜치·커밋 순서.)

**역할 ↔ 폴더(02·04·README와 동일)**: 담당을 나눌 때 **어느 폴더를 주로 만질지**를 아래처럼 맞추면 충돌이 줄어듭니다.

| 담당 성격 | 한글 이름 | 주로 만지는 경로(권장) |
|-----------|-----------|-------------------------|
| 코어 프로그래밍(입력·전투 규칙·씬 전환·런 상태) | **기반** | `Assets/_Framework/` |
| 기획 연동·레벨·적·맵·SO | **컨텐츠** | `Assets/_Content/` (Data, Prefabs, Scenes) |
| UI·HUD·메뉴 | **UI** | `Assets/_UI/` |
| 아트(스프라이트·타일) | (컨텐츠가 참조) | `Assets/Sprites/` 등 → 컨텐츠 프리팹에 연결 |

기획자·디자이너 표의 산출물은 대부분 **`_Content/` 또는 Sprites**로 모이게 하면 됩니다.

## 1. 역할 구분 — 기획자 / 디자이너 / 프로그래머1 / 프로그래머2

아래는 **기획자**, **디자이너**, **프로그래머1**, **프로그래머2**가 각각 해야 할 일을 구분한 목록입니다.  
프로그래머가 1명이면 프로그래머1 업무만 수행하고, 2명이면 2.절에서 업무를 나눈 뒤 **담당명**을 지정합니다.

**디자이너는** 스크립트를 작성하지 않고, **그래픽(스프라이트·타일·UI 소스)** 제작과 Unity 에셋 준비를 담당합니다. 리소스가 준비되기 전에는 프로그래머가 **플레이스홀더로 먼저 작업한 뒤 나중에 교체**하는 방식으로 진행합니다. → 아래 **1.6** 참고.

---

### 1.6 리소스 없을 때 (플레이스홀더 → 교체)

**프로그래머는 리소스(스프라이트·타일·사운드)가 없어도 대기하지 않습니다.**  
먼저 **플레이스홀더(임시 비주얼)**로 로직·프리팹·씬을 완성한 뒤, 디자이너 에셋이 준비되면 **같은 슬롯만 교체**하면 됩니다.

| 단계 | 할 일 |
|------|--------|
| 1 | 캐릭터·맵·UI 등 **프리팹과 스크립트**를 먼저 구성. 이때 Sprite/Image에는 **임시 스프라이트**(단색 Quad, Unity 기본 2D 스프라이트, 또는 간단 PNG) 또는 **색상만** 넣어 두기. |
| 2 | 이동·전투·맵 배치·UI 연동 등 **기능과 데이터 연동**을 플레이스홀더 상태로 완성. |
| 3 | 디자이너가 최종 스프라이트·타일을 주면, **해당 프리팹/오브젝트의 Sprite·Image만 교체**. 구조와 스크립트는 그대로 둠. |

**연결**: 이 방식은 02(에셋 준비), 05(에셋 목록), 07(Phase별 작업), 11(맵 리소스) 등에서 **리소스 없을 때** 언급할 때 이 절로 연결하면 됩니다.

## 4. 협업 시 유의사항

- **공통 규칙**: 폴더 구조(`05-필요-에셋-목록.md`), 씬/프리팹 네이밍, 브랜치 전략(사용 시)을 미리 정해 두기.
- **인터페이스**: “HP 변경 시 UI 갱신”처럼 **담당자 간 연동 포인트**를 요구사항·시스템 설계 문서에 명시.
- **산출물 공유**: 스프라이트·사운드는 지정 폴더에 넣고, 데이터(수치)는 기획이 정한 포맷으로 제공.
