# 분야별 분리·통합 및 제품 출시

**분야별(기획·에셋·코딩)** 로 문서·역할이 **분리**되어 있되 **통합**되어 하나의 제품으로 완성되며,  
순서대로 진행하면 **제품 출시**까지 이어지도록 정리한 문서입니다.

---

# Part 1. 분야별 분리 구조

## 1.1 분야 구분

각 **분야**는 자기만의 참고 문서를 가지고, **해당 분야의 산출물**만 책임집니다.

| 분야 | 담당 범위 | 참고 문서 | 산출물 |
|------|-----------|------------------------|--------|
| **기획** | 게임 정의, 요구사항, 시스템·데이터·컨텐츠 설계 | 17, 03, 04, 05(일부) | GDD, 시스템 목록, 데이터·컨텐츠 테이블, 요구사항 목록 |
| **에셋 제작** | 데이터 에셋, 그래픽, 사운드, 프리팹·리소스 | 05, 08, 11(맵), 15(이펙트), 10(UI 리소스) | ScriptableObject, 스프라이트, 오디오, 프리팹 |
| **코딩(구현)** | 게임로직, UI·시스템, 씬·연결 | 04, 07, 08~16 | 스크립트, 씬, 에셋 연결·동작 구현 |

→ **분리**의 의미: 각 분야는 **자기 문서만** 보고 작업해도 되고, **자기 산출물**만 완성하면 됩니다.

---

## 1.2 분야별 진행 순서 (단계적 개발)

**분야별로** 아래 순서로 진행하면 **준비된 것을 기준**으로 다음 분야가 이어집니다.

| 순서 | 분야 | 참고 문서 | 완료 시 나오는 것 |
|:----:|------|-----------|-------------------|
| 1 | **기획** | 17 (1~5단계) | GDD, 시스템 목록, 데이터·컨텐츠 테이블, 요구사항 목록 |
| 2 | **에셋** | 05, 08, 11, 15 (기획 테이블 기준) | 데이터 에셋, 프리팹, 스프라이트·오디오(선택) |
| 3 | **코딩** | 02 → 07 (Phase 0~5), 08~16 (필요 시) | 프로젝트·씬·스크립트·에셋 연결, 동작하는 게임 |
| 4 | **통합 확인** | 18 Part 4 | 기획↔에셋↔코딩 연결 확인 완료 |
| 5 | **출시 준비** | 본 문서 Part 4 | 빌드·실행 파일·출시 체크리스트 통과 |

→ 1→2→3 순서로 분야별로 진행하고, 4에서 **통합** 확인, 5에서 **제품 출시** 가능 여부를 점검합니다.

---

# Part 2. 통합 구조 — 하나의 제품으로

## 2.1 통합이 되는 지점

**분리**된 분야의 산출물이 아래 **한 지점**에서 맞닿을 때 **하나의 제품**으로 통합됩니다.

- **기획**의 테이블·요구사항 = **에셋**의 에셋 형식·목록 = **코딩**의 참조·구현 규격  
→ 이 셋이 **같은 문서·형식**을 기준으로 하면, **연결 시 동작**하고 **한 게임**으로 실행됩니다.

자세한 연결도·인터페이스·문서 기준 개발은 **18번 문서** 를 참고하세요.

## 2.2 통합 확인 (분야별 준비 후)

- **기획** 준비 완료 → **에셋**이 기획 테이블대로 제작 → **코딩**이 기획·에셋 기준으로 구현  
→ **18 Part 4 연결 확인 체크리스트**를 실행하면, **분야별로 분리**된 작업이 **통합**되어 있는지 확인할 수 있습니다.

---

# Part 3. 단계적 개발을 통한 제품 완성 흐름

## 3.1 단계적 개발 흐름 = 분야 단계 + 통합 + 출시

아래는 **어떤 순서로 무엇을 하면 제품 출시까지 이어지는지** 정리한 흐름입니다.

| 단계 | 내용 | 참고 문서 | 결과 |
|:----:|------|-------------|------|
| **1. 기획** | GDD 작성 → 시스템 기획 → 데이터·컨텐츠 테이블 → 요구사항 정리 → 검토 확정 | 17 | 기획 산출물 완료, 개발 기준 확정 |
| **2. 에셋** | 기획 테이블·목록에 맞춰 데이터 에셋·프리팹·리소스 제작 | 05, 08, 11, 15 | 에셋 준비 완료, 코딩에서 참조 가능 |
| **3. 코딩** | 환경 설정 → Phase 0~5 단계별 구현 → 에셋 연결 | 02, 07, 08~16 | 실행 가능한 게임 (에디터·빌드) |
| **4. 통합 확인** | 기획↔에셋↔코딩 연결 체크, 동작 검증 | 18 | 통합 완료, 제품으로 인정 |
| **5. 출시 준비** | 빌드·실행 테스트·출시 체크리스트 | 본 문서 Part 4 | 제품 출시 가능 상태 |

→ 위 순서대로 진행하면, **분야별 분리**를 유지하면서 **통합**된 **제품**을 만들고, **출시 가능**까지 이어집니다.

## 3.2 제품이란 (본 문서 기준)

- **에디터에서** 플레이 버튼으로 실행 가능하고,  
- **빌드**한 실행 파일로도 실행 가능하며,  
- **기획서·요구사항**에 정의된 **필수 기능**(이동, 맵, 전투, HP, 게임오버, 재시작 등)이 동작하는 상태를 **제품**으로 봅니다.  
- **출시**는 “배포 가능한 빌드 + 출시 체크리스트 통과”로 정의합니다.

---

# Part 4. 제품 출시 가능 조건 및 체크리스트

## 4.1 출시 가능 조건

다음이 모두 충족되면 **제품 출시가 가능**한 상태입니다.

1. **기획** 산출물이 확정되었고 (17 Part 5 검토 통과), **요구사항 목록**이 있다.  
2. **에셋**이 기획 테이블·목록에 맞게 준비되었고, **코딩**에서 참조·연결되어 있다.  
3. **코딩** Phase 0~5가 **07 완료 조건**을 만족하고, **필수 요구사항**(04)이 구현되었다.  
4. **18 Part 4 연결 확인** 체크리스트가 통과되었다.  
5. **빌드**가 성공하고, **출시 체크리스트**(아래)를 통과했다.

## 4.2 제품 출시 체크리스트

**제품 출시**를 선언하기 전에 아래를 확인하세요.

| 번호 | 확인 항목 | ✓ |
|:----:|-----------|---|
| 1 | 기획 문서(GDD·테이블·요구사항)가 최종 확정되어 있다. | ☐ |
| 2 | 필수 요구사항(이동, 맵, 전투, HP, 게임오버, 재시작)이 구현되어 동작한다. | ☐ |
| 3 | 기획↔에셋↔코딩 연결 확인(18 Part 4)이 통과되었다. | ☐ |
| 4 | Unity에서 **빌드**(PC 등)가 성공한다. | ☐ |
| 5 | 빌드된 **실행 파일**을 실행했을 때, 플레이·게임오버·재시작이 정상 동작한다. | ☐ |
| 6 | (선택) 실행에 필요한 최소 사양·안내가 정리되어 있다. | ☐ |

→ 위를 모두 통과하면 **제품 출시가 가능**한 상태로 볼 수 있습니다.

## 4.3 출시 후 (보강·업데이트)

- 기획 문서를 **보강**하거나 콘텐츠를 추가할 때는 **18, 17**에 따라 변경분만 반영하고, **연결 확인**과 **출시 체크리스트**를 필요한 범위에서 다시 실행하면 됩니다.

# Part 5. 한눈에 보기

| 항목 | 내용 |
|------|------|
| **분야별 분리** | 기획 / 에셋 제작 / 코딩 — 각 분야별 참고 문서·산출물이 있음 (Part 1) |
| **통합** | 기획 문서를 기준으로 에셋·코딩이 연결되고, 18 연결 확인으로 **하나의 제품**으로 통합 (Part 2) |
| **단계적 개발 흐름** | 기획 → 에셋 → 코딩 → 통합 확인 → 출시 준비 (Part 3) |
| **제품 출시 가능** | 기획 확정 + 에셋·코딩 준비 + Phase 완료 + 연결 확인 + 빌드·실행 정상 → 출시 체크리스트 통과 (Part 4) |

위 순서대로 진행하면 **분야별로 분리**된 문서·역할이 **통합**되어 하나의 제품이 되고, **제품 출시**까지 이어지도록 구성되어 있습니다.
