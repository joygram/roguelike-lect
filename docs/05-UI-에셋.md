# 05. UI — 에셋

> **시스템**: UI 시스템 | **에셋 담당**: D2 (HP 바 · 버튼 · 슬롯 프레임 · 패널 배경)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

UI 에셋은 게임 스프라이트와 다른 규칙으로 만들어야 한다. 캐릭터 스프라이트는 고정 크기로 씬에 배치되지만, UI 이미지는 화면 해상도에 따라 늘어나거나 줄어든다. 이 차이를 이해하고 설정하지 않으면 다른 해상도에서 UI가 깨져 보인다.

## UI 스프라이트 크기 이론

게임 스프라이트는 PPU 기준으로 씬에서 물리적 크기가 결정된다. 반면 UI 이미지는 Canvas 위에서 픽셀 크기를 기준으로 배치된다.

Canvas Scaler를 `Scale With Screen Size`, Reference Resolution = `1280×720`으로 설정하면, 모든 UI 요소의 크기는 1280×720 화면 기준으로 계산된다. 다른 해상도에서는 Canvas 전체가 비례해서 확대·축소된다.

따라서 UI 이미지는 **1280×720 기준에서 보이길 원하는 픽셀 크기**로 만들면 된다.

| UI 요소 | 권장 크기 | 이유 |
|---------|:--------:|------|
| HP 바 배경 | 200×20 px | 화면 왼쪽 상단에서 인식 가능한 크기 |
| HP 바 충전 | 200×20 px | 배경과 같은 크기 (겹쳐서 사용) |
| 버튼 | 160×48 px | 클릭 영역이 충분히 커야 함 |
| 아이콘 | 32×32 px | HP 바 옆 작은 아이콘 |

## Mesh Type — Full Rect가 필요한 이유

Unity의 스프라이트 임포트 설정에 `Mesh Type`이 있다.

- **Tight**: 스프라이트의 불투명한 픽셀에 딱 맞는 최소 메쉬를 생성한다. 렌더링 최적화에 유리하지만, 이미지를 늘릴 때 예상치 못한 방식으로 잘릴 수 있다.
- **Full Rect**: 이미지 전체를 사각형 메쉬로 처리한다. Image 컴포넌트에서 가로·세로를 늘릴 때 이미지가 의도한 대로 스케일된다.

HP 바처럼 크기가 동적으로 변하는 UI 이미지는 반드시 **Full Rect**를 써야 한다.

## UI 스프라이트 스펙

| 항목 | 값 | 이유 |
|------|---|------|
| 파일 형식 | PNG | 투명 배경 지원 |
| Texture Type | Sprite (2D and UI) | UI에서 사용 가능하도록 |
| Pixels Per Unit | 100 (또는 기본값) | UI는 PPU 영향을 거의 받지 않음. 기본값 유지 |
| Filter Mode | Point (no filter) | 도트 스타일 유지 |
| Mesh Type | **Full Rect** | 크기 조절 시 이미지 깨짐 방지 |

## 폰트 — 한국어 지원

Unity 기본 폰트는 한국어 문자가 없다. 한국어 텍스트를 쓰려면 폰트를 추가해야 한다.

### 어떤 폰트를 고를까

| 폰트 | 특징 | 적합한 분위기 |
|------|------|-------------|
| **Noto Sans KR** | 가독성 높은 산세리프, 모든 한국어 지원 | 깔끔한 UI, 정보 전달 |
| **둥근모 고딕** | 픽셀 느낌의 둥근 고딕체 | 도트 게임, 레트로 느낌 |
| **배달의민족 한나체** | 굵고 강렬한 서체 | 타이틀, 강조 텍스트 |

폰트는 게임 분위기에 맞게 선택한다. 본문 텍스트에 너무 개성 강한 폰트를 쓰면 가독성이 떨어진다.

### TextMeshPro 폰트 설치 순서

TextMeshPro는 Unity 기본 Text보다 렌더링 품질이 좋고, 폰트 크기를 키워도 흐릿해지지 않는다.

1. `.ttf` 또는 `.otf` 파일을 `Assets/Fonts/`에 넣기
2. `Window > TextMeshPro > Font Asset Creator` 열기
3. **Source Font File** → 폰트 파일 드래그
4. **Character Set** → `Custom Range` → `0x0020-0x007E,0xAC00-0xD7A3`  
   (기본 ASCII + 한글 전체 범위)
5. **Generate Font Atlas** 클릭 → 완료되면 **Save** 클릭
6. 생성된 `.asset` 파일을 TextMeshProUGUI의 **Font Asset** 슬롯에 드래그

## 플레이스홀더로 진행하기

UI 이미지 없이도 Unity 기본 컴포넌트로 동작하는 UI를 만들 수 있다.

| UI 요소 | 임시 처리 |
|---------|---------|
| HP 바 배경 | Image 컴포넌트 + Color = 짙은 회색 |
| HP 바 충전 | Image + Color = 빨간색 + Image Type = Filled |
| 버튼 | 기본 Button 컴포넌트 (흰 배경 + 검은 텍스트) |
| 텍스트 | TextMeshProUGUI + 시스템 폰트 임시 사용 |

에셋이 완성되면 Image 컴포넌트의 **Source Image** 슬롯에 스프라이트를 드래그해서 교체한다.
