# 02. 이동·점프 — 기획

> **시스템**: 코어 시스템 | **담당**: P1 (수치 확정) · 팀 전체 (조작 합의)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

이동과 점프는 이 게임의 가장 기본적인 조작이다. 플레이어가 이것을 어떻게 느끼는지가 게임 전체의 첫인상이 된다. 수치 하나 차이로 "답답하다", "미끄럽다", "무겁다"는 느낌이 완전히 달라진다.

기획 단계에서는 코드에 들어갈 수치를 결정한다. 정확한 값은 게임을 실제로 플레이해보면서 조정하게 되지만, 시작점이 있어야 작업을 시작할 수 있다.

## 이동·점프에서 결정해야 하는 것

### 이동 속도

이동 속도(`moveSpeed`)는 Unity 단위/초(units per second) 기준이다. Unity에서 기본 PPU(Pixels Per Unit)를 32로 맞추면, 32픽셀짜리 캐릭터가 화면에서 1 unit 크기가 된다.

1280픽셀 화면 너비를 기준으로 계산하면:
- 화면 너비 = 1280 ÷ 32 = **약 40 units**
- `moveSpeed = 5`이면 화면을 가로지르는 데 약 8초

도트 액션 플랫포머는 보통 빠릿한 느낌을 원하므로 `5~8` 범위에서 시작한다.

### 점프력

점프력(`jumpForce`)은 Rigidbody2D에 적용하는 순간 힘의 크기다. 중력(`gravityScale`)과 함께 조율해야 실제 점프 높이가 결정된다.

- `jumpForce = 10`, `gravityScale = 3`: 빠르게 올라갔다가 빠르게 떨어지는 액션 느낌
- `jumpForce = 8`, `gravityScale = 1`: 느리게 올라가고 느리게 떨어지는 느낌

일반적으로 액션 게임은 중력을 높게(2.5~4.0) 설정한다. 그래야 "붕 떠있는" 느낌 없이 땅에 발이 빨리 닿는다.

### 수치 기획표

아래 값을 채워서 코드 담당에게 전달한다.

| 변수명 | 제안값 | 팀 결정값 | 설명 |
|--------|:------:|:---------:|------|
| `moveSpeed` | `6` | | 초당 이동 거리 (unit/s) |
| `jumpForce` | `12` | | 점프 순간 적용되는 힘 |
| `gravityScale` | `3` | | 중력 배율 (1이 기본 Unity 중력) |
| `groundCheckRadius` | `0.1` | | 땅 감지 원의 반지름 |

## 땅 감지 방식 이해

점프할 때 "지금 땅 위에 있는지"를 코드가 판단해야 한다. 그래야 공중에서 점프를 막을 수 있다.

땅 감지 방식은 두 가지다:

**방법 1: `OnCollisionStay2D`**  
플레이어가 바닥 오브젝트와 충돌 중인지 확인한다. 간단하지만 경사면 모서리에서 감지가 불안정하다.

**방법 2: `Physics2D.OverlapCircle` (권장)**  
플레이어 발 바로 아래에 작은 원을 그려서 Ground 레이어에 닿는지 확인한다. 위치·반지름을 직접 설정할 수 있어 정밀하다.

이 강좌에서는 방법 2를 사용한다. 구현 방법은 코드 문서에서 다룬다.

## 레이어 이름 합의

코드, 에셋, 씬 작업이 모두 같은 레이어 이름을 써야 한다. 지금 정해두지 않으면 나중에 "나는 Ground라고 했는데 왜 Grounded로 만들었어?"가 생긴다.

| 레이어 이름 | 사용처 |
|------------|--------|
| `Player` | 플레이어 오브젝트 |
| `Ground` | 바닥·플랫폼 (충돌 대상) |
| `Enemy` | 적 오브젝트 |
