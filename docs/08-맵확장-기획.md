## 이 단계에서 만드는 것

03 맵에서는 타일맵으로 한 개의 스테이지를 만들었습니다.  
이 단계에서는 **여러 스테이지**, **층 이동 구조**, **장애물** 등을 추가해 게임에 진행감을 만듭니다.

---

## 1. 03과 08의 차이

| 항목 | 03 기본 맵 | 08 맵 확장 |
|------|-----------|------------|
| 스테이지 수 | 1개 | 여러 개 (씬 또는 그리드 재생성) |
| 장애물 | 없음 | 컨베이어, 발판 등 |
| 다음 층 이동 | 없음 | 출구 오브젝트 → 씬 전환 |
| 스테이지 설정 | 하드코딩 | ScriptableObject로 관리 |

## 2. 스테이지 구성 설계

### 스테이지 테마와 장애물 계획

우리 게임은 아래 구간으로 구성합니다. 팀 규모에 맞게 만들 구간을 선택하세요.

| 구간 | 배경 분위기 | 새로 추가할 장애물 |
|------|-------------|-------------------|
| **1-1 ~ 1-3** | 밝은 공장 | 플랫폼 (03에서 이미 있음) |
| **2-1 ~ 2-3** | 어두운 공장 | 컨베이어 벨트 |
| **3-1 ~ 3-3** | 녹슨 공장 | 사라지는 발판 |
| **4-1 ~ 4-3** | 보스 구간 | 위 모두 + 파이프 |

> **처음 구현할 때는 1-1, 1-2, 1-3 세 개만 만들어도 충분합니다.**  
> 나머지는 09 보스, 10 증강 단계 이후에 추가합니다.

## 3. 층 이동 방식 결정

층 이동이란 "현재 스테이지가 끝나면 다음 스테이지로 이동"하는 것입니다.  
아래 두 방식 중 팀이 하나를 선택합니다.

### 방식 A: 씬(Scene) 단위 이동 (권장)

각 스테이지가 별도 씬입니다.  
`Stage_1_1`, `Stage_1_2`... 씬을 만들고, 출구 오브젝트를 밟으면 다음 씬을 로드합니다.

**장점**: 스테이지별로 타일맵·적·오브젝트를 완전히 독립 설계 가능.  
**단점**: 씬 수가 많아지면 관리가 번거로움.

### 방식 B: 같은 씬 재생성

하나의 씬에서 "출구 도달 시 DungeonGenerator를 다시 호출"해 새 맵을 만듭니다.  
로그라이크처럼 절차적으로 맵이 다르게 나올 때 적합합니다.

**장점**: 씬 수가 적고 맵이 매번 달라 재플레이 가능성이 높음.  
**단점**: 적·아이템·이벤트도 코드로 배치해야 해서 구현 난도가 있음.

> **이 문서는 방식 A(씬 단위)를 기준으로 설명합니다.**  
> 방식 B를 선택하면 08-코드에서 DungeonGenerator 재호출 부분을 참고하세요.

## 4. 장애물 종류별 기획

### 4.1 컨베이어 벨트

플레이어가 올라서면 **일정 방향으로 밀려나는** 플랫폼입니다.

| 항목 | 설계 |
|------|------|
| 방향 | 왼쪽 또는 오른쪽 (Inspector에서 설정) |
| 속도 | 2 ~ 4 유닛/초 (수치는 SO에서 관리) |
| 시각 | 화살표 방향을 그림으로 표현 |
| 판정 방식 | 플레이어가 위에 있을 때만 밀림 (OnCollisionStay2D) |

### 4.2 사라지는 발판

플레이어가 올라서면 **잠깐 후 사라지고 일정 시간 후 복원**되는 발판입니다.

| 항목 | 설계 |
|------|------|
| 대기 시간 | 밟은 후 1초 |
| 사라지는 시간 | 2 ~ 3초 |
| 복원 시간 | 사라진 후 3초 |
| 시각 | 밟으면 흔들리다 사라지는 연출 |

### 4.3 출구 (층 이동 트리거)

다음 스테이지로 이동하는 트리거입니다.

| 항목 | 설계 |
|------|------|
| 형태 | 문 또는 계단 오브젝트 |
| 발동 조건 | 방의 모든 적을 처치한 뒤 활성화 (선택) 또는 항상 활성화 |
| 이동 방식 | `GameManager.Instance.GoToScene("Stage_1_2")` |

## 5. LevelSettingsData 설계

스테이지마다 다른 설정(배경, 사용 장애물 종류)을 SO로 관리합니다.  
코드는 SO에서 값을 읽어 해당 스테이지에 맞는 오브젝트를 활성화합니다.

| 필드 | 타입 | 예시 |
|------|------|------|
| `stageId` | string | "Stage_1_1" |
| `stageName` | string | "공장 1층" |
| `hasConveyor` | bool | true/false |
| `hasFallingPlatform` | bool | true/false |
| `nextSceneName` | string | "Stage_1_2" |
