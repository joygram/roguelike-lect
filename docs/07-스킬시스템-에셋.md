## 1. 이펙트 프리팹 이론

### 왜 이펙트를 프리팹으로 만드는가

이펙트는 발생 위치에 **순간 생성**하고 일정 시간 후 **자동 삭제**하는 오브젝트입니다.  
프리팹으로 만들면 `Instantiate(effectPrefab, position, Quaternion.identity)` 한 줄로 어디서든 생성할 수 있습니다.

### 이펙트 프리팹 구조

```
[이펙트 루트 오브젝트]
├── AutoDestroy 스크립트 (lifeTime 설정)
└── [시각 요소]
    └── SpriteRenderer (또는 Particle System)
```

- **루트**: 생성 위치의 빈 오브젝트.
- **AutoDestroy**: `lifeTime`초 후 오브젝트 삭제. 쌓이지 않게 합니다.
- **시각 요소**: 스프라이트 또는 파티클.

### 필요한 이펙트 종류

| 이름 | 용도 | 색상 가이드 | 권장 크기 |
|------|------|-------------|-----------|
| `Effect_Hit_Normal` | 기본 공격 타격 | 노랑/흰색 섬광 | 32×32 |
| `Effect_Hit_Heavy` | 강타 타격 | 주황/빨강, 기본보다 크게 | 48×48 |
| `Effect_Skill_Area` | 범위 공격 발동 | 파랑/보라 원형 | 64×64 |
| `Effect_Hurt` | 피격 시 | 빨간 플래시 | 32×32 |

이미지를 직접 그릴 수 없다면 **Unity의 기본 흰색 원 스프라이트**를 색만 바꿔 임시로 쓰고, 나중에 교체합니다.

## 2. 이펙트 스프라이트 제작

### 스프라이트 규격

| 항목 | 값 | 이유 |
|------|-----|------|
| 포맷 | PNG (투명 배경) | 배경이 없어야 이펙트처럼 보임 |
| 크기 | 32×32 ~ 64×64 px | 게임 내 타일/캐릭터 크기와 균형 |
| Pixels Per Unit | 32 또는 팀 PPU 통일값 | 다른 스프라이트와 크기 기준 맞춤 |
| Mesh Type | Full Rect | 투명 영역 있는 이펙트는 Full Rect |
| Filter Mode | Point (no filter) | 픽셀아트 스타일 유지 |

### Unity 임포트 설정

1. Project에 PNG 드래그.
2. Inspector → Texture Type: **Sprite (2D and UI)**.
3. Pixels Per Unit: 팀 통일값 (예: 32).
4. Mesh Type: **Full Rect**.
5. Filter Mode: **Point (no filter)**.
6. **Apply**.

## 3. 이펙트 프리팹 만들기

### Step 1 — 루트 오브젝트 생성

1. Hierarchy → **Create Empty** → 이름: `Effect_Hit_Normal`.
2. Transform 위치 (0, 0, 0) 확인.

### Step 2 — 시각 요소 추가

1. `Effect_Hit_Normal` 우클릭 → **Create Empty** → 이름: `Visual`.
2. `Visual` 선택 → **Add Component** → `SpriteRenderer`.
3. Sprite 슬롯에 앞에서 만든 스프라이트 할당.
4. Color로 색 조정 (예: 노란색 섬광).

### Step 3 — AutoDestroy 스크립트 추가

아직 스크립트가 없다면 07-코드 문서에서 만들 예정입니다.  
지금은 **루트에 Component 자리를 비워두고** 코드 작업 후 추가합니다.

### Step 4 — 프리팹 저장

1. `Effect_Hit_Normal`을 `Assets/Prefabs/Effects/` 폴더에 드래그.
2. 파란 큐브 아이콘이 되면 프리팹 저장 완료.
3. Hierarchy에서 삭제.

> 폴더가 없으면 Project에서 `Prefabs` → `Effects` 폴더를 먼저 만드세요.

위 과정을 **이펙트 종류마다 반복**합니다.

## 4. SkillData ScriptableObject 에셋 만들기

### 전제 조건

`SkillData.cs` 스크립트가 있어야 에셋을 만들 수 있습니다.  
07-코드 문서에서 스크립트를 먼저 만들고 돌아오세요.

### SkillData 에셋 생성

1. Project에서 `Assets/Data/Skills/` 폴더 만들기.
2. 폴더 안에서 우클릭 → **Create → Roguelike → Skill Data**.  
   (메뉴 이름은 스크립트의 `[CreateAssetMenu]` 설정에 따라 다릅니다.)
3. 에셋 이름: `Skill_BasicAttack`.
