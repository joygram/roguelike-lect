## 구분하기: 내가 만드는 것 vs Unity(API)가 주는 것

| 구분 | 이번 단계 예시 | 설명 |
|------|----------------|------|
| **내가 만드는 것** | 스킬별 수치·키 매핑, 단타/범위 분기, 쿨타임 판정 규칙, 이벤트로 UI에 알리는 타이밍 | 스킬 규칙·데이터 설계는 **우리 책임** |
| **Unity(API)가 주는 것** | `Physics2D.OverlapBox`, `OverlapCircleAll`, `Time.time`, `Input.GetKeyDown`, `Destroy(gameObject, delay)` | 물리 검사·시간·입력·오브젝트 제거는 **엔진**이 제공 |

## 구현 개요

```
Player
  └─ [스크립트] PlayerCombat — skills[]·skillKeys[] 연결

HUD / SkillSlots
  ├─ SkillSlot_0  ← SkillSlotUI (slotIndex=0, SkillData 연결)
  ├─ SkillSlot_1  ← SkillSlotUI (slotIndex=1)
  └─ SkillSlot_2  ← SkillSlotUI (slotIndex=2)
```

**스크립트 역할 요약**

| 스크립트 | 부착 위치 | 역할 |
|----------|-----------|------|
| **AutoDestroy** | 이펙트 프리팹 루트 | 생성 후 일정 시간 뒤 자동 삭제 |
| **SkillData** | 에셋(ScriptableObject) | 스킬 하나의 수치·아이콘·이펙트·사운드 |
| **PlayerCombat** | Player | 키 입력 → 쿨타임 확인 → 단타/범위 실행 → GameEvents.RaiseSkillUsed |
| **SkillSlotUI** | SkillSlot_0/1/2 | SkillUsed 이벤트 구독 → 쿨타임 링(fillAmount) 갱신 |

## AutoDestroy — 이펙트 자동 삭제

### 개요

- **역할**: `Instantiate`로 생성된 오브젝트가 **지정 시간 후** 스스로 `Destroy`되게 한다. 이펙트·투사체처럼 일시적인 오브젝트에 붙인다.
- **부착**: 이펙트 프리팹의 **루트 오브젝트**. 프리팹이 씬에 생성될 때마다 Start에서 타이머가 걸린다.
- **흐름**: `Start()`에서 `Destroy(gameObject, lifetime)` 한 번 호출 → 엔진이 lifetime초 후 제거.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **Start()에서 Destroy 예약** | 생성 직후 한 번만 예약하면 된다. Update로 타이머를 돌릴 필요가 없어 코드가 단순하다. |
| **Destroy(gameObject, delay)** | Unity API가 delay 초 후 자동 제거를 처리한다. 우리가 시간을 세거나 코루틴을 쓸 필요가 없다. |

### SerializeField를 쓴 이유

| 필드 | SerializeField 쓴 이유 |
|------|-------------------------|
| `lifetime` | **튜닝**. 이펙트가 화면에 머무는 시간은 에펙트마다 다르다. Inspector에서 조절할 수 있게 둔다. |

### 사용하는 Unity API (AutoDestroy)

| API | 역할 |
|-----|------|
| `Destroy(gameObject, delay)` | `delay`초 후 이 오브젝트를 씬에서 제거 |
| `void Start()` | 오브젝트가 씬에 생성된 직후 1회 실행 |

### 코드 (AutoDestroy.cs)

```csharp
using UnityEngine;

/// <summary>
/// 프리팹 생성 후 일정 시간이 지나면 자동으로 삭제됩니다.
/// 이펙트, 투사체 등 일시적으로 사용하는 오브젝트에 붙입니다.
/// </summary>
public class AutoDestroy : MonoBehaviour
{
    [SerializeField] float lifetime = 0.3f;

    void Start()
    {
        // lifetime초 후 이 게임오브젝트를 삭제
        Destroy(gameObject, lifetime);
    }
}
```

**파일**: `Assets/Scripts/AutoDestroy.cs`. 이펙트 프리팹 루트에 부착.

## SkillData — 스킬 수치·에셋 데이터

### 개요

- **역할**: 스킬 하나의 **이름·데미지·쿨타임·사거리·범위·아이콘·이펙트 프리팹·사운드**를 에셋 파일로 담는다. 같은 `PlayerCombat` 코드에 에셋만 바꿔 끼우면 스킬 종류를 늘릴 수 있다.
- **부착**: 코드가 아닌 **ScriptableObject 에셋**. Project에서 Create → Roguelike → Skill Data로 생성.
- **흐름**: PlayerCombat이 `skills[i]`로 SkillData를 읽어 데미지·쿨·범위·이펙트를 사용한다.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **ScriptableObject** | 수치를 코드에 직접 쓰면(하드코딩) 스킬 하나 바꿀 때마다 컴파일이 필요하다. SO는 에셋만 수정하면 되고, 같은 로직으로 여러 스킬을 재사용할 수 있다. |
| **radius = 0이면 단타, 0 초과면 범위** | 한 필드로 "단타 vs 범위"를 구분해, PlayerCombat에서 `if (s.radius > 0f)` 한 줄로 분기할 수 있다. |

### SerializeField를 쓴 이유

SkillData 필드는 모두 `public`이다. ScriptableObject는 **에셋 자체가 데이터**이므로 Inspector에서 편집 가능하게 두기 위해 public이 적합하다. (SerializeField로 바꿔도 동작은 같다.)

### 사용하는 Unity API (SkillData)

| API | 역할 |
|-----|------|
| `ScriptableObject` | 씬·프리팹과 분리된 데이터 에셋. 코드를 통해 수치만 담음 |
| `[CreateAssetMenu]` | Project 우클릭 → Create 메뉴에 이 SO 생성 항목 추가 |
| `GameObject` | 이펙트 프리팹 참조 타입 |
| `AudioClip` | 사운드 에셋 참조 타입 |

### 구현

```csharp
using UnityEngine;

/// <summary>
/// 스킬 하나의 수치와 연결 에셋을 담는 데이터 에셋입니다.
/// 코드 수정 없이 에셋 값만 바꿔서 스킬 종류를 늘릴 수 있습니다.
/// </summary>
[CreateAssetMenu(fileName = "Skill", menuName = "Roguelike/Skill Data")]
public class SkillData : ScriptableObject
{
    public string skillName = "기본 공격";
    public Sprite icon;            // UI 슬롯에 표시할 아이콘
    public int damage = 10;
    public float cooldown = 0.5f;
    public float range = 1.5f;     // 단타 사거리. radius > 0이면 무시
    public float radius = 0f;      // 0 = 단타, 0 초과 = 범위 스킬
    public GameObject effectPrefab;
    public AudioClip soundClip;
}
```

**파일**: `Assets/Scripts/SkillData.cs`. 에셋은 Project 우클릭 → **Create → Roguelike → Skill Data**. 에셋 제작은 `07-스킬시스템-에셋.md` 참고.

## PlayerCombat — 다중 스킬 발동

### 개요

- **역할**: **키 입력**에 따라 해당 인덱스의 스킬을 발동한다. 쿨타임은 `Time.time`과 `_nextUseTime[index]`로 판별하고, 스킬 사용 시 **GameEvents.RaiseSkillUsed**로 UI에 알린다. 단타(OverlapBox)와 범위(OverlapCircleAll)를 SkillData.radius로 구분한다.
- **부착**: Player 오브젝트.
- **흐름**: `Update`에서 skillKeys[i] 입력 감지 → `TryUseSkill(i)` → 쿨타임 확인 → `DoSkill(s)` → 사운드·단타/범위 분기 → 데미지·이펙트.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **skills[] + skillKeys[]** | 스킬 에셋과 키를 같은 인덱스로 매핑하면, 스킬을 추가·제거해도 "i번 스킬 = i번 키"만 맞추면 된다. |
| **Time.time &lt; _nextUseTime[index]** | 쿨타임을 bool이나 별도 타이머로 두지 않고, "다음 사용 가능 시각" 하나로 관리한다. |
| **OverlapBox(단타) vs OverlapCircleAll(범위)** | 단타는 "플레이어 앞쪽 일직선 박스" 한 칸만 검사하면 되고, 범위는 원 안의 모든 적을 검사해야 하므로 API를 나눈다. |
| **GameEvents.RaiseSkillUsed** | PlayerCombat이 UI를 직접 참조하면 슬롯 개수·구성이 바뀔 때마다 코드를 수정해야 한다. 이벤트로 "i번 스킬, 쿨타임 N초"만 보내면 UI가 알아서 링을 그린다. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `skills` | **씬에서 연결**. Inspector에서 스킬 에셋을 순서대로 드래그. 코드에 스킬 종류를 박지 않는다. |
| `skillKeys` | **기획/조작**. 팀마다 키 배치가 다르므로 Inspector에서 KeyCode 배열로 지정한다. |

`_nextUseTime`은 런타임 상태이므로 SerializeField 불필요.

### 사용하는 Unity API (PlayerCombat)

| API | 역할 |
|-----|------|
| `Input.GetKeyDown(KeyCode)` | 해당 키가 이번 프레임에 눌렸는지 확인 |
| `Time.time` | 게임 시작 후 경과 시간(초). 쿨타임 계산에 사용 |
| `Physics2D.OverlapBox(center, size, angle, layerMask)` | 지정 위치의 사각형 범위 안 콜라이더 1개 반환 |
| `Physics2D.OverlapCircleAll(center, radius, layerMask)` | 지정 위치의 원 범위 안 모든 콜라이더 배열 반환 |
| `LayerMask.GetMask("레이어명")` | 레이어 이름 → LayerMask 값 변환 |
| `Instantiate(prefab, position, rotation)` | 프리팹을 씬에 생성 |
| `transform.localScale.x` | 스케일 X가 음수이면 왼쪽을 보고 있음 (플립 방향 판별) |

### 쿨타임 로직 설명

```csharp
// 쿨타임 판별
if (Time.time < _nextUseTime[index]) return;  // 아직 쿨타임 중

// 사용 후 다음 사용 가능 시간 갱신
_nextUseTime[index] = Time.time + s.cooldown;
```

`Time.time`은 단조 증가한다. 스킬 사용 시점의 `Time.time + 쿨타임`을 저장해 두고, 현재 `Time.time`이 그 값보다 작으면 아직 쿨타임이라고 판단한다.

### 코드 (PlayerCombat.cs)

```csharp
using UnityEngine;

/// <summary>
/// 키 입력에 따라 스킬을 발동합니다.
/// 스킬의 수치와 에셋은 SkillData 배열에서 읽습니다.
/// 직접 수치를 가지지 않아 스킬 추가 시 코드 수정이 불필요합니다.
/// </summary>
public class PlayerCombat : MonoBehaviour
{
    [Header("스킬 설정")]
    [Tooltip("Inspector에서 SkillData 에셋을 순서대로 연결합니다")]
    public SkillData[] skills;

    [Tooltip("skills[i] 에 대응하는 키. 같은 인덱스끼리 매핑됩니다")]
    public KeyCode[] skillKeys;

    // 인덱스별 다음 사용 가능 시간 (Time.time 기준)
    private float[] _nextUseTime;

    void Start()
    {
        _nextUseTime = new float[skills != null ? skills.Length : 0];
    }

    void Update()
    {
        if (skills == null) return;
        for (int i = 0; i < skills.Length; i++)
        {
            if (skills[i] == null) continue;

            // skillKeys[i]가 없으면 None (발동 안 됨)
            KeyCode key = (skillKeys != null && i < skillKeys.Length)
                ? skillKeys[i] : KeyCode.None;

            if (key != KeyCode.None && Input.GetKeyDown(key))
                TryUseSkill(i);
        }
    }

    void TryUseSkill(int index)
    {
        if (index < 0 || index >= skills.Length || skills[index] == null) return;

        SkillData s = skills[index];

        // 쿨타임 확인: 현재 시간이 다음 사용 가능 시간보다 작으면 리턴
        if (Time.time < _nextUseTime[index]) return;

        // 다음 사용 가능 시간 갱신
        _nextUseTime[index] = Time.time + s.cooldown;

        // P3 스킬 슬롯 UI에 쿨타임 시작 알림
        GameEvents.RaiseSkillUsed(index, s.cooldown);

        DoSkill(s);
    }

    void DoSkill(SkillData s)
    {
        // 사운드 재생 (AudioManager가 있을 때만)
        if (s.soundClip != null)
            AudioManager.Instance?.PlaySFX(s.soundClip);

        // radius가 0이면 단타, 0 초과면 범위
        if (s.radius > 0f)
            DoAreaSkill(s);
        else
            DoMeleeSkill(s);
    }

    // 단타: 플레이어 앞쪽 박스 범위에서 적 1마리 판정
    void DoMeleeSkill(SkillData s)
    {
        // localScale.x 음수 = 왼쪽, 양수 = 오른쪽
        float dir = transform.localScale.x >= 0 ? 1f : -1f;
        Vector2 origin = (Vector2)transform.position + Vector2.right * dir * 0.5f;

        // OverlapBox: origin 중심, (range × 1) 크기 박스 안의 Enemy 레이어 콜라이더 1개
        Collider2D hit = Physics2D.OverlapBox(
            origin,
            new Vector2(s.range, 1f),
            0f,
            LayerMask.GetMask("Enemy")
        );

        if (hit != null)
        {
            var enemy = hit.GetComponent<EnemyController>();
            if (enemy != null) enemy.TakeDamage(s.damage);

            if (s.effectPrefab != null)
                Instantiate(s.effectPrefab, hit.transform.position, Quaternion.identity);
        }
        else
        {
            // 맞은 적이 없어도 이펙트는 앞쪽에 표시
            if (s.effectPrefab != null)
                Instantiate(s.effectPrefab, origin, Quaternion.identity);
        }
    }

    // 범위: 플레이어 주변 원 안의 모든 적에 데미지
    void DoAreaSkill(SkillData s)
    {
        // OverlapCircleAll: transform.position 중심, radius 반경의 모든 Enemy 콜라이더
        Collider2D[] hits = Physics2D.OverlapCircleAll(
            transform.position,
            s.radius,
            LayerMask.GetMask("Enemy")
        );

        foreach (var h in hits)
        {
            var enemy = h.GetComponent<EnemyController>();
            if (enemy != null) enemy.TakeDamage(s.damage);
        }

        // 이펙트는 플레이어 위치에 1개만 생성
        if (s.effectPrefab != null)
            Instantiate(s.effectPrefab, transform.position, Quaternion.identity);
    }
}
```

**파일**: `Assets/Scripts/PlayerCombat.cs`. Player에 부착. Inspector에서 **Skills** 배열에 스킬 에셋, **Skill Keys**에 X, Z, C. Enemy Layer는 **Enemy**로 설정.

## SkillSlotUI — 스킬 슬롯 HUD

### 개요

- **역할**: 스킬 슬롯 **하나**의 아이콘·키 라벨·**쿨타임 링**을 제어한다. `GameEvents.SkillUsed`를 구독해 `slotIndex`가 일치할 때만 쿨타임을 시작하고, `Update`에서 남은 시간 비율을 `fillAmount`로 그린다.
- **부착**: HUD/SkillSlots 아래의 각 `SkillSlot_0`, `SkillSlot_1`, `SkillSlot_2`에 **슬롯당 하나씩** 부착.
- **흐름**: `OnEnable`에서 SkillUsed 구독 → `OnSkillUsed(index, cooldown)`에서 index 일치 시 `_cooldownEnd` 설정 → `Update`에서 `fillAmount = remaining / total`.

### 채택한 방식과 이유

| 선택 | 이유 |
|------|------|
| **이벤트 구독** | PlayerCombat을 직접 참조해 매 프레임 쿨타임을 읽지 않는다. 스킬을 쓴 순간에만 이벤트가 오므로, 그때만 쿨타임 시작하고 Update에서는 남은 시간만 비율로 표시하면 된다. |
| **slotIndex** | 슬롯이 3개이고 각각 같은 스크립트를 쓰므로, "내가 몇 번 슬롯인지"를 Inspector에서 지정해 SkillUsed(index, …)의 index와 맞춘다. |
| **fillAmount로 쿨타임 링** | Image Type = Filled, Radial360이면 1→0으로 줄어드는 비율이 시계 반대 방향 링으로 보인다. 0이면 사용 가능. |

### SerializeField를 쓴 이유

| 필드 | 이유 |
|------|------|
| `slotIndex` | **슬롯 식별**. 0=X, 1=Z, 2=C와 매핑. 씬에서 슬롯마다 다르게 지정한다. |
| `iconImage` | **씬에서 연결**. 슬롯의 아이콘 Image 드래그 연결 |
| `cooldownFill` | **씬에서 연결**. Filled·Radial360 Image 드래그 연결 |
| `keyLabel` | **씬에서 연결**. "X", "Z", "C" 텍스트 표시용 |
| `skillData` | **씬에서 연결**. 이 슬롯에 표시할 스킬 에셋. 아이콘 초기 표시용 |

### 사용하는 Unity API (SkillSlotUI)

| API | 역할 |
|-----|------|
| `Image.fillAmount` | 0~1 값으로 쿨타임 링을 표시 (Radial360 모드) |
| `Image.type = Filled`, `fillMethod = Radial360` | 시계 방향으로 차오르는 원형 이미지 |
| `Time.time` | 쿨타임 남은 비율 계산용 |
| `GameEvents.SkillUsed` | 스킬 사용 알림 이벤트 (PlayerCombat이 발행) |

### 씬에서 슬롯 만들기

```
HUD
  └─ SkillSlots
        ├─ SkillSlot_0
        │    ├─ IconImage    (Image — 스킬 아이콘)
        │    ├─ CooldownFill (Image — Filled, Radial360, 반투명 검정)
        │    └─ KeyLabel     (TextMeshProUGUI — "X")
        ├─ SkillSlot_1  (위와 동일, 키 "Z")
        └─ SkillSlot_2  (위와 동일, 키 "C")
```

각 `SkillSlot_X`에 `SkillSlotUI.cs`를 하나씩 붙인다.

### 코드 (SkillSlotUI.cs)

**파일**: `Assets/Scripts/SkillSlotUI.cs`. SkillSlot_0/1/2 각각에 부착.

```csharp
using UnityEngine;
using UnityEngine.UI;
using TMPro;

/// <summary>
/// 스킬 슬롯 하나의 아이콘과 쿨타임 링을 제어합니다.
/// slotIndex가 일치하는 SkillUsed 이벤트를 받아서 쿨타임을 시작합니다.
/// </summary>
public class SkillSlotUI : MonoBehaviour
{
    [Header("슬롯 인덱스 (0=X, 1=Z, 2=C)")]
    [SerializeField] int slotIndex;

    [Header("UI 요소")]
    [SerializeField] Image iconImage;
    [SerializeField] Image cooldownFill;  // Image Type = Filled, Radial360
    [SerializeField] TMP_Text keyLabel;

    [Header("스킬 데이터 (Inspector에서 SkillData 연결)")]
    [SerializeField] SkillData skillData;

    float _cooldownEnd;   // 쿨타임이 끝나는 Time.time
    float _cooldownTotal; // 총 쿨타임 (비율 계산용)

    void Start()
    {
        // 아이콘과 키 표시 초기화
        if (skillData != null && iconImage != null)
            iconImage.sprite = skillData.icon;

        if (cooldownFill != null)
            cooldownFill.fillAmount = 0f;
    }

    void OnEnable()  { GameEvents.SkillUsed += OnSkillUsed; }
    void OnDisable() { GameEvents.SkillUsed -= OnSkillUsed; }

    void OnSkillUsed(int index, float cooldown)
    {
        // 내 슬롯 인덱스와 일치할 때만 처리
        if (index != slotIndex) return;
        _cooldownEnd = Time.time + cooldown;
        _cooldownTotal = cooldown;
        if (cooldownFill != null) cooldownFill.fillAmount = 1f;
    }

    void Update()
    {
        if (cooldownFill == null || _cooldownTotal <= 0f) return;

        float remaining = _cooldownEnd - Time.time;
        if (remaining > 0f)
        {
            // 남은 쿨타임 비율 → fillAmount (1=가득 = 쿨타임 시작, 0=비어있음 = 사용 가능)
            cooldownFill.fillAmount = remaining / _cooldownTotal;
        }
        else
        {
            cooldownFill.fillAmount = 0f;
            _cooldownTotal = 0f;
        }
    }
}
```

**에디터 연결**: Slot Index 0/1/2, Icon Image·Cooldown Fill·Key Label·Skill Data 각 슬롯에 드래그. Cooldown Fill은 Image Type = Filled, Radial360, Fill Origin = Top.
