# 02. 이동·점프 — 에셋

> **시스템**: 코어 시스템 | **에셋 담당**: D1 (플레이어 스프라이트)  
> 업무 분장 전체 기준 → [팀 역할 배분 문서](팀개발-역할배분-개발단계.md)

게임에서 "보이는 것"은 코드가 아니라 스프라이트가 담당한다. 플레이어 캐릭터가 화면에서 어떤 크기로, 어떤 느낌으로 보이는지는 스프라이트를 어떻게 만드느냐에 달려 있다. 이 단계에서는 플레이어 스프라이트를 만들고 Unity에 가져오는 과정을 다룬다.

## 스프라이트 크기 이론

스프라이트 크기를 결정하기 전에 **PPU(Pixels Per Unit)** 개념을 이해해야 한다.

Unity는 2D 좌표를 픽셀이 아닌 **unit(유닛)** 단위로 계산한다. PPU는 "1 unit이 몇 픽셀짜리 스프라이트인지"를 정의한다. 이 프로젝트는 PPU = 32를 기준으로 한다.

```
PPU = 32  →  32px 스프라이트 = 화면에서 1 unit 크기
PPU = 32  →  64px 스프라이트 = 화면에서 2 unit 크기
```

팀 전체가 PPU를 다르게 쓰면 같은 씬에서 오브젝트 크기가 제각각이 된다. 스프라이트를 만들기 전에 팀의 PPU 기준을 확인하고 맞춰야 한다.

### 크기별 용도

| 크기 | 화면 크기 (PPU 32 기준) | 주요 용도 |
|------|:----------------------:|----------|
| 16×16 px | 0.5 unit | 아이콘, 작은 이펙트, UI 장식 |
| 32×32 px | 1 unit | **기본 캐릭터, 타일** — 이 단계에서 사용 |
| 64×64 px | 2 unit | 조금 큰 캐릭터, 강조 요소 |
| 128×128 px | 4 unit | 보스, 대형 오브젝트 |

플레이어 캐릭터는 32×32px로 시작한다. 화면이 좁거나 캐릭터가 너무 작아 보이면 64×64px로 올린다.

## 파일 형식 — PNG를 써야 하는 이유

게임 스프라이트는 반드시 **PNG** 형식을 써야 한다.

- **PNG**: 투명 배경(Alpha 채널) 지원. 캐릭터 외곽 이외 부분이 투명하게 처리된다.
- **JPG**: 투명 배경 불가. 캐릭터 주변에 흰 사각형 박스가 같이 보인다.

도트 그래픽은 색상이 경계에서 갑자기 바뀌는데, JPG는 경계에서 색상을 보간(블러 처리)하기 때문에 픽셀이 흐릿해진다. PNG는 압축 없이 정확한 색상을 유지한다.

---

## Filter Mode — 도트 그래픽이 흐릿해지는 이유

Unity에서 스프라이트를 임포트하면 기본값이 `Bilinear`다. 이 설정은 스프라이트를 확대할 때 픽셀 사이를 부드럽게 보간(blend)한다. 사실적인 그래픽에서는 자연스러워 보이지만, 도트 그래픽은 픽셀 경계가 선명해야 도트 느낌이 살아난다.

`Point (no filter)`는 보간 없이 픽셀을 그대로 확대한다. 도트 아트를 쓰는 게임은 반드시 이 설정을 써야 한다.

---

## 스프라이트 스펙 요약

| 항목 | 값 | 이유 |
|------|---|------|
| 파일 형식 | PNG | 투명 배경 필수, 픽셀 정확도 유지 |
| 크기 | 32×32 px | PPU 32 기준 1 unit 크기 |
| Pixels Per Unit | 32 | 팀 전체 동일하게 유지 |
| Texture Type | Sprite (2D and UI) | Unity가 2D 스프라이트로 인식 |
| Filter Mode | Point (no filter) | 도트 그래픽 픽셀 선명하게 유지 |
| Compression | None | 압축 시 픽셀 색상 손상 가능 |

## Unity 임포트 설정

파일을 폴더에 넣은 뒤 Unity Project 창에서 스프라이트 파일을 선택하면 Inspector에 임포트 설정이 나타난다.

1. **Texture Type** → `Sprite (2D and UI)`
2. **Pixels Per Unit** → `32`
3. **Filter Mode** → `Point (no filter)`
4. **Compression** → `None`
5. 하단 **Apply** 클릭

## 에셋이 없을 때 — 플레이스홀더

에셋 제작이 늦어져도 코드 구현은 먼저 진행할 수 있다. Unity 기본 도형을 임시 비주얼로 쓴다.

1. Project 창 → `Create > Sprites > Square`
2. 이름: `placeholder_player`
3. Player 오브젝트의 `SpriteRenderer > Sprite`에 드래그

역할별로 색을 다르게 지정하면 씬에서 구분이 쉽다. `SpriteRenderer > Color`에서 변경한다.

| 오브젝트 | 색상 |
|---------|------|
| 플레이어 | 흰색 |
| 적 | 빨간색 |
| 바닥·플랫폼 | 회색 |

에셋이 완성되면 `SpriteRenderer > Sprite` 슬롯에 실제 스프라이트를 드래그해서 교체한다. 코드는 어떤 스프라이트가 들어있는지 신경 쓰지 않으므로 교체해도 동작은 변하지 않는다.
