## 1. 2D 도트 액션 플랫포머란?

**2D 도트 액션 플랫포머** — 다음 특징을 가진 게임입니다.  
참고 레퍼런스: **산나비(SANABI)**, 셀레스테, 할리우드 닌자 등.

| 특징 | 쉬운 설명 |
|------|-----------|
| **2D·도트(픽셀)** | 2D 화면에서 캐릭터·배경이 픽셀/도트 스타일로 보여요. |
| **플랫폼·점프** | 땅·발판(플랫폼) 위를 달리고 점프해서 다음 구역으로 이동해요. |
| **스테이지 진행** | 고정된 스테이지(레벨)를 차례대로 클리어하거나, 한 스테이지 안에서 목표를 달성해요. |
| **액션·전투** | 적을 공격해서 처치하고, 맞으면 HP가 줄어들어요. |
| **게임 오버·재시작** | HP가 0이 되면 게임 오버 후 다시 시작해요. |

초등학생이 이해하기 쉽게 **“플랫폼을 뛰어다니며 적을 때리는 2D 액션 게임”** 이라고 생각하면 됩니다.

😊 **한줄 조크**: 산나비처럼 “사슬로 날아다니기”까지는 어렵다면, **이동·점프·공격**만 있어도 플랫포머의 재미를 느낄 수 있어요. 하나씩 추가해 보세요.

## 1.1 이 프로젝트에서 사용하는 개발 영역·지식 (공통 기준)

이 프로젝트는 **경험**만 하는 것이 아니라, **어떤 개발 영역·어떤 지식**을 쓰는지 문서 곳곳에 표기합니다.  
아래 분류는 04(요구사항·시스템), 07(Phase별 개발), 08·09·10 등에서 **"여기서 배우는 것"**을 일관되게 가리킬 때 씁니다.

| 영역·지식 | 설명 | 이 프로젝트에서 쓰이는 곳 (예시) |
|-----------|------|----------------------------------|
| **데이터 설계** | 데이터 구조 정하기, 필드 정의, 테이블(에셋) 설계, 기획 수치→변수 매핑 | 04 시스템 설계, 05·08 데이터 에셋, ScriptableObject, 플레이어/적/맵 수치 |
| **알고리즘** | 그리드·맵 채우기, 방/복도 생성, 스폰 위치 결정, 적 AI(추적·순찰), 거리·판정 | 08 맵 생성, 07 Phase 2·3, 적 이동·공격 판정 |
| **소프트웨어 설계** | 모듈 나누기, 클래스 역할 분리, 상태 관리(게임 진행/오버), 이벤트 흐름, 코드·에셋 구조 | 04 모듈 목록, 07 Phase 0·4, GameManager, 폴더·네이밍 규칙 |

## 2. 목표 범위 (만들 게임의 범위)

- **대상**: 초등학생 (코딩/게임 제작 초급 가능)
- **플랫폼** (Platform, 플랫폼 [플랫폼]): 게임이 돌아가는 기기. 여기서는 PC (Windows/Mac).
- **엔진** (Engine, 엔진 [엔진]): 게임을 만드는 도구. **Unity** (유니티 [유니티]) LTS 버전 권장.

### 2.1 게임에서 꼭 넣을 기능 (필수)

| 번호 | 기능 | 설명 | 난이도 |

## 3. 사용자(플레이어) 관점 요구사항

- **조작**: 키보드 좌우 + 점프 + 공격 등 **버튼 3~5개 이내**
- **화면**: 2D 한 화면에 플랫폼·캐릭터·적이 구분되게
- **규칙**: “플랫폼을 점프하고 적을 피하거나 때려서 스테이지를 진행한다” 정도로 단순
- **난이도**: 처음엔 쉬운 적만, 나중에 강한 적·장애물 추가 가능

## 4. 개발자 관점 요구사항

- **Unity 버전**: Unity 6.3 LTS (이 문서 기준)
- **템플릿**: **2D** 프로젝트 권장 (픽셀/스프라이트에 적합)
- **언어**: C# 스크립트 (초등학생용으로 주석·변수명 한글 가능)
- **아트**: 리소스가 없어도 **대기하지 않고** 플레이스홀더(임시 비주얼)로 먼저 작업한 뒤, 준비되면 같은 슬롯만 교체. → [06 문서 §1.6](06-팀원-역할-분담.md#16-리소스-없을-때-플레이스홀더--교체)
- **개발 기간**: 기능별로 나누면 약 **8~12주** 분량 가정 (팀 규모에 따라 조정)
